This error occurs when a Laravel database migration fails during `php artisan migrate`. It is thrown as `Illuminate\Database\QueryException` when the underlying SQL statement cannot be executed.

## Common Causes

- Migration references a column or table that already exists
- Foreign key references a table that has not been created yet
- Column type mismatch with existing data
- Missing database permissions to alter tables
- Migration file contains invalid SQL syntax

## How to Fix

1. Check the SQL being generated by the migration:

```php
Schema::create('users', function (Blueprint $table) {
    $table->id();
    $table->string('email')->unique();
    $table->timestamps();
});

// Debug the query
$blueprint->build(new MySqlGrammar, new Connection);
```

2. Wrap migrations in try-catch for better error handling:

```php
use Illuminate\Database\QueryException;

try {
    Schema::table('users', function (Blueprint $table) {
        $table->foreign('team_id')->references('id')->on('teams');
    });
} catch (QueryException $e) {
    Log::error('Migration failed: ' . $e->getMessage());
    // Handle the error appropriately
}
```

3. Use `migrate:status` to check which migrations have run:

```bash
php artisan migrate:status
```

4. Rollback and retry if needed:

```bash
php artisan migrate:rollback
php artisan migrate
```

## Examples

```php
// Migration that references a table that doesn't exist yet
Schema::table('orders', function (Blueprint $table) {
    $table->foreignId('payment_id')->constrained();
    // QueryException if 'payments' table doesn't exist
});
```
