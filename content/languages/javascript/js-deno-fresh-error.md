## Why It Happens

Deno Fresh errors occur when the full-stack framework encounters routing conflicts, island hydration failures, or server/client code separation issues. Fresh uses Preact and islands architecture which has specific requirements.

Common causes include:
- Component not registered as an island for client-side interactivity
- Server-only code accidentally imported in client bundle
- Route file naming convention not matching Fresh expectations
- Tailwind CSS configuration missing for styling
- Middleware ordering causing request processing failures

## Common Error Messages

```
Error: Component is not an island. Did you forget to add the island directive?
```

```
TypeError: Cannot read property 'render' of undefined
```

```
Error: Route not found: /api/missing
```

## How to Fix It

### 1. Configure Fresh Project Structure

Set up proper file-based routing and islands.

```javascript
// fresh.config.js
import { defineConfig } from "$fresh/server.ts";
import tailwind from "$fresh/plugins/tailwind.ts";

export default defineConfig({
  plugins: [tailwind()],
  build: {
    target: "es2022",
  },
  server: {
    port: 8000,
    onListen: ({ port, hostname }) => {
      console.log(`Server listening on http://${hostname}:${port}`);
    },
  },
});

// routes/index.tsx
import { Handlers, PageProps } from "$fresh/server.ts";
import Counter from "../islands/Counter.tsx";

export const handler: Handlers = {
  async GET(req, ctx) {
    const data = await fetchData();
    return ctx.render(data);
  },
};

export default function Home({ data }: PageProps) {
  return (
    <div>
      <h1>Welcome to Fresh</h1>
      <Counter count={data.initialCount} />
    </div>
  );
}

// islands/Counter.tsx
import { useSignal } from "@preact/signals";

export default function Counter({ count }: { count: number }) {
  const counter = useSignal(count);
  
  return (
    <div>
      <p>Count: {counter.value}</p>
      <button onClick={() => counter.value++}>Increment</button>
    </div>
  );
}
```

### 2. Handle Server-Side Data Loading

Use Fresh handlers for data fetching.

```javascript
// routes/users/[id].tsx
import { Handlers, PageProps } from "$fresh/server.ts";
import { db } from "../../utils/db.ts";

interface User {
  id: string;
  name: string;
  email: string;
}

export const handler: Handlers<User | null> = {
  async GET(_, ctx) {
    const { id } = ctx.params;
    const user = await db.users.findUnique({ where: { id } });
    
    if (!user) {
      return ctx.render(null);
    }
    
    return ctx.render(user);
  },
  
  async POST(req, ctx) {
    const form = await req.formData();
    const name = form.get("name")?.toString();
    const email = form.get("email")?.toString();
    
    await db.users.update({
      where: { id: ctx.params.id },
      data: { name, email }
    });
    
    return new Response(null, {
      status: 302,
      headers: { Location: `/users/${ctx.params.id}` }
    });
  }
};

export default function UserPage({ data }: PageProps<User | null>) {
  if (!data) {
    return <h1>User not found</h1>;
  }
  
  return (
    <div>
      <h1>{data.name}</h1>
      <p>{data.email}</p>
    </div>
  );
}
```

### 3. Configure Middleware and Plugins

Set up Fresh middleware for authentication and logging.

```javascript
// fresh.gen.ts
// Auto-generated by Fresh - do not edit manually
import "$fresh/plugins/tailwind.ts";
import counter from "./islands/Counter.tsx";

export default {
  islands: { counter },
  routes: [],
};

// middleware.ts
import { MiddlewareHandler } from "$fresh/server.ts";

export const handler: MiddlewareHandler = async (req, next) => {
  const url = new URL(req.url);
  
  // Logging
  console.log(`${req.method} ${url.pathname}`);
  
  // Authentication
  if (url.pathname.startsWith("/admin")) {
    const session = req.headers.get("cookie");
    if (!session?.includes("authenticated=true")) {
      return new Response("Unauthorized", { status: 401 });
    }
  }
  
  // CORS
  const response = await next();
  response.headers.set("Access-Control-Allow-Origin", "*");
  
  return response;
};
```

## Common Scenarios

### Scenario 1: API Routes in Fresh

Creating API endpoints:

```javascript
// routes/api/users.ts
import { Handlers } from "$fresh/server.ts";

export const handler: Handlers = {
  async GET(req) {
    const url = new URL(req.url);
    const page = parseInt(url.searchParams.get("page") || "1");
    const limit = parseInt(url.searchParams.get("limit") || "10");
    
    const users = await getUsers(page, limit);
    
    return Response.json({
      users,
      page,
      limit,
      total: await getUserCount()
    });
  },
  
  async POST(req) {
    const body = await req.json();
    
    if (!body.name || !body.email) {
      return Response.json(
        { error: "Name and email are required" },
        { status: 400 }
      );
    }
    
    const user = await createUser(body);
    return Response.json(user, { status: 201 });
  }
};
```

## Prevent It

- Register interactive components as islands in `islands/` directory
- Use `export const handler` for server-side data loading in routes
- Keep client-side code in islands; server code in routes/utils
- Run `deno task start` to verify the project builds correctly
- Use Fresh's built-in Tailwind plugin for consistent styling
