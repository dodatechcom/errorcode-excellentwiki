# E0106: Missing Lifetime Specifier

The compiler cannot determine how long a reference should live — add an explicit lifetime annotation.

## Common Causes

```rust
// Compiler doesn't know which reference lives longer
struct Foo {
    x: &str,        // ERROR: E0106
}

fn bad(x: &str, y: &str) -> &str {  // ERROR: E0106
    if x.len() > y.len() { x } else { y }
}
```

## How to Fix

1. **Add explicit lifetime annotations**

```rust
struct Foo<'a> {
    x: &'a str,
}

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

2. **Use `'_` for simple cases**

```rust
impl Foo<'_> {
    fn name(&self) -> &str { self.x }
}
```

3. **Avoid returning references when possible**

```rust
fn longest_owned(x: &str, y: &str) -> String {
    if x.len() > y.len() { x.to_string() } else { y.to_string() }
}
```

## Examples

```rust
fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' { return &s[..i]; }
    }
    s
}

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

fn main() {
    let s = String::from("hello world");
    let word = first_word(&s);
    println!("First word: {}", word);

    let result = longest("abc", "ab");
    println!("Longest: {}", result);
}
```
