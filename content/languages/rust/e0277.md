# E0277: Trait Not Satisfied

A type does not implement a required trait — implement the trait or add bounds.

## Common Causes

```rust
fn print_item<T: std::fmt::Display>(item: T) { println!("{}", item); }
struct MyStruct;
print_item(MyStruct); // ERROR: MyStruct doesn't implement Display
```

## How to Fix

1. **Implement the trait**

```rust
use std::fmt;

struct MyStruct;

impl fmt::Display for MyStruct {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "MyStruct")
    }
}
```

2. **Use derive macros**

```rust
#[derive(Debug, Clone, PartialEq)]
struct Point { x: f64, y: f64 }
```

3. **Add trait bounds to generic code**

```rust
fn print_sorted<T: Ord + std::fmt::Debug>(mut items: Vec<T>) {
    items.sort();
    println!("{:?}", items);
}
```

## Examples

```rust
use std::fmt;

#[derive(Debug)]
struct Point { x: f64, y: f64 }

impl fmt::Display for Point {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "({}, {})", self.x, self.y)
    }
}

fn midpoint<T>(a: T, b: T) -> String
where
    T: std::ops::Add<Output = T> + std::ops::Div<i32, Output = T> + Copy + fmt::Display,
{
    let mid = (a + b) / 2;
    format!("{}", mid)
}

fn main() {
    let p = Point { x: 1.0, y: 2.0 };
    println!("{}", p);
}
```
