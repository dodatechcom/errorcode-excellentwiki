# E0666: `&` Not Allowed in Body

A `&` operator was used incorrectly in a function body — bitwise AND vs reference.

## Common Causes

```rust
fn check(x: bool, y: bool) -> bool {
    x & y // ERROR on some versions; use && for logical AND
}
```

## How to Fix

1. **Use logical operators**

```rust
fn check(x: bool, y: bool) -> bool {
    x && y // logical AND
}
```

2. **Use bitwise operators explicitly**

```rust
fn check(x: u32, y: u32) -> u32 {
    x & y // bitwise AND
}
```

3. **Use methods**

```rust
fn all_true(values: &[bool]) -> bool {
    values.iter().all(|&v| v)
}
```

## Examples

```rust
fn main() {
    let a = 0b1010;
    let b = 0b1100;
    let bitwise_and = a & b; // 0b1000 = 8
    let logical_and = true && false; // false
    println!("Bitwise: {:b}, Logical: {}", bitwise_and, logical_and);
}
```
