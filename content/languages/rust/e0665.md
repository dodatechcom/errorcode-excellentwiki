# E0665: const Contains References

A `const` or `static` contains references that may not have a `'static` lifetime.

## Common Causes

```rust
static REF: &str = &String::from("hello"); // ERROR: temporary
const ITEMS: &[i32] = &[1, 2, 3]; // ERROR on some versions
```

## How to Fix

1. **Use const evaluation**

```rust
const ITEMS: &[i32] = &[1, 2, 3]; // Works in modern Rust
```

2. **Use lazy_static or once_cell**

```rust
use std::sync::LazyLock;

static ITEMS: LazyLock<Vec<i32>> = LazyLock::new(|| vec![1, 2, 3]);
```

3. **Use &'static str**

```rust
const GREETING: &str = "Hello, World!"; // OK: string literal
```

## Examples

```rust
use std::sync::LazyLock;

static CONFIG: LazyLock<Vec<String>> = LazyLock::new(|| {
    vec!["debug".into(), "verbose".into()]
});

fn main() {
    for item in CONFIG.iter() {
        println!("Config: {}", item);
    }
}
```
