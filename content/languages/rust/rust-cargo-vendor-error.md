# Cargo Vendor Error

Cargo vendor errors occur when using `cargo vendor` to copy dependencies locally for offline builds. Issues include missing sources, checksum mismatches, and misconfigured paths.

## Common Causes

```bash
# Vendor directory not configured after running cargo vendor
$ cargo vendor vendor/
# Missing .cargo/config.toml to redirect crate sources
# Builds still try to fetch from crates.io

# Disk space or permission issues
$ cargo vendor
error: No space left on device
```

## How to Fix

1. **Run `cargo vendor` and configure `.cargo/config.toml`**

```bash
$ cargo vendor vendor/
$ mkdir -p .cargo
$ cargo vendor vendor/ > .cargo/config.toml
```

2. **Include vendor in version control for reproducible builds**

```bash
$ git add vendor/ .cargo/config.toml
$ git commit -m "vendor dependencies"
```

3. **Handle workspace vendoring from the workspace root**

```bash
$ cd workspace-root/
$ cargo vendor vendor/
$ cargo vendor vendor/ > .cargo/config.toml
```

## Examples

```bash
# Complete vendoring workflow
$ cargo vendor vendor/ 2>/dev/null
$ cargo vendor vendor/ > .cargo/config.toml
$ cargo build --offline

# Update vendored deps
$ rm -rf vendor/
$ cargo vendor vendor/ 2>/dev/null
$ cargo vendor vendor/ > .cargo/config.toml
```

```toml
# .cargo/config.toml generated by cargo vendor
[source.crates-io]
replace-with = "vendored-sources"

[source.vendored-sources]
directory = "vendor"
```
