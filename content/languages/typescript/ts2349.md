# TS2349 — This expression is not callable

TS2349 occurs when you try to call a value as a function using `()`, but its type doesn't have a call signature. This happens when a variable holds a non-function value or when a type is not callable.

## Common Causes

```typescript
// Cause 1: Calling a non-function value
const name = "Alice";
name(); // TS2349: This expression is not callable

// Cause 2: Calling an object as a function
const obj = { greet: "hello" };
obj(); // TS2349

// Cause 3: Calling a class instance as a function
class Greeter {
  greet() { return "hello"; }
}
const g = new Greeter();
g(); // TS2349

// Cause 4: Property that resolves to a non-function
interface Service {
  handler: string | (() => void);
}
function invoke(service: Service) {
  service.handler(); // TS2349: handler might be a string
}
```

## Solutions

### Fix 1: Verify the value is actually a function

```typescript
// Wrong
const name = "Alice";
name();

// Correct
const greet = () => "Hello, Alice!";
greet();
```

### Fix 2: Use a type guard before calling

```typescript
interface Service {
  handler: string | (() => void);
}

function invoke(service: Service) {
  if (typeof service.handler === "function") {
    service.handler(); // OK
  }
}
```

### Fix 3: Call the correct method

```typescript
// Wrong
const obj = { greet: "hello" };
obj();

// Correct
const obj = { greet: () => "hello" };
obj.greet();
```

### Fix 4: Ensure proper function assignment

```typescript
// Wrong — property is not a function
const config = {
  init: false,
};
config.init(); // TS2349

// Correct
const config = {
  init: () => console.log("initialized"),
};
config.init();
```

## Examples

```typescript
// Example 1: Module default export is not a function
import myModule from "./my-module";
myModule(); // TS2349 if default export is an object

// Example 2: Array methods confused with call
const arr = [1, 2, 3];
arr(); // TS2349: arrays are not callable
```
