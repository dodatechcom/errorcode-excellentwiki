# TS1128 — Declaration or statement expected

TS1128 occurs when TypeScript encounters a token where it expects a declaration (variable, function, class, etc.) or a statement. This often indicates a structural syntax problem like extra closing braces, misplaced code, or invalid top-level constructs.

## Common Causes

```typescript
// Cause 1: Extra closing brace
function greet() {
  console.log("hello");
}}  // TS1128: extra '}' — declaration or statement expected

// Cause 2: Code outside of a function or class at invalid location
function process() {}
  const x = 10;  // TS1128: unexpected indentation / misplaced code

// Cause 3: Missing function body or expression
function compute()  // TS1128: expected '{' or '=>'

// Cause 4: Invalid top-level expression
42  // TS1128 if not in a module or script context properly
```

## Solutions

### Fix 1: Remove extra closing braces

```typescript
// Wrong
function greet() {
  console.log("hello");
}}

// Correct
function greet() {
  console.log("hello");
}
```

### Fix 2: Ensure code is inside proper blocks

```typescript
// Wrong
function process() {}
  const x = 10; // outside the function

// Correct
function process() {
  const x = 10; // inside the function
}
```

### Fix 3: Complete function declarations

```typescript
// Wrong
function compute()

// Correct
function compute(): number {
  return 42;
}
```

### Fix 4: Check for misplaced imports or exports

```typescript
// Wrong — export outside module context
export default {} // TS1128 if file is not a module

// Correct — ensure the file has at least one import/export
import { something } from "./other";
export default {};
```

## Examples

```typescript
// Example 1: Accidentally closing a block early
if (condition) {
  doSomething();
}  // Extra close
doOtherThing(); // TS1128

// Example 2: Mixing declaration and expression
const obj = {
  method() {}
}
const x = 10  // Missing semicolon, may cause TS1128 on next line
```
