# TS5001 — Cannot compile modules with '--isolatedModules'

TS5001 occurs when the `--isolatedModules` flag is enabled but a file contains a construct that cannot be compiled in isolation. This flag requires every file to be a module (using `import` or `export`) and prohibits const enums and other cross-file optimizations.

## Common Causes

```typescript
// Cause 1: File is a script, not a module (no import/export)
// isolated-module.ts
const x = 5; // TS5001 — file is not a module

// Cause 2: Using const enum with isolatedModules
// isolatedModules doesn't allow const enums
const enum Color { Red, Green, Blue } // TS5001

// Cause 3: Using export = or import = require
// isolatedModules prohibits export assignments
export = SomeClass; // TS5001

// Cause 4: Ambient namespace declarations
declare namespace MyNS {
  export function f(): void;
}
```

## Solutions

### Fix 1: Add an import or export to make it a module

```typescript
// Wrong
const x = 5;

// Correct
export const x = 5;

// Or add an import
import { someHelper } from "./helper";
const x = 5;
```

### Fix 2: Convert const enums to regular enums

```typescript
// Wrong
const enum Color { Red, Green, Blue }

// Correct
enum Color { Red, Green, Blue }
```

### Fix 3: Replace export = with standard exports

```typescript
// Wrong
export = SomeClass;

// Correct
export default SomeClass;
// or
export { SomeClass };
```

### Fix 4: Disable isolatedModules if not needed

```json
// tsconfig.json
{
  "compilerOptions": {
    "isolatedModules": false
  }
}
```

## Examples

```typescript
// Example 1: Babel-transpiled project
// Babel compiles files one at a time — needs isolatedModules
// Solution: always use import/export in every file

// Example 2: Re-export barrel file
export { A, B } from "./a";
export { C } from "./b"; // OK
```
