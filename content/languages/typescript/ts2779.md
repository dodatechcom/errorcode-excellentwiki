# TS2779 — Unused '@ts-expect-error' directive

TS2779 occurs when you add a `@ts-expect-error` comment above a line that does not actually have a TypeScript error. This helps catch suppression comments that are no longer needed after code changes.

## Common Causes

```typescript
// Cause 1: Comment placed above a line without errors
// @ts-expect-error
const x = 5; // TS2779 — no error on this line

// Cause 2: The underlying error was fixed but comment wasn't removed
// @ts-expect-error
const result: string = "hello"; // Was an error, now it's correct

// Cause 3: Comment placed on the wrong line
const z: number = 5;
// @ts-expect-error
const y = z; // TS2779 — no error here, comment meant for previous line
```

## Solutions

### Fix 1: Remove the unused directive

```typescript
// Wrong
// @ts-expect-error
const x = 5;

// Correct
const x = 5;
```

### Fix 2: Replace with `@ts-ignore` if you need future-proof suppression

```typescript
// If you want to suppress future errors without getting TS2779:
// @ts-ignore
const x = 5; // OK — ts-ignore doesn't warn if unused
// But prefer ts-expect-error when you expect a specific error
```

### Fix 3: Move the directive to the correct line

```typescript
// Wrong
const z: number = 5;
// @ts-expect-error
const y: string = z;

// Correct
const z: number = 5;
// @ts-expect-error — z is number, not string
const y: string = z;
```

## Examples

```typescript
// Example 1: Used correctly
// @ts-expect-error — we know this returns string but type says unknown
const result: string = (someApi() as unknown);

// Example 2: Unused after fix
// Previously: const result: number = "hello"; // Error
// Now fixed:
// @ts-expect-error
const result: string = "hello"; // TS2779 — remove the comment

// Example 3: Comment on wrong line
// @ts-expect-error
const a = 1; // fine
const b: number = "hello"; // error — nothing suppresses it
```
