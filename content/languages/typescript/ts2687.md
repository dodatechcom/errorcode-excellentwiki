# TS2687 — Optional parameter cannot be both optional and have initializer

TS2687 occurs when a parameter has both an optional marker (`?`) and a default value (initializer). Since a default value already makes the parameter optional, the `?` is redundant and causes an error.

## Common Causes

```typescript
// Cause 1: Parameter with both ? and initializer
function greet(name: string = "World") { // OK
  // But if you add ?:
}
function greet(name?: string = "World") {} // TS2687

// Cause 2: Constructor parameter with both
class Service {
  constructor(private url?: string = "https://api.example.com") {} // TS2687
}

// Cause 3: Method parameter
class Calculator {
  add(a: number, b?: number = 0): number { // TS2687
    return a + b;
  }
}

// Cause 4: Arrow function parameter
const format = (value?: string = "N/A"): string => value; // TS2687

// Cause 5: Function type with both
type Formatter = (value?: string = "default") => string; // TS2687
```

## Solutions

### Solution 1: Remove the optional marker since the default value is sufficient

```typescript
// Wrong
function greet(name?: string = "World") {} // TS2687

// Correct
function greet(name: string = "World") {}
```

### Solution 2: Remove the default value if you want the parameter to truly be optional

```typescript
// Wrong
function greet(name?: string = "World") {} // TS2687

// Correct — if you want name to be undefined when not provided
function greet(name?: string) {
  const n = name ?? "World";
}
```

### Solution 3: Fix constructor parameters

```typescript
// Wrong
class Service {
  constructor(private url?: string = "https://api.example.com") {} // TS2687
}

// Correct
class Service {
  constructor(private url: string = "https://api.example.com") {}
}
```

### Solution 4: Fix method parameters

```typescript
// Wrong
class Calculator {
  add(a: number, b?: number = 0): number { // TS2687
    return a + b;
  }
}

// Correct
class Calculator {
  add(a: number, b: number = 0): number {
    return a + b;
  }
}
```

## Examples

```typescript
// Valid — default value makes parameter optional
function createUser(name: string, role: string = "user"): User {
  return { name, role };
}
createUser("Alice"); // role defaults to "user"
createUser("Bob", "admin");

// Valid — truly optional parameter
function log(message: string, level?: string): void {
  console.log(`[${level ?? "INFO"}] ${message}`);
}
log("Hello");
log("Error", "ERROR");
```
