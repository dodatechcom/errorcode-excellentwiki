# TS2374 — Duplicate string index signature

TS2374 occurs when an interface or type declares more than one string index signature. TypeScript only allows one string index signature per type. Duplicate signatures create ambiguity about what type indexed values should have.

## Common Causes

```typescript
// Cause 1: Two string index signatures on same interface
interface Config {
  [key: string]: string;
  [key: string]: number; // TS2374 — duplicate
}

// Cause 2: String index signature duplicated in extended interface
interface Base {
  [key: string]: string;
}

interface Child extends Base {
  [key: string]: string; // TS2374 — duplicate from parent
}

// Cause 3: String index signature in type literal
type Dict = {
  [key: string]: string;
  [key: string]: string; // TS2374
};

// Cause 4: Multiple index signatures with same key type
class Collection {
  [key: string]: string;
  [key: string]: string; // TS2374
}

// Cause 5: Accidental duplicate in complex interface
interface AppState {
  [key: string]: unknown;
  [key: string]: Record<string, unknown>; // TS2374
}
```

## Solutions

### Solution 1: Remove the duplicate index signature

```typescript
// Wrong
interface Config {
  [key: string]: string;
  [key: string]: number; // TS2374
}

// Correct — use a union type
interface Config {
  [key: string]: string | number;
}
```

### Solution 2: Use a single index signature with a broader type

```typescript
// Wrong
interface AppState {
  [key: string]: unknown;
  [key: string]: Record<string, unknown>; // TS2374
}

// Correct
interface AppState {
  [key: string]: unknown;
}
```

### Solution 3: Use named properties instead of multiple index signatures

```typescript
// Wrong
interface Config {
  [key: string]: string;
  [key: string]: number; // TS2374
}

// Correct — use named properties for specific keys
interface Config {
  [key: string]: string | number;
  // Or use specific properties:
  host: string;
  port: number;
}
```

## Examples

```typescript
// Valid — single string index signature
interface Headers {
  [key: string]: string;
}

// Valid — string index with named properties
interface Route {
  path: string;
  method: string;
  [key: string]: string; // All values must be string
}

// Valid — different index key types (string and number are different)
interface TypedArray {
  [index: number]: number;
  [key: string]: unknown; // This is actually not allowed — see TS2375
}
```
