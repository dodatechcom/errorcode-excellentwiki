# TS3070 — Unknown compiler option

TS3070 occurs when TypeScript encounters a compiler option in `tsconfig.json` or on the command line that it does not recognize. This usually means there's a typo, the option was added in a newer version, or the option was removed.

## Common Causes

```typescript
// Cause 1: Typo in the option name
// tsconfig.json
{
  "compilerOptions": {
    "strictNullChekcs": true // TS3070 — should be strictNullChecks
  }
}

// Cause 2: Using an option not available in the current TS version
{
  "compilerOptions": {
    "customConditions": ["browser"] // TS3070 — added in TS 4.7
  }
}

// Cause 3: Using a removed or deprecated option
{
  "compilerOptions": {
    "outDir": "./dist",
    "outFile": "./bundle.js" // TS3070 — removed in some versions
  }
}
```

## Solutions

### Fix 1: Fix typos in option names

```json
// Wrong
{
  "compilerOptions": {
    "strictNullChekcs": true
  }
}

// Correct
{
  "compilerOptions": {
    "strictNullChecks": true
  }
}
```

### Fix 2: Check TypeScript version compatibility

```bash
# Check your TypeScript version
npx tsc --version

# Consult the TypeScript release notes for available options
# Update TypeScript if needed
npm install typescript@latest
```

### Fix 3: Remove unsupported options

```json
// If an option is not available, remove it
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext"
    // Remove "customConditions" if not supported
  }
}
```

## Examples
