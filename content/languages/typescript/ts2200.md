# TS2200 — Unused '@ts-ignore' directive

TS2200 occurs when a `@ts-ignore` comment is present but the next line does not produce a TypeScript error. The directive is unnecessary and should be removed. This often happens after fixing the underlying error or when migrating to `@ts-expect-error`.

## Common Causes

```typescript
// Cause 1: The error was fixed but @ts-ignore was left behind
// @ts-ignore — was ignoring a type error that no longer exists
const x: number = 42; // TS2200 — no error to suppress

// Cause 2: Incorrect line placement — ignore is on the wrong line
function add(a: number, b: number): number {
  // @ts-ignore — this line is fine
  return a + b; // TS2200 — no error here
}

// Cause 3: @ts-ignore used on a line that doesn't produce errors
// @ts-ignore
const validString: string = "hello"; // TS2200

// Cause 4: Multiple @ts-ignore directives on the same error
// @ts-ignore
// @ts-ignore
const y: boolean = "not boolean"; // TS2200 — second one is unused

// Cause 5: @ts-ignore in a comment block where it's not recognized
/*
 * @ts-ignore
 */
const z = 10; // TS2200 — @ts-ignore inside block comment
```

## Solutions

### Solution 1: Remove the unused @ts-ignore

```typescript
// Wrong
// @ts-ignore
const x: number = 42; // TS2200

// Correct
const x: number = 42;
```

### Solution 2: Migrate to @ts-expect-error

```typescript
// Wrong — @ts-ignore silently suppresses
// @ts-ignore
const x: number = "not a number"; // No error, but dangerous

// Correct — @ts-expect-error will error if no error is suppressed
// @ts-expect-error — should report a type error
const x: number = "not a number";
```

### Solution 3: Verify @ts-ignore placement

```typescript
// Wrong — @ts-ignore is on the wrong line
function add(a: number, b: number): number {
  // @ts-ignore
  return a + b; // TS2200 — no error on this line
}

// Correct — place @ts-ignore directly above the error line
function add(a: string, b: number): number {
  // @ts-ignore
  return a + b; // TS2345 — error is suppressed
}
```

## Examples

```typescript
// Migrating from @ts-ignore to @ts-expect-error
// Before:
// @ts-ignore
document.getElementById("app").innerHTML = "Hello";

// After (if the error is fixed):
document.getElementById("app")!.innerHTML = "Hello";

// Or using @ts-expect-error (which fails if no error exists):
// @ts-expect-error — getElementById can return null
document.getElementById("app").innerHTML = "Hello";
```
