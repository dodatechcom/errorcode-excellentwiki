# TS2689 — Cannot extend an interface from a class in a different file

TS2689 occurs when an interface tries to extend a class that is defined in a different file. TypeScript only allows an interface to extend a class when they are in the same file, due to how private and protected members are handled.

## Common Causes

```typescript
// Cause 1: Interface in file A extends a class from file B
// file1.ts
export class BaseClass {
  protected x: number = 5;
}

// file2.ts
import { BaseClass } from "./file1";
interface ExtendedInterface extends BaseClass {} // TS2689

// Cause 2: Using a class-like type from another module as interface parent
// types.ts
export class Model { id: string = ""; }

// service.ts
import { Model } from "./types";
interface Product extends Model {} // TS2689
```

## Solutions

### Fix 1: Use a type alias instead of an interface

```typescript
// Wrong
interface ExtendedInterface extends BaseClass {}

// Correct
type ExtendedType = BaseClass & { extraProp: string };
```

### Fix 2: Move the class and interface to the same file

```typescript
// Same file:
export class BaseClass {
  protected x: number = 5;
}

export interface ExtendedInterface extends BaseClass {
  extraProp: string;
}
```

### Fix 3: Use interface extension of a structural type

```typescript
// Instead of extending the class, define a matching interface
interface BaseShape {
  x: number;
  // Replicate the public interface
}

interface ExtendedInterface extends BaseShape {
  extraProp: string;
}
```

### Fix 4: Use declaration merging

```typescript
// If the class is in the same file as your interface,
// you can augment it:
export class BaseClass { x: number = 5; }
export interface BaseClass { extraProp?: string; } // merging
```

## Examples

```typescript
// Example: Cannot extend a class from node_modules
import { EventEmitter } from "events";
interface MyEmitter extends EventEmitter {} // TS2689

// Fix: Use a type alias
type MyEmitter = EventEmitter & { customMethod(): void };
```
