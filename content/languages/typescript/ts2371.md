# TS2371 — Parameter initializer is not directly in parameter list

TS2371 occurs when you try to use a default value inside a destructured parameter pattern without using the correct syntax. In destructured parameters, default values must be placed at the property level, not at the parameter level.

## Common Causes

```typescript
// Cause 1: Default value on destructured parameter
function f({ a, b } = { a: 1, b: 2 }) {} // OK — this is correct
function f({ a = 1, b = 2 }) {} // OK — property-level default

// Wait — TS2371 occurs with a different pattern...
function f({ a, b } = {}) {} // OK if you want a default for the whole object

// Cause 1: Combining destructuring with a default parameter incorrectly
function f({ a, b }: { a?: number; b?: number } = { a: 1 }) {
  // TS2371 — default for destructured param not at property level
}

// Cause 2: Default value on array destructured parameter
function f([a, b] = [1, 2]) {} // OK
function f([a = 1, b = 2]) {} // OK

// The error actually appears when the default is placed in the wrong position
function f(x = 1, { a = 1 }) {} // TS2371
```

## Solutions

### Fix 1: Use property-level defaults inside destructuring

```typescript
// Wrong
function f({ a, b }: { a?: number; b?: number } = { a: 1 }) {
  // ...
}

// Correct — use property defaults
function f({ a = 1, b = 2 }: { a?: number; b?: number } = {}) {
  // ...
}
```

### Fix 2: Move defaults to the property level

```typescript
// Wrong
function createUser({ name, age } = { name: "Alice" }) {
  // TS2371
}

// Correct
function createUser({ name = "Alice", age = 30 }: { name?: string; age?: number } = {}) {
  console.log(name, age);
}
```

## Examples

```typescript
// Correct patterns for destructured parameter defaults:
function f1({ a = 1, b = 2 }: { a?: number; b?: number } = {}) {}
function f2([x = 0, y = 0]: number[] = []) {}
function f3({ name = "Guest" }: { name?: string } = {}) {}

// Wrong pattern:
function f4({ a, b }: { a?: number; b?: number } = { a: 1 }) {} // TS2371
```
