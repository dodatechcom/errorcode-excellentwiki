# TS2471 — Cannot redeclare exported function 'X'

TS2471 occurs when you declare the same exported function name more than once in the same scope or in conflicting ambient declarations. Each exported function must have a unique name within its module.

## Common Causes

```typescript
// Cause 1: Two exported functions with the same name in the same file
export function process() {}
export function process() {} // TS2471

// Cause 2: Ambient declarations with conflicting exports
declare module "my-lib" {
  export function helper(x: string): void;
}
declare module "my-lib" {
  export function helper(x: number): void; // TS2471
}

// Cause 3: Function declaration and export combined twice
function setup() {}
export { setup };
export function setup() {} // TS2471

// Cause 4: Redeclaration across script and module
// In a .d.ts file:
export function fetchData(): Promise<void>;
// Later in same file:
export function fetchData(): Promise<string>; // TS2471

// Cause 5: Multiple declarations merging in ambient module
declare global {
  function helper(): void;
}
declare global {
  function helper(): string; // TS2471
}
```

## Solutions

### Solution 1: Use function overloads instead

```typescript
// Wrong
export function process(input: string): void {}
export function process(input: number): void {} // TS2471

// Correct — use function overloads
export function process(input: string): void;
export function process(input: number): void;
export function process(input: string | number): void {
  // implementation
}
```

### Solution 2: Give the functions different names

```typescript
// Wrong
export function handle() {}
export function handle() {} // TS2471

// Correct
export function handleRequest() {}
export function handleResponse() {}
```

### Solution 3: Merge the ambient declarations properly

```typescript
// Wrong
declare module "my-lib" {
  export function helper(x: string): void;
}
declare module "my-lib" {
  export function helper(x: number): void; // TS2471
}

// Correct
declare module "my-lib" {
  export function helper(x: string): void;
  export function helper(x: number): void; // Overloads in same declaration
}
```

## Examples

```typescript
// Function overloads — valid
export function formatDate(date: Date): string;
export function formatDate(date: string): string;
export function formatDate(date: Date | string): string {
  if (typeof date === "string") {
    return date;
  }
  return date.toISOString();
}

// Different names — valid
export function createUser(name: string): User {
  return { name };
}

export function createAdmin(name: string, role: string): Admin {
  return { name, role };
}
```
