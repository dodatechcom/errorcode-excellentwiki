# TS7006 — Parameter 'X' implicitly has an 'any' type

TS7006 occurs when `noImplicitAny` is enabled (part of `strict` mode) and a function parameter has no type annotation and TypeScript can't infer its type. This forces you to explicitly declare types for all parameters.

## Common Causes

```typescript
// Cause 1: Untyped function parameter
function add(a, b) { // TS7006: Parameter 'a' implicitly has an 'any' type
  return a + b;
}

// Cause 2: Callback without type annotations
[1, 2, 3].map(x => x * 2); // TS7006 if strict mode is on

// Cause 3: Event handler without type
document.addEventListener("click", (event) => { // TS7006: event implicitly has 'any'
  console.log(event.target);
});

// Cause 4: Destructured parameters without types
function process({ name, age }) { // TS7006
  console.log(name, age);
}
```

## Solutions

### Fix 1: Add explicit type annotations

```typescript
// Wrong
function add(a, b) {
  return a + b;
}

// Correct
function add(a: number, b: number): number {
  return a + b;
}
```

### Fix 2: Type callback parameters

```typescript
// Wrong
[1, 2, 3].map(x => x * 2);

// Correct — TypeScript can usually infer this from the array type
// But if needed:
[1, 2, 3].map((x: number) => x * 2);
```

### Fix 3: Use typed DOM event handlers

```typescript
// Wrong
document.addEventListener("click", (event) => {});

// Correct
document.addEventListener("click", (event: MouseEvent) => {});
```

### Fix 4: Type destructured parameters

```typescript
// Wrong
function process({ name, age }) {}

// Correct
interface UserData {
  name: string;
  age: number;
}
function process({ name, age }: UserData) {}
```

## Examples

```typescript
// Example 1: Arrow function in array method
const doubled = [1, 2, 3].map(n => n * 2); // TS7006: 'n' implicitly has 'any'

// Example 2: Event listener
button.addEventListener("click", e => {
  e.target; // TS7006
});

// Fix: button.addEventListener("click", (e: MouseEvent) => { ... });
```
