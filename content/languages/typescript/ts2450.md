# TS2450 — Enum member must have initializer

TS2450 occurs in a `const enum` (or a regular enum with `isolatedModules`) when a member lacks an initializer. In certain contexts, TypeScript requires all enum members to have explicit values because it cannot compute the value from context.

## Common Causes

```typescript
// Cause 1: Const enum member without initializer
const enum Color {
  Red,    // TS2450 — needs initializer in some contexts
  Green,
  Blue,
}

// Cause 2: Ambient enum declaration without initializer
declare const enum Direction {
  Up,    // TS2450 in isolatedModules
  Down,
}

// Cause 3: Enum in a module with isolatedModules
// tsconfig: isolatedModules: true
export enum Status {
  Active,  // TS2450
  Inactive,
}
```

## Solutions

### Fix 1: Add explicit initializers to all members

```typescript
// Wrong
const enum Color {
  Red,
  Green,
  Blue,
}

// Correct
const enum Color {
  Red = 0,
  Green = 1,
  Blue = 2,
}
```

### Fix 2: Convert const enum to regular enum

```typescript
// Wrong
const enum Color { Red, Green, Blue }

// Correct
enum Color { Red, Green, Blue }
```

### Fix 3: Remove isolatedModules or use explicit values

```typescript
// If isolatedModules is required, provide explicit values:
export enum Status {
  Active = "active",
  Inactive = "inactive",
  Pending = "pending",
}
```

## Examples

```typescript
// Example 1: String enum in isolatedModules
export enum HttpMethod {
  GET,     // TS2450
  POST,
}

// Fix:
export enum HttpMethod {
  GET = "GET",
  POST = "POST",
}

// Example 2: Computed values are also OK
export enum HttpStatus {
  OK = 200,
  NotFound = 404,
  InternalError = 500,
}
```
