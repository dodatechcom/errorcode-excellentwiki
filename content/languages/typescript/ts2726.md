# TS2726 — Cannot use JSX unless '--jsx' flag is provided

TS2726 occurs when TypeScript encounters JSX syntax (`<Component />` or `<div>`) but the `jsx` compiler option is not enabled. TypeScript needs this flag to know how to handle JSX syntax in `.tsx` files.

## Common Causes

```typescript
// Cause 1: Using JSX in a .ts file
// File: component.ts (NOT .tsx)
const element = <div>Hello</div>; // TS2726

// Cause 2: Missing jsx option in tsconfig.json
// tsconfig.json without "jsx" setting
// File: component.tsx
const element = <div>Hello</div>; // TS2726

// Cause 3: Using JSX in a .tsx file but jsx is set to "none"
// tsconfig.json: { "jsx": "none" }
const element = <div>Hello</div>; // TS2726

// Cause 4: Comparison operator < in .tsx context
const x = 10;
const y = <number>5; // TS2726 — <number> looks like JSX
```

## Solutions

### Fix 1: Rename .ts files to .tsx

```typescript
// Wrong: component.ts
// component.ts
const element = <div>Hello</div>;

// Correct: component.tsx
// component.tsx
const element = <div>Hello</div>;
```

### Fix 2: Enable JSX in tsconfig.json

```json
// tsconfig.json
{
  "compilerOptions": {
    "jsx": "react-jsx" // or "react", "preserve", "react-native"
  }
}
```

### Fix 3: Use `as` syntax for type assertions instead of angle brackets

```typescript
// Wrong — angle bracket type assertion in .tsx
const y = <number>5;

// Correct
const y = 5 as number;
```

### Fix 4: Install React types if using React JSX

```bash
npm install @types/react @types/react-dom
```

## Examples

```typescript
// .tsx with jsx enabled:
import React from "react";

function App() {
  return <div className="app">Hello World</div>;
}

// If you get TS2726, check:
// 1. File extension is .tsx
// 2. tsconfig has "jsx" set
// 3. React types are installed (if using React)
```
