# TS2749 — 'X' refers to a value, but is being used as a type here

TS2749 occurs when you use a value (like a class instance or variable) in a type position where TypeScript expects a type. This typically happens when importing a value instead of a type, or confusing class instances with class types.

## Common Causes

```typescript
// Cause 1: Using a class instance as a type annotation
class User {
  name: string = "";
}
const user = new User();
type MyType = typeof user; // This is the instance type

let x: user; // TS2749: 'user' refers to a value, but is being used as a type

// Cause 2: Importing a value instead of a type
import MyComponent from "./MyComponent";
type Props = MyComponent; // TS2749 if MyComponent is a value

// Cause 3: Using a variable name as a type
const config = { host: "localhost" };
type T = config; // TS2749

// Cause 4: Forgetting to use 'typeof' for value-to-type conversion
function process(input: string) {}
const fn = process;
type FnType = fn; // TS2749
```

## Solutions

### Fix 1: Use 'typeof' to convert a value to its type

```typescript
class User {
  name: string = "";
}
const user = new User();

// Wrong
let x: user; // TS2749

// Correct
let x: typeof user; // OK — x has the same shape as user

// Better — use the class name directly
let x: User; // OK — this is the instance type
```

### Fix 2: Import types using 'import type'

```typescript
// Wrong — imports the value
import MyComponent from "./MyComponent";

// Correct — imports only the type
import type MyComponent from "./MyComponent";
type Props = MyComponent; // OK
```

### Fix 3: Use the class name as the type

```typescript
class User {
  name: string = "";
}

// Wrong — use the class name, not an instance
const user = new User();
function process(u: user) {} // TS2749

// Correct
function process(u: User) {} // OK
```

### Fix 4: Extract the type from a variable using typeof

```typescript
const config = { host: "localhost", port: 3000 };

// Wrong
type T = config;

// Correct
type T = typeof config; // { host: string; port: number }
```

## Examples

```typescript
// Example 1: Common mistake with module exports
// utils.ts
export const helper = () => {};
export type HelperType = { name: string };

// app.ts
import { helper, HelperType } from "./utils";
type T = helper; // TS2749 — helper is a value
type T2 = HelperType; // OK — HelperType is a type

// Example 2: Using enum as a type
enum Color { Red, Green, Blue }
type T = Color; // TS2749 — Color is a value (enum)
type T2 = typeof Color; // OK
```
