# TS2589 — Type instantiation is excessively deep and possibly infinite

TS2589 occurs when TypeScript detects that type evaluation is recursing too deeply, typically due to complex conditional types, recursive type aliases, or deeply nested generics. TypeScript has a built-in recursion limit to prevent infinite type computations.

## Common Causes

```typescript
// Cause 1: Recursive type alias without base case
type Infinite<T> = Infinite<T>; // TS2589

// Cause 2: Deeply nested conditional types
type Deep<T> = T extends number
  ? { value: T }
  : T extends string
    ? { value: T }
    : Deep<Extract<T, number>>; // TS2589 — too deep

// Cause 3: Recursive mapped types
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object ? DeepPartial<T[K]> : T[K];
};
// Fine for shallow objects, but deep nesting triggers TS2589

// Cause 4: Template literal type recursion
type Repeat<S extends string, N extends number> = N extends 0
  ? ""
  : `${S}${Repeat<S, Subtract<N, 1>>}`; // TS2589 if N is large
```

## Solutions

### Fix 1: Add a base case to recursive types

```typescript
// Wrong
type Flatten<T> = T extends Array<infer U> ? Flatten<U> : [T];

// Correct — add base case
type Flatten<T> = T extends Array<infer U>
  ? U extends Array<any> ? Flatten<U> : [U]
  : [T];
```

### Fix 2: Reduce nesting depth

```typescript
// Instead of deep nesting, flatten your types
type Result<T> = T extends string
  ? StringResult
  : T extends number
    ? NumberResult
    : DefaultResult;

// Avoid chaining DeepPartial on deeply nested objects
type Config = DeepPartial<DeeplyNestedConfig>;
// Break it into smaller parts instead
```

### Fix 3: Use simpler type constructs

```typescript
// Wrong — overly complex mapped type
type Complex<T> = {
  [K in keyof T]: T[K] extends (...args: infer A) => infer R
    ? (...args: A) => Promise<R>
    : T[K];
};

// Simplify by processing fewer properties at once
```

## Examples

```typescript
// Example 1: Deeply nested Promise unwrapping
type Unwrap<T> = T extends Promise<infer U> ? Unwrap<U> : T;
// Fine for Promise<Promise<string>>, but fails for deeply nested

// Fix: limit depth explicitly
type Unwrap1<T> = T extends Promise<infer U> ? U : T;
type Unwrap2<T> = T extends Promise<infer U> ? Unwrap1<U> : T;
```
