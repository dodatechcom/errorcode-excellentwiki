# TS2484 — Export assignment is not allowed with module augmentation

TS2484 occurs when you use `export =` (CommonJS-style default export) in a module augmentation (`declare module`). Module augmentation uses named exports only.

## Common Causes

```typescript
// Cause 1: export = in module augmentation
declare module "my-lib" {
  export = MyLib; // TS2484 — not allowed in augmentation
  class MyLib {
    static create(): MyLib;
  }
}

// Cause 2: Combining export = with augmentation
declare module "my-lib" {
  interface Config {
    debug: boolean;
  }
}
declare module "my-lib" {
  export = MyLib; // TS2484
  class MyLib {}
}

// Cause 3: In a .d.ts file augmenting a third-party module
declare module "express" {
  export = express; // TS2484 — can't use export = in augmentation
}

// Cause 4: Attempting to set default export via augmentation
declare module "lodash" {
  export = _; // TS2484
  const _: any;
}

// Cause 5: Mixed export = and module augmentation in same file
import express from "express";
declare module "express" {
  export = express; // TS2484
}
```

## Solutions

### Solution 1: Use named exports in module augmentation

```typescript
// Wrong
declare module "my-lib" {
  export = MyLib; // TS2484
  class MyLib {}
}

// Correct
declare module "my-lib" {
  export class MyLib {
    static create(): MyLib;
  }
}
```

### Solution 2: Augment with named exports only

```typescript
// Wrong
declare module "my-lib" {
  export = myLib; // TS2484
}

// Correct
declare module "my-lib" {
  export interface Config {
    debug: boolean;
  }
  export function helper(): void;
}
```

### Solution 3: Use namespace merging for default-like exports

```typescript
// Wrong
declare module "my-lib" {
  export = MyLib; // TS2484
}

// Correct
declare module "my-lib" {
  export default class MyLib {}
  // or
  export class MyLib {}
  export default MyLib;
}
```

## Examples

```typescript
// Valid module augmentation
declare module "express" {
  interface Request {
    userId?: string;
  }
}

// Valid — adding named exports via augmentation
declare module "my-lib" {
  export interface ExtendedConfig {
    apiKey: string;
  }
  export function configure(config: ExtendedConfig): void;
}

// For adding a default export, use export default
declare module "my-lib" {
  export default class MyLib {
    static create(): MyLib;
  }
}
```
