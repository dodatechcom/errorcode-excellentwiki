# TS1109 — Expression expected

TS1109 is a parse error indicating TypeScript encountered a token where it expected an expression. This usually happens with malformed function calls, missing operands, or incomplete code.

## Common Causes

```typescript
// Cause 1: Empty function call argument
const result = Math.random()  // Missing closing paren or argument

// Cause 2: Missing right operand
let x = 5 + ;  // TS1109: expression expected after '+'

// Cause 3: Incomplete arrow function
const fn = () =>  // TS1109: expression expected after '=>'

// Cause 4: Missing value in assignment
const obj = {
  key:   // TS1109: expression expected after ':'
};
```

## Solutions

### Fix 1: Complete the expression

```typescript
// Wrong
let x = 5 + ;

// Correct
let x = 5 + 10;
```

### Fix 2: Close parentheses and provide arguments

```typescript
// Wrong
const result = Math.random()

// Correct
const result = Math.random();
```

### Fix 3: Provide a return value for arrow functions

```typescript
// Wrong
const fn = () =>

// Correct
const fn = () => 42;

// Or with block body
const fn = () => {
  return 42;
};
```

### Fix 4: Complete object property values

```typescript
// Wrong
const obj = {
  key:
};

// Correct
const obj = {
  key: "value"
};
```

## Examples

```typescript
// Example 1: Incomplete template literal
const name = `Hello, ${;  // TS1109

// Fix:
const name = `Hello, ${user.name}`;

// Example 2: Missing argument in new expression
const el = new HTMLElement();  // If HTMLElement isn't defined
// TS1109 if the expression can't be resolved
```
