# TS1212 — Identifier expected with 'import' declaration

TS1212 occurs when an `import` declaration is missing a valid identifier. This typically happens when the import specifier is malformed, or when an `import =` statement is missing its left-hand side.

## Common Causes

```typescript
// Cause 1: Missing identifier in import = statement
import = require("./myModule"); // TS1212

// Cause 2: Empty destructured import
import { } from "./myModule"; // TS1212 in some contexts

// Cause 3: Invalid syntax in import statement
import "MyClass" from "./myClass"; // TS1212 — wrong syntax

// Cause 4: Missing variable name in default import
import from "./myModule"; // TS1212

// Cause 5: Import assignment without identifier
import "./module"; // Not TS1212, but if written as: import = ...  (missing name)
```

## Solutions

### Solution 1: Provide the correct identifier

```typescript
// Wrong
import = require("./myModule"); // TS1212

// Correct
import myModule = require("./myModule");
```

### Solution 2: Fix import syntax

```typescript
// Wrong
import "MyClass" from "./myClass"; // TS1212

// Correct — default import
import MyClass from "./myClass";

// Correct — named import
import { MyClass } from "./myClass";
```

### Solution 3: Provide a name for default imports

```typescript
// Wrong
import from "./myModule"; // TS1212

// Correct
import myModule from "./myModule";
```

## Examples

```typescript
// Valid import = require syntax
import express = require("express");
const app = express();

// Valid named import
import { readFile, writeFile } from "fs";

// Valid default import
import React from "react";

// Valid side-effect import
import "core-js/stable";
```
