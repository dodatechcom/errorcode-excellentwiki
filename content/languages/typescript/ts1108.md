# TS1108 — A 'return' statement can only be used within a function body

The `return` keyword is only valid inside a function, method, getter, setter, or arrow function. Using it at the top level of a module, class, or block causes TS1108.

## Common Causes

```typescript
// Cause 1: return at module level
const x = 5;
return x; // TS1108

// Cause 2: return inside a class body (outside a method)
class MyClass {
  private value = 10;
  return this.value; // TS1108
}

// Cause 3: return inside an if/for block that is not in a function
if (true) {
  return 42; // TS1108
}

// Cause 4: return outside a function in a module
export const result = "done";
return result; // TS1108

// Cause 5: Accidental return in a non-function context
const doWork = () => {
  // this is fine — inside arrow function
};
return "oops"; // TS1108 — outside the arrow function body
```

## Solutions

### Solution 1: Wrap the return statement in a function

```typescript
// Wrong
const x = 5;
return x; // TS1108

// Correct
function getX(): number {
  const x = 5;
  return x;
}
```

### Solution 2: Move return inside a method

```typescript
// Wrong
class MyClass {
  value = 10;
  return this.value; // TS1108
}

// Correct
class MyClass {
  value = 10;
  getValue(): number {
    return this.value;
  }
}
```

### Solution 3: Remove the return at module level

```typescript
// Wrong
const result = computeSomething();
return result; // TS1108

// Correct
const result = computeSomething();
export default result;
```

## Examples

```typescript
// Using return inside an IIFE — this is valid
const result = (function () {
  const data = [1, 2, 3];
  return data.filter(n => n > 1); // OK — inside function body
})();

// Using return in a top-level await context — not valid
await fetch("https://api.example.com"); // OK
return await response.json(); // TS1108 — not in a function

// Correct — wrap in an async function
async function fetchData() {
  const response = await fetch("https://api.example.com");
  return await response.json(); // OK
}
```
