# TS1127 — Invalid character

TS1127 occurs when TypeScript encounters a character that is not valid in the current context. This often happens with non-printable characters, Unicode homoglyphs, smart quotes, or other invisible characters that look like valid syntax but aren't.

## Common Causes

```typescript
// Cause 1: Smart/curly quotes instead of straight quotes
const name = “Alice”; // TS1127 — opening curly quote
const name = ”Alice”; // TS1127 — closing curly quote

// Cause 2: Non-breaking space or zero-width characters
const name = "Alice"; // TS1127 — non-breaking space before semicolon

// Cause 3: Em-dash or en-dash instead of minus
let x = 5 – 3; // TS1127 — en-dash instead of minus

// Cause 4: Right-to-left override or other control characters
// Hidden Unicode control characters in source
```

## Solutions

### Fix 1: Replace smart quotes with straight quotes

```typescript
// Wrong
const greeting = “Hello”;

// Correct
const greeting = "Hello";
```

### Fix 2: Fix dashes and hyphens

```typescript
// Wrong
let x = 5 – 3; // en-dash
let y = 5 — 3; // em-dash

// Correct
let x = 5 - 3; // standard hyphen-minus
```

### Fix 3: Remove invisible Unicode characters

```typescript
// Remove non-printing characters:
// - Zero-width space (U+200B)
// - Zero-width non-joiner (U+200C)
// - Non-breaking space (U+00A0)
// - BOM (U+FEFF)

// Use a linter or editor plugin to highlight invisible characters.
```

## Examples

```typescript
// Example 1: Copied code from a word processor
// Word processors often convert quotes and dashes
const result = 42; // OK with straight quotes

// Example 2: BOM in file
// Some editors add a BOM at the start of UTF-8 files
// TypeScript may show TS1127 on the first line

// Example 3: Zero-width space in identifier
const my\u200BVar = 5; // TS1127
const myVar = 5; // Correct
```
