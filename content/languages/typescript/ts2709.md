# TS2709 — Cannot use namespace as a type

TS2709 occurs when you try to use a namespace (or module) as a type annotation. Namespaces are value-level constructs containing other declarations, and they cannot be used directly as types.

## Common Causes

```typescript
// Cause 1: Using a namespace as a type
namespace MyNamespace {
  export interface Config { url: string; }
}
const config: MyNamespace = { url: "..." }; // TS2709

// Cause 2: Using a namespace as a parameter type
namespace Utils {
  export function log(msg: string) {}
}
function useUtils(u: Utils) {} // TS2709

// Cause 3: Referring to the namespace instead of its exported type
namespace API {
  export interface Response { data: string; }
}
let response: API = {}; // TS2709 — use API.Response instead
```

## Solutions

### Fix 1: Use an exported type from the namespace

```typescript
// Wrong
const config: MyNamespace = { url: "..." };

// Correct
const config: MyNamespace.Config = { url: "..." };
```

### Fix 2: Use the namespace's interface or class inside it

```typescript
namespace Shapes {
  export interface Circle { radius: number; }
  export interface Square { size: number; }
}

// Use the specific type
let shape: Shapes.Circle = { radius: 5 };
```

### Fix 3: Convert namespace to a module with exported types

```typescript
// Instead of a namespace, use a module:
export interface Config { url: string; }
export function helper() {}

// Then import:
import { Config } from "./utils";
const config: Config = { url: "..." };
```

## Examples

```typescript
// Example: jQuery-style namespace pattern
declare namespace $ {
  function ajax(): void;
  interface AjaxOptions { url: string; }
}

// Wrong:
const opts: $ = { url: "..." }; // TS2709

// Correct:
const opts: $.AjaxOptions = { url: "..." };
```
