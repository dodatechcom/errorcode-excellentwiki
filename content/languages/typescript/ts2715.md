# TS2715 — Abstract methods can only appear in an abstract class

TS2715 occurs when you declare an abstract method inside a non-abstract class. Abstract methods are only allowed in classes marked with the `abstract` keyword. Non-abstract classes must provide implementations for all their methods.

## Common Causes

```typescript
// Cause 1: Abstract method in a concrete class
class Base {
  abstract doSomething(): void; // TS2715
}

// Cause 2: Abstract method without abstract class modifier
class Service {
  abstract initialize(): Promise<void>; // TS2715
}

// Cause 3: Forgetting the abstract keyword on the class
abstract class Base {
  abstract doSomething(): void; // OK
}
class Derived extends Base {} // TS2515: non-abstract class missing implementation
```

## Solutions

### Fix 1: Mark the class as abstract

```typescript
// Wrong
class Base {
  abstract doSomething(): void;
}

// Correct
abstract class Base {
  abstract doSomething(): void;
}
```

### Fix 2: Provide a concrete implementation

```typescript
// Wrong
class Base {
  abstract doSomething(): void;
}

// Correct — provide implementation
class Base {
  doSomething(): void {
    console.log("default implementation");
  }
}
```

### Fix 3: Convert to an interface

```typescript
// Instead of abstract in concrete class, use an interface
interface Base {
  doSomething(): void;
}

class Implementation implements Base {
  doSomething(): void {
    console.log("implemented");
  }
}
```

## Examples

```typescript
// Example 1: Abstract class with abstract methods
abstract class Vehicle {
  abstract start(): void;
  abstract stop(): void;
  honk(): void { console.log("beep"); }
}

class Car extends Vehicle {
  start(): void { console.log("car started"); }
  stop(): void { console.log("car stopped"); }
}
// Car is OK — it implements all abstract methods
```
