# TS2352 — Conversion of type 'X' to type 'Y' may be a mistake

TS2352 occurs when you use a type assertion (`as` or angle brackets) to convert between two unrelated types. TypeScript warns that the assertion might be incorrect because the types don't overlap structurally. This is stricter in `--strict` mode.

## Common Causes

```typescript
// Cause 1: Asserting between unrelated types
const value: string = "hello";
const num: number = value as number; // TS2352: Conversion may be a mistake

// Cause 2: Asserting object literals to incompatible types
interface User {
  name: string;
  age: number;
}
const data = { x: 1, y: 2 };
const user = data as User; // TS2352: types don't overlap

// Cause 3: Using angle bracket assertion
const value = "hello";
const num = <number>value; // TS2352

// Cause 4: Assertion between primitive types
let a: boolean = true;
let b: number = a as number; // TS2352
```

## Solutions

### Fix 1: Use a more appropriate assertion

```typescript
// Wrong
const value: string = "123";
const num: number = value as number; // TS2352

// Correct — use actual conversion
const num: number = Number(value);
const num2: number = parseInt(value, 10);
```

### Fix 2: Use a type guard instead of assertion

```typescript
function isUser(obj: unknown): obj is User {
  return (
    typeof obj === "object" &&
    obj !== null &&
    "name" in obj &&
    "age" in obj
  );
}

const data = { name: "Alice", age: 30 };
if (isUser(data)) {
  // data is User
}
```

### Fix 3: Use unknown as intermediate type

```typescript
// Wrong
const value: string = "hello";
const num: number = value as number;

// Correct — go through unknown
const intermediate: unknown = value;
// Then narrow or convert properly
```

### Fix 4: Ensure structural compatibility

```typescript
// If types need to be compatible, ensure they share structure
interface Source {
  name: string;
  age: number;
}
interface Target {
  name: string;
  age: number;
}
const source: Source = { name: "Alice", age: 30 };
const target: Target = source as Target; // OK — structural match
```

## Examples

```typescript
// Example 1: API response typed incorrectly
interface ApiResponse {
  data: { id: number; name: string };
}
const response = await fetch("/api");
const json = await response.json();
const result = json as ApiResponse; // TS2352 — json is 'any'

// Example 2: DOM element type mismatch
const div = document.getElementById("app");
const canvas = div as HTMLCanvasElement; // TS2352 if strict
```
