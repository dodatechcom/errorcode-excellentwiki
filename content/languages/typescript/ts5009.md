# TS5009 — File name differs from already included file name

TS5009 occurs when TypeScript detects two file paths that differ only in casing (e.g., `User.ts` vs `user.ts`) on a case-sensitive file system. TypeScript treats these as potentially conflicting, especially when the `--forceConsistentCasingInFileNames` option is enabled.

## Common Causes

```typescript
// Cause 1: Two files with same name but different casing
// File: /src/User.ts
// File: /src/user.ts  // TS5009 — differs in casing

// Cause 2: Import uses different casing than actual file
// File: /src/helpers.ts
import { helper } from "./Helpers"; // TS5009 — Helpers vs helpers

// Cause 3: Renamed file on case-insensitive OS then pushed
// On macOS/Windows, User.ts and user.ts are the same file
// On Linux, they are different — TS5009

// Cause 4: Git renamed file with different casing
```

## Solutions

### Fix 1: Ensure consistent file name casing

```typescript
// Use one casing convention throughout
// PascalCase for components: UserProfile.ts
// camelCase for utilities: formatDate.ts
// kebab-case for config: api-config.ts

// Fix imports to match actual file casing
// File: /src/helpers/format.ts
import { format } from "./helpers/Format"; // Wrong
import { format } from "./helpers/format"; // Correct
```

### Fix 2: Remove duplicate files

```bash
# On Linux, check for casing duplicates
ls /src/User* /src/user*

# Remove the incorrect one
rm /src/user.ts
```

### Fix 3: Use forceConsistentCasingInFileNames

```json
{
  "compilerOptions": {
    "forceConsistentCasingInFileNames": true
  }
}
```

### Fix 4: For git users, fix casing on case-insensitive OS

```bash
# On macOS/Windows:
git mv src/user.ts src/User_temp.ts
git mv src/User_temp.ts src/User.ts
```

## Examples

```typescript
// Example 1: Import casing mismatch
// File exists as: /src/services/AuthService.ts
import { AuthService } from "./services/authService"; // TS5009

// Fix:
import { AuthService } from "./services/AuthService";
```
