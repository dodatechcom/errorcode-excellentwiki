# TS2497 — Only named exports may use 'export type'

TS2497 occurs when you use `export type` with a default export or a non-named export. The `type` modifier only works with named exports (i.e., `export type { X }` or `export type X = ...`).

## Common Causes

```typescript
// Cause 1: Using export type with default export
type Internal = string;
export type default Internal; // TS2497

// Cause 2: Using export type with export = (CommonJS)
type Config = { debug: boolean };
export type = Config; // TS2497

// Cause 3: Using export type in a re-export with default
export type { default } from "./module"; // TS2497

// Cause 4: Using export type with a namespace export
type Helpers = {
  formatDate: (d: Date) => string;
};
export type Helpers; // TS2497 — must be: export type { Helpers };

// Cause 5: Attempting inline type export of a default
export type default class MyClass {} // TS2497
```

## Solutions

### Solution 1: Use named exports with export type

```typescript
// Wrong
type Internal = string;
export type default Internal; // TS2497

// Correct
export type Internal = string;
// or
type Internal = string;
export type { Internal };
```

### Solution 2: Re-export with named exports

```typescript
// Wrong
export type { default } from "./module"; // TS2497

// Correct
export type { SomeNamedExport } from "./module";
// For default, use:
import type DefaultExport from "./module";
export type { DefaultExport };
```

### Solution 3: Remove the type modifier if not needed

```typescript
// Wrong
type Config = { debug: boolean };
export type = Config; // TS2497

// Correct
type Config = { debug: boolean };
export default Config;
// or
export { Config };
```

## Examples

```typescript
// Valid type-only exports
export type { User, UserProps };
export type UserID = string | number;

// Valid — inline type export
export type Status = "active" | "inactive";

// Valid — type re-export from another module
export type { ApiResponse } from "./api";

// Invalid — cannot use export type with default
// export type { default as MyType } from "./types"; // TS2497
```
