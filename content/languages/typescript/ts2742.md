# TS2742 — The inferred type of X references a local type Y that is not exported

TS2742 occurs when a public API (exported function, variable, or class) has an inferred return type or type that references a type not exported from the module. TypeScript requires all types reachable from public APIs to be exported.

## Common Causes

```typescript
// Cause 1: Function returns a type from a private interface
interface InternalResult {
  data: string;
}

export function process(): InternalResult {
  // TS2742 — InternalResult is not exported
  return { data: "hello" };
}

// Cause 2: Exported variable with inline type referencing a local type
type LocalState = { count: number };
export const state: LocalState = { count: 0 }; // TS2742

// Cause 3: Class method returns a type referencing a private type
class Internal { }
export class Service {
  create(): Internal { return new Internal(); } // TS2742
}
```

## Solutions

### Fix 1: Export the referenced type

```typescript
// Wrong
interface InternalResult { data: string; }
export function process(): InternalResult { return { data: "hello" }; }

// Correct
export interface InternalResult { data: string; }
export function process(): InternalResult { return { data: "hello" }; }
```

### Fix 2: Explicitly annotate the return type without the local type

```typescript
// Instead of using the local type, use a structural type
export function process(): { data: string } {
  return { data: "hello" };
}
```

### Fix 3: Move the type to the same file as the export

```typescript
// If the type is internal, make the function internal too
interface InternalResult { data: string; }

// Don't export the function that uses it, or...
function internalProcess(): InternalResult {
  return { data: "hello" };
}
```

### Fix 4: Use `@internal` or a namespace pattern

```typescript
// If using API extractor tools, mark internal types
/** @internal */
export interface InternalResult { data: string; }
// Exporting is required even if marked internal for TS
```

## Examples

```typescript
// Example: Barrel file re-exporting with inferred types
// internal.ts
interface UserData { name: string; }
export function getUser(): UserData { return { name: "Alice" }; }

// Fix: export UserData
export interface UserData { name: string; }
export function getUser(): UserData { return { name: "Alice" }; }
```
