# TS2368 — Type parameter name cannot be 'X'

TS2368 occurs when a type parameter name conflicts with a reserved keyword or an already-in-use identifier. TypeScript forbids certain names for type parameters because they shadow built-in types or cause confusion.

## Common Causes

```typescript
// Cause 1: Using 'any' as a type parameter
class Container<any> { // TS2368
  value: any;
}

// Cause 2: Using 'never' as a type parameter
type Result<never> = { // TS2368
  value: never;
};

// Cause 3: Using 'unknown' as a type parameter
function process<unknown>(x: unknown): unknown { // TS2368
  return x;
}

// Cause 4: Using 'void' as a type parameter
interface PromiseLike<void> { // TS2368
  then(cb: (value: void) => void): void;
}

// Cause 5: Using 'number' or 'string' as a type parameter
function identity<string>(x: string): string { // TS2368
  return x;
}
```

## Solutions

### Solution 1: Rename the type parameter

```typescript
// Wrong
class Container<any> {} // TS2368

// Correct
class Container<T> {
  value: T;
}
```

### Solution 2: Use descriptive names that don't conflict

```typescript
// Wrong
function process<unknown>(x: unknown): unknown {} // TS2368

// Correct
function process<T>(x: T): T {
  return x;
}
```

### Solution 3: Use conventional type parameter naming

```typescript
// Wrong
interface PromiseLike<void> {} // TS2368

// Correct
interface PromiseLike<T> {
  then(cb: (value: T) => void): void;
}
```

## Examples

```typescript
// Good type parameter names
class Container<T> {}
class Pair<A, B> {}
function identity<T>(x: T): T { return x; }
type Nullable<T> = T | null;
type Mapper<TInput, TOutput> = (input: TInput) => TOutput;

// Common conventions
// T — first type parameter
// U, V — additional type parameters
// K — key type
// V — value type
// E — error type
// R — return type

interface Map<K, V> {
  get(key: K): V | undefined;
  set(key: K, value: V): this;
}
```
