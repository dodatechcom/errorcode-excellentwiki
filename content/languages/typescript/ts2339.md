# TS2339 — Property 'X' does not exist on type 'Y'

TS2339 occurs when you try to access a property on a type that doesn't declare that property. TypeScript enforces property access based on the declared shape of types, so accessing a non-existent property is a compile-time error.

## Common Causes

```typescript
// Cause 1: Accessing a property not in the interface
interface User {
  name: string;
  age: number;
}
const user: User = { name: "Alice", age: 30 };
console.log(user.email); // TS2339: Property 'email' does not exist on type 'User'

// Cause 2: Typo in property name
console.log(user.nme); // TS2339: Did you mean 'name'?

// Cause 3: Using 'any' bypass but the real type is stricter
const response: any = fetch("/api");
console.log(response.data.id); // Compiles with any, fails at runtime

// Cause 4: Accessing array methods on non-array
const num: number = 42;
num.map(x => x); // TS2339: Property 'map' does not exist on type 'number'
```

## Solutions

### Fix 1: Add the property to the type definition

```typescript
// Wrong
interface User {
  name: string;
  age: number;
}

// Correct — add email to the interface
interface User {
  name: string;
  age: number;
  email: string;
}
```

### Fix 2: Use optional properties or index signatures

```typescript
interface User {
  name: string;
  age: number;
  email?: string; // Optional
}

// Or allow any extra properties
interface User {
  name: string;
  age: number;
  [key: string]: unknown;
}
```

### Fix 3: Fix typos in property names

```typescript
// Wrong
console.log(user.nme);

// Correct
console.log(user.name);
```

### Fix 4: Use type guards before accessing properties

```typescript
function processUser(user: User | Admin) {
  if ("email" in user) {
    console.log(user.email); // OK — TypeScript knows it's Admin
  }
}
```

## Examples

```typescript
// Example 1: API response typed too narrowly
interface ApiResponse {
  data: string;
}
const res: ApiResponse = { data: "hello" };
console.log(res.data.length); // TS2339 if data could be undefined

// Example 2: Class without a property
class Animal {
  name: string = "";
}
const cat: Animal = new Animal();
cat.sound = "meow"; // TS2339: Property 'sound' does not exist on type 'Animal'
```
