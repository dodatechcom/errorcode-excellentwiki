# TS2688 — Type 'X' has no matching call signature

TS2688 occurs when you try to call a value as a function, but its type doesn't have a call signature that matches the arguments provided. This differs from TS2349 (not callable at all) — here the type might be callable, but not with your specific arguments.

## Common Causes

```typescript
// Cause 1: Calling with wrong argument types
type Callback = (x: number) => void;
const fn: Callback = (x) => console.log(x);
fn("hello"); // TS2688: string is not assignable to number

// Cause 2: Calling union type that isn't callable in all cases
type MaybeFn = (() => void) | null;
const value: MaybeFn = null;
value(); // TS2688: 'null' has no call signature

// Cause 3: Calling a type that isn't a function
type Config = { host: string; port: number };
const config: Config = { host: "localhost", port: 3000 };
config(); // TS2688: Config has no call signature

// Cause 4: Generic type without call signature
type Container<T> = { value: T };
const container: Container<number> = { value: 42 };
container(); // TS2688
```

## Solutions

### Fix 1: Provide arguments that match the call signature

```typescript
// Wrong
type Callback = (x: number) => void;
const fn: Callback = (x) => console.log(x);
fn("hello"); // TS2688

// Correct
fn(42);
```

### Fix 2: Narrow the type before calling

```typescript
// Wrong
type MaybeFn = (() => void) | null;
const value: MaybeFn = null;
value(); // TS2688

// Correct
if (value) {
  value(); // OK — value is () => void
}
```

### Fix 3: Use a type guard or typeof check

```typescript
type ConfigOrFunction = Config | (() => void);
const value: ConfigOrFunction = getSomething();

if (typeof value === "function") {
  value(); // OK — TypeScript knows it's callable
}
```

### Fix 4: Ensure the type has a call signature

```typescript
// Wrong
type Config = { host: string };
const config: Config = { host: "localhost" };
config(); // TS2688

// Correct — define a callable interface
interface Factory {
  (): Config;
  create(): Config;
}
```

## Examples

```typescript
// Example 1: Array method type mismatch
const nums: number[] = [1, 2, 3];
nums.includes("1"); // TS2688: string not assignable to number

// Example 2: Promise.resolve type
const p: Promise<number> = Promise.resolve(42);
p.then("not a function"); // TS2688
```
