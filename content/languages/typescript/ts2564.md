# TS2564 — Property has no initializer and is not definitely assigned

TS2564 occurs when `strictPropertyInitialization` is enabled and a class property is neither initialized at declaration nor assigned in the constructor. TypeScript ensures every property is definitely assigned before use.

## Common Causes

```typescript
// Cause 1: Property declared without initializer
class User {
  name: string; // TS2564 — not initialized
  age: number;  // TS2564 — not initialized
}

// Cause 2: Property initialized conditionally
class User {
  name: string;
  constructor(useDefault: boolean) {
    if (useDefault) {
      this.name = "default";
    }
    // TS2564 — not assigned in all paths
  }
}

// Cause 3: Property assigned in a method, not constructor
class User {
  name: string; // TS2564
  setName(name: string) {
    this.name = name;
  }
}
```

## Solutions

### Fix 1: Initialize properties at declaration

```typescript
// Wrong
class User {
  name: string;
  age: number;
}

// Correct
class User {
  name: string = "";
  age: number = 0;
}
```

### Fix 2: Assign in constructor

```typescript
// Wrong
class User {
  name: string;
}

// Correct
class User {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
}

// Or use parameter property
class User {
  constructor(public name: string) {}
}
```

### Fix 3: Use definite assignment assertion (`!`)

```typescript
class User {
  name!: string; // Assertion — I promise it will be set
  constructor() {
    this.initialize();
  }
  initialize() {
    this.name = "Alice";
  }
}
```

### Fix 4: Make the property optional

```typescript
class User {
  name?: string; // optional — undefined is allowed
  age?: number;
}
```

## Examples

```typescript
// Example 1: Property initialized via async method
class DataLoader {
  data!: string[]; // definite assignment assertion

  async load() {
    this.data = await fetchData();
  }
}

// Example 2: Using the definite assignment assertion with DI frameworks
class Service {
  private readonly http!: HttpClient; // set by DI
}
```
