# TS18046 — 'X' is of type 'unknown'

TS18046 occurs when you try to use a value typed as `unknown` without first narrowing its type. The `unknown` type is TypeScript's type-safe alternative to `any` — you must verify the type before accessing properties or methods.

## Common Causes

```typescript
// Cause 1: Catch clause variable is unknown by default
try {
  throw new Error("fail");
} catch (e) {
  console.log(e.message); // TS18046: 'e' is of type 'unknown'
}

// Cause 2: Using unknown from a generic
function parse<T>(input: unknown): T {
  return input; // TS18046 if you try to access input properties
}

// Cause 3: Fetching untyped data
const response = await fetch("/api");
const data = await response.json(); // data is 'any', but if typed as 'unknown'...
console.log(data.name); // TS18046

// Cause 4: Value from a Map with unknown value type
const map = new Map<string, unknown>();
const val = map.get("key");
console.log(val.toString()); // TS18046
```

## Solutions

### Fix 1: Use a type guard to narrow the type

```typescript
// Wrong
try {
  throw new Error("fail");
} catch (e) {
  console.log(e.message); // TS18046
}

// Correct
try {
  throw new Error("fail");
} catch (e) {
  if (e instanceof Error) {
    console.log(e.message); // OK — e is Error
  }
}
```

### Fix 2: Use typeof checks for primitives

```typescript
function process(value: unknown) {
  if (typeof value === "string") {
    console.log(value.toUpperCase()); // OK — value is string
  }
}
```

### Fix 3: Use type assertions when you know the type

```typescript
const data = await response.json() as User;
console.log(data.name); // OK
```

### Fix 4: Use a validated parsing library

```typescript
import { z } from "zod";

const UserSchema = z.object({
  name: z.string(),
  age: z.number(),
});

const data = UserSchema.parse(await response.json());
console.log(data.name); // OK — typed as { name: string; age: number }
```

## Examples

```typescript
// Example 1: forEach with unknown
const items: unknown[] = [1, "hello", true];
items.forEach(item => {
  console.log(item.toFixed()); // TS18046
});

// Fix:
items.forEach(item => {
  if (typeof item === "number") {
    console.log(item.toFixed()); // OK
  }
});

// Example 2: JSON.parse returns any, but explicit unknown needs narrowing
const parsed: unknown = JSON.parse('{"x": 10}');
console.log(parsed.x); // TS18046
```
