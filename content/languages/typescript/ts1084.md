# TS1084 — Invalid reference directive syntax

TS1084 occurs when a triple-slash reference directive (`/// <reference ... />`) has invalid syntax, an unrecognized attribute, or is improperly formatted. Triple-slash directives must follow a strict XML-like format.

## Common Causes

```typescript
// Cause 1: Missing or incorrect path attribute
/// <reference /> // TS1084 — no path or other required attribute

// Cause 2: Unrecognized directive type
/// <reference typescript="lib.d.ts" /> // TS1084 — wrong attribute

// Cause 3: Mismatched quotes
/// <reference path='lib.d.ts" /> // TS1084 — mismatched quotes

// Cause 4: Missing closing slash
/// <reference path="lib.d.ts"> // TS1084 — missing />

// Cause 5: Invalid characters inside directive
/// <reference path="lib.d.ts" extra /> // TS1084 — invalid extra content
```

## Solutions

### Fix 1: Use the correct directive format

```typescript
// Wrong
/// <reference />
/// <reference path="lib.d.ts">

// Correct
/// <reference path="lib.d.ts" />
```

### Fix 2: Use recognized directive types

```typescript
// Wrong
/// <reference typescript="lib.d.ts" />

// Correct — valid directives:
/// <reference path="..." />   // Relative file reference
/// <reference types="..." />  // Package types reference
/// <reference lib="..." />    // Built-in lib reference
```

### Fix 3: Ensure proper quoting

```typescript
// Wrong
/// <reference path='lib.d.ts" />
/// <reference path=lib.d.ts />

// Correct
/// <reference path="lib.d.ts" />
/// <reference path='lib.d.ts' />
```

## Examples

```typescript
// Valid triple-slash directives:
/// <reference path="./types/global.d.ts" />
/// <reference types="node" />
/// <reference lib="es2015" />
/// <reference lib="dom" />

// All of the above are valid. Only use these in ambient declaration files (.d.ts).
```
