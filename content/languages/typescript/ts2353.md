# TS2353 — Object literal may only specify known properties

TS2353 occurs when an object literal contains a property that is not part of its target type. TypeScript performs excess property checks on object literals to catch typos and unused properties.

## Common Causes

```typescript
// Cause 1: Typo in property name
interface User {
  name: string;
  age: number;
}
const user: User = { name: "Alice", aje: 30 }; // TS2353 — 'aje' typo

// Cause 2: Extra property not in the target type
interface Config {
  url: string;
}
const config: Config = { url: "...", timeout: 5000 }; // TS2353

// Cause 3: Misspelled optional property
interface Options {
  color?: string;
  width: number;
}
const opts: Options = { colour: "red", width: 100 }; // TS2353 — 'colour' instead of 'color'
```

## Solutions

### Fix 1: Fix typos in property names

```typescript
// Wrong
const user: User = { name: "Alice", aje: 30 };

// Correct
const user: User = { name: "Alice", age: 30 };
```

### Fix 2: Add the missing property to the interface

```typescript
// Wrong
interface Config { url: string; }
const config: Config = { url: "...", timeout: 5000 };

// Correct
interface Config { url: string; timeout?: number; }
const config: Config = { url: "...", timeout: 5000 };
```

### Fix 3: Use type assertion to bypass excess property check

```typescript
// Wrong
const config: Config = { url: "...", timeout: 5000 };

// Bypass with type assertion (use sparingly)
const config = { url: "...", timeout: 5000 } as Config;
```

### Fix 4: Use index signature for flexible properties

```typescript
interface Config {
  url: string;
  [key: string]: unknown;
}
const config: Config = { url: "...", timeout: 5000 }; // OK
```

## Examples

```typescript
// Example 1: Passing object literal directly to function
function setConfig(config: { url: string }) {}
setConfig({ url: "...", timeout: 5000 }); // TS2353

// Fix: assign to variable first
const cfg = { url: "...", timeout: 5000 };
setConfig(cfg); // OK — no excess property check on variable
```
