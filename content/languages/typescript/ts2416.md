# TS2416 — Property 'X' in type 'Y' is not assignable to the same property in base type 'Z'

TS2416 occurs when a derived type (class or interface) overrides a property from a base type with an incompatible type. The overridden property must be assignable to the base property's type.

## Common Causes

```typescript
// Cause 1: Narrowing the type in a derived class
class Animal {
  name: string = "";
}

class Dog extends Animal {
  name: number = 0; // TS2416 — number is not assignable to string
}

// Cause 2: Changing a required property to optional
interface Base {
  value: string;
}

interface Derived extends Base {
  value?: string; // TS2416 — optional is not assignable to required
}

// Cause 3: Changing a property type in a derived interface
interface Shape {
  area(): number;
}

interface Circle extends Shape {
  area(): string; // TS2416 — string not assignable to number
}

// Cause 4: Incompatible function signature in property
interface Callback {
  handler: (x: number) => void;
}

interface StrictCallback extends Callback {
  handler: (x: string) => void; // TS2416
}

// Cause 5: Incompatible array element type
class BaseList {
  items: string[] = [];
}

class NumberList extends BaseList {
  items: number[] = []; // TS2416 — number[] not assignable to string[]
}
```

## Solutions

### Solution 1: Use the same type as the base

```typescript
// Wrong
class Animal {
  name: string = "";
}

class Dog extends Animal {
  name: number = 0; // TS2416
}

// Correct
class Dog extends Animal {
  name: string = "Rex";
}
```

### Solution 2: Use a compatible (wider) type

```typescript
// Wrong
interface Base {
  value: string;
}

interface Derived extends Base {
  value?: string; // TS2416 — optional not assignable to required
}

// Correct
interface Derived extends Base {
  value: string; // Must match
}
```

### Solution 3: Use a compatible function signature

```typescript
// Wrong
interface Callback {
  handler: (x: number) => void;
}

interface StrictCallback extends Callback {
  handler: (x: string) => void; // TS2416
}

// Correct — contravariant parameter
interface StrictCallback extends Callback {
  handler: (x: number | string) => void; // Wider param type is OK
}
```

### Solution 4: Use the override keyword and fix the type

```typescript
// Wrong
class Base {
  value: string = "";
}

class Derived extends Base {
  override value: number = 0; // TS2416
}

// Correct
class Derived extends Base {
  override value: string = "derived";
}
```

## Examples

```typescript
class BaseComponent {
  protected state: Record<string, unknown> = {};
}

class FormComponent extends BaseComponent {
  // Must match the base type
  protected state: {
    values: Record<string, string>;
    errors: Record<string, string>;
  } = { values: {}, errors: {} }; // TS2416 — different type

  // Correct — compatible type
  protected state: Record<string, string> = {};
}
```
