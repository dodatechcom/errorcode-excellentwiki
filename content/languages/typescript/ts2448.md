# TS2448 — Block-scoped variable not available under target

TS2448 occurs when using `let` or `const` in a context where the target ECMAScript version does not support block-scoped variables. This typically happens when the `--target` is set to ES3 or a very old ECMAScript version.

## Common Causes

```typescript
// Cause 1: Using let/const with target ES3
// tsconfig.json: { "target": "ES3" }
let x = 5; // TS2448
const y = 10; // TS2448

// Cause 2: Using let/const in for...of loop with old target
// target: ES3
for (const item of items) {} // TS2448

// Cause 3: Using destructuring with let/const
// target: ES3
let [a, b] = [1, 2]; // TS2448
```

## Solutions

### Fix 1: Update the target to ES2015 or later

```json
// tsconfig.json
{
  "compilerOptions": {
    "target": "ES2015" // or "ES6", "ES2020", "ESNext"
  }
}
```

### Fix 2: Use var instead of let/const

```typescript
// Instead of:
let x = 5;
const y = 10;

// Use:
var x = 5;
// const has no equivalent — use var with convention
var y = 10;
```

### Fix 3: Use a polyfill or downlevel compilation

```json
{
  "compilerOptions": {
    "target": "ES5",
    "downlevelIteration": true
  }
}
```

## Examples

```typescript
// Example 1: Cannot use let/const with ES3/ES5 target
// Error: Block-scoped variable not available under target

// Fix by updating tsconfig.json:
// { "compilerOptions": { "target": "ES2015" } }

// Example 2: For...of requires target ES2015+
for (const key of Object.keys(obj)) {
  console.log(key);
}
```
