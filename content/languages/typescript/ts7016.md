# TS7016 — Could not find a declaration file for module

TS7016 occurs when you import a module that has no type declarations (`.d.ts` files). TypeScript needs type information for all imported modules. If a library doesn't include types, you need to install them from DefinitelyTyped or create your own.

## Common Causes

```typescript
// Cause 1: Library has no bundled types and no @types package
import * as lodash from "lodash"; // TS7016 — no types for lodash

// Cause 2: Missing @types package
import express from "express"; // TS7016 — need @types/express

// Cause 3: Importing a .js file without declaration
// mylib.js
export function helper() {}
// app.ts
import { helper } from "./mylib"; // TS7016

// Cause 4: Path alias without corresponding types
// tsconfig has paths: { "@/*": ["./src/*"] }
import { something } from "@/local-module"; // TS7016
```

## Solutions

### Fix 1: Install @types package from DefinitelyTyped

```bash
npm install --save-dev @types/lodash
npm install --save-dev @types/express
```

### Fix 2: Create a custom declaration file

```typescript
// types/mylib.d.ts
declare module "mylib" {
  export function helper(): void;
  export const version: string;
}
```

### Fix 3: Use a shorthand ambient module declaration

```typescript
// types/ambient.d.ts
declare module "library-without-types";
// This allows any import from this module, typed as any
```

### Fix 4: Enable allowJs for JS files in the project

```json
{
  "compilerOptions": {
    "allowJs": true,
    "checkJs": false
  }
}
```

## Examples

```typescript
// Example 1: Library without types
// Install with: npm install --save-dev @types/some-library

// If no @types package exists, create a declaration:
// types/some-library.d.ts
declare module "some-library" {
  export function doThing(): void;
  export const VERSION: string;
}
```
