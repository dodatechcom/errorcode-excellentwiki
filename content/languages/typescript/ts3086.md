# TS3086 — Cannot find global type

TS3086 occurs when TypeScript cannot find a globally declared type that it expects to exist. This often happens when a required type library (like `lib.d.ts` or `@types/node`) is not available or the `lib` compiler option is misconfigured.

## Common Causes

```typescript
// Cause 1: Missing lib configuration
// tsconfig.json without "lib" setting when needed
const el = document.getElementById("app"); // TS3086 — global 'document' not found

// Cause 2: Missing @types/node for Node.js globals
import * as fs from "fs";
// TS3086 — global 'Buffer' not found

// Cause 3: No lib specified in tsconfig and target is too low
// tsconfig: { "target": "ES3" }
// TS3086 — many ES2015+ globals missing

// Cause 4: skipLibCheck hides the issue elsewhere
```

## Solutions

### Fix 1: Add the appropriate lib to tsconfig

```json
// For browser projects:
{
  "compilerOptions": {
    "lib": ["dom", "es2020"]
  }
}

// For Node.js projects:
{
  "compilerOptions": {
    "lib": ["es2020"],
    "types": ["node"]
  }
}
```

### Fix 2: Install missing type packages

```bash
# For Node.js globals
npm install --save-dev @types/node

# For DOM globals, ensure "dom" is in the lib array
```

### Fix 3: Declare the missing global type

```typescript
// If you need a custom global type
declare global {
  const MY_APP_VERSION: string;
}

// Or in a .d.ts file:
// globals.d.ts
declare const MY_APP_VERSION: string;
```

## Examples

```typescript
// Example 1: Browser globals
// Error: Cannot find global type 'Document'
// Fix: Add "dom" to lib in tsconfig

// Example 2: Node.js globals
// Error: Cannot find global type 'NodeJS.Timeout'
// Fix: npm install @types/node

// Example 3: Custom global variable
// Declare in a .d.ts file:
// types/global.d.ts
interface Window {
  myAppConfig: { apiUrl: string };
}
```
