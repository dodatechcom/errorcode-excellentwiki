# TS2656 — Exports and export assignments not permitted in module augmentations

TS2656 occurs when you try to use `export` or `export =` inside a module augmentation (a `declare module` block that augments an existing module). Module augmentations can only add to existing declarations, not export new names.

## Common Causes

```typescript
// Cause 1: Export inside a module augmentation
declare module "some-library" {
  export function newFunction(): void; // TS2656
}

// Cause 2: Export default inside augmentation
declare module "some-library" {
  export default class Helper {} // TS2656
}

// Cause 3: Export assignment inside augmentation
declare module "some-library" {
  export = SomeClass; // TS2656
}

// Cause 4: Re-export from augmentation
declare module "some-library" {
  export { helper } from "./helper"; // TS2656
}
```

## Solutions

### Fix 1: Add to existing exports instead of creating new ones

```typescript
// Wrong
declare module "some-library" {
  export function newFunction(): void;
}

// Correct — augment an existing interface or add to existing module
declare module "some-library" {
  interface ExistingInterface {
    newMethod(): void;
  }
}
```

### Fix 2: Use a separate declaration file instead of augmentation

```typescript
// Instead of augmenting, create a new module declaration file
// some-library-extensions.d.ts
declare module "some-library-extensions" {
  export function newFunction(): void;
}
```

### Fix 3: Add members to existing exported interfaces

```typescript
// Wrong
declare module "some-library" {
  export class NewClass {} // TS2656
}

// Correct
declare module "some-library" {
  // Augment an existing class
  interface ExistingClass {
    newMethod(): void;
  }
}
```

## Examples

```typescript
// Example: Proper augmentation vs invalid export

// Wrong:
declare module "express" {
  export function myMiddleware(): void; // TS2656
}

// Correct — augment an existing interface:
declare module "express" {
  interface Application {
    myMiddleware(): void;
  }
}
```
