# TS2766 — Cannot delegate iteration to value because it is not an array or does not have Symbol.iterator

TS2766 occurs when you use `yield*` to delegate iteration to a value that doesn't implement the iterable protocol. The value must either be an array or an object with a `[Symbol.iterator]` method.

## Common Causes

```typescript
// Cause 1: yield* on a plain object
function* gen() {
  yield* { a: 1, b: 2 }; // TS2766 — object is not iterable
}

// Cause 2: yield* on a non-iterable value
function* gen() {
  yield* 42; // TS2766 — number is not iterable
}

// Cause 3: yield* on null or undefined
function* gen() {
  yield* null; // TS2766
}

// Cause 4: yield* on a type that could be a string
function* gen(value: string | number[]) {
  yield* value; // TS2766 — string is iterable but TS might not see it
}
```

## Solutions

### Fix 1: Only use yield* on arrays or iterables

```typescript
// Wrong
function* gen() {
  yield* { a: 1, b: 2 };
}

// Correct — use array
function* gen() {
  yield* [1, 2, 3];
}
```

### Fix 2: Convert non-iterable to array

```typescript
function* gen(obj: Record<string, number>) {
  yield* Object.values(obj); // OK — Object.values returns array
  yield* Object.keys(obj);   // OK — Object.keys returns array
  yield* Object.entries(obj); // OK — returns array of tuples
}
```

### Fix 3: Implement Symbol.iterator on custom objects

```typescript
class Range {
  constructor(private start: number, private end: number) {}

  *[Symbol.iterator](): Iterator<number> {
    for (let i = this.start; i <= this.end; i++) {
      yield i;
    }
  }
}

function* gen() {
  yield* new Range(1, 5); // OK — Range is now iterable
}
```

## Examples

```typescript
// Example 1: yield* with Map and Set
function* iterateMap(map: Map<string, number>) {
  yield* map; // OK — Map is iterable
}

function* iterateSet(set: Set<string>) {
  yield* set; // OK — Set is iterable
}

// Example 2: String is iterable but TS may flag it
function* chars(s: string) {
  yield* s; // OK — strings are iterable
}
```
