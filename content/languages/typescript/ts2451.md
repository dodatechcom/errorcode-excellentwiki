# TS2451 — Cannot redeclare block-scoped variable

TS2451 occurs when a block-scoped variable (`let` or `const`) is declared more than once in the same scope. Unlike `var`, which allows redeclaration, `let` and `const` enforce unique declarations within their scope.

## Common Causes

```typescript
// Cause 1: Double declaration with let
let x = 5;
let x = 10; // TS2451 — cannot redeclare block-scoped 'x'

// Cause 2: Mixing let and const with same name
let name = "Alice";
const name = "Bob"; // TS2451

// Cause 3: Declaring the same name in a function scope
function f() {
  let y = 5;
  let y = 10; // TS2451
}

// Cause 4: Global scope conflict with another file's declaration
// file1.ts
let globalVar = "foo";
// file2.ts
let globalVar = "bar"; // TS2451 — if both are global scope
```

## Solutions

### Fix 1: Remove the duplicate declaration

```typescript
// Wrong
let x = 5;
let x = 10;

// Correct
let x = 5;
x = 10; // reassign, don't redeclare
```

### Fix 2: Use different variable names

```typescript
// Wrong
const name = "Alice";
const name = "Bob";

// Correct
const firstName = "Alice";
const lastName = "Bob";
```

### Fix 3: Use a block scope to isolate declarations

```typescript
let x = 5;
{
  let x = 10; // OK — different block scope
  console.log(x); // 10
}
console.log(x); // 5
```

### Fix 4: For global scope conflicts, use modules

```typescript
// Instead of global declarations, use modules
// file1.ts
export const globalVar = "foo";
// file2.ts
import { globalVar } from "./file1";
export const otherVar = "bar";
```

## Examples

```typescript
// Example 1: Accidental redeclaration in switch case
switch (x) {
  case 1:
    let value = "a";
    break;
  case 2:
    let value = "b"; // TS2451 — same block scope
    break;
  // Fix: wrap each case in braces
  case 1: {
    let value = "a";
    break;
  }
  case 2: {
    let value = "b";
    break;
  }
}
```
