# TS7022 — Implicit 'any' type

TS7022 occurs when TypeScript cannot infer the type of a declaration and falls back to `any`, but the `noImplicitAny` compiler option is enabled. This forces you to provide explicit type annotations when types cannot be inferred.

## Common Causes

```typescript
// Cause 1: Variable declaration without type and no initializer
let result; // TS7022 — implicitly any

// Cause 2: Function parameter without type
function process(data) { // TS7022 — 'data' is implicitly any
  return data;
}

// Cause 3: Function return type cannot be inferred
function getData() {
  // TS7022 — return type implicitly any
  if (condition) {
    return fetch("/api");
  }
}

// Cause 4: Array with no type annotation
const items = []; // TS7022
```

## Solutions

### Fix 1: Add explicit type annotation

```typescript
// Wrong
let result;
function process(data) { return data; }

// Correct
let result: string;
function process(data: unknown): unknown { return data; }
```

### Fix 2: Provide an initializer for inference

```typescript
// Wrong
let result;

// Correct
let result = "";
const items: number[] = [];
```

### Fix 3: Use a proper type instead of any

```typescript
// Wrong
function process(data) { return data; }

// Correct
function process<T>(data: T): T { return data; }
// or
function process(data: Record<string, unknown>): Record<string, unknown> {
  return data;
}
```

### Fix 4: Disable noImplicitAny if necessary

```json
{
  "compilerOptions": {
    "noImplicitAny": false
  }
}
```

## Examples

```typescript
// Example 1: Event handler without typed parameter
button.addEventListener("click", function(event) {
  // TS7022 — 'event' is implicitly any
});

// Fix: use correct Event type
button.addEventListener("click", function(event: MouseEvent) {
  console.log(event.clientX);
});
```
