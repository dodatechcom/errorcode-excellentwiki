# TS7032 — Property is declared but its value is never read

TS7032 occurs when a class property is assigned a value but is never read anywhere in the code. This is part of TypeScript's `noUnusedLocals` and `noUnusedParameters` checks applied at the class property level.

## Common Causes

```typescript
// Cause 1: Property assigned but never accessed
class User {
  name: string;
  private internalId: number; // TS7032 — never read

  constructor(name: string, id: number) {
    this.name = name;
    this.internalId = id;
  }

  getName() { return this.name; }
}

// Cause 2: Property only written to in constructor
class Logger {
  private logCount: number; // TS7032 — never read

  constructor() {
    this.logCount = 0;
  }

  log(message: string) {
    // logCount is incremented but never read
    this.logCount++;
    console.log(message);
  }
}

// Cause 3: Property set but only used in the setter
class Config {
  private _url: string; // TS7032 — never read outside

  set url(value: string) {
    this._url = value;
  }
}
```

## Solutions

### Fix 1: Remove the unused property

```typescript
// Wrong
class User {
  private internalId: number;
  constructor(name: string, id: number) {
    this.internalId = id;
  }
}

// Correct
class User {
  constructor(public name: string) {}
}
```

### Fix 2: Use the property somewhere

```typescript
class Logger {
  private logCount = 0;

  log(message: string) {
    this.logCount++;
    console.log(message);
  }

  getLogCount(): number {
    return this.logCount; // now it's read
  }
}
```

### Fix 3: Prefix with underscore to indicate intentional unused

```typescript
class Config {
  private _url: string;

  set url(value: string) {
    this._url = value; // OK if used elsewhere, or remove
  }

  get url(): string {
    return this._url;
  }
}
```

### Fix 4: Remove the property if not needed

```typescript
class Logger {
  private logCount = 0; // TS7032

  log(message: string) {
    console.log(message);
    // Remove logCount entirely if not needed
  }
}

// Fixed:
class Logger {
  log(message: string) {
    console.log(message);
  }
}
```

## Examples

```typescript
// Example 1: Parameter property not used externally
class Service {
  constructor(private apiKey: string) {} // TS7032 if apiKey never read

  getData() {
    // Not using this.apiKey
    return fetch("/api/data");
  }
}

// Fix: don't make it a parameter property
class Service {
  constructor() {}

  getData() {
    return fetch("/api/data");
  }
}
```
