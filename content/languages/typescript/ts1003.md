# TS1003 — Identifier expected

TS1003 is a parse error that occurs when TypeScript encounters a token where it expected an identifier — a name for a variable, function, class, parameter, or import. This typically follows keywords like `var`, `let`, `const`, `function`, `import`, or `from`.

## Common Causes

```typescript
// Cause 1: Missing variable name after declaration keyword
let = 10; // TS1003: Identifier expected

// Cause 2: Missing function name
function () {} // TS1003: Identifier expected

// Cause 3: Missing parameter name
function greet(, name) {} // TS1003: Identifier expected

// Cause 4: Missing identifier in import
import { from "./utils"; // TS1003 after '{'

// Cause 5: Using a reserved word as identifier
let class = "foo"; // TS1003: class is reserved
```

## Solutions

### Fix 1: Provide the missing identifier

```typescript
// Wrong
let = 10;
function () {}
import { from "./utils";

// Correct
let x = 10;
function greet() {}
import { helper } from "./utils";
```

### Fix 2: Use a non-reserved identifier name

```typescript
// Wrong
let class = "myClass";
let const = 42;

// Correct
let className = "myClass";
let constant = 42;
```

### Fix 3: Fix destructuring syntax

```typescript
// Wrong — missing destructuring target
const { } = obj; // TS1003
const [ ] = arr; // TS1003

// Correct
const { a } = obj;
const [ b ] = arr;
```

## Examples

```typescript
// Example 1: Typo in variable declaration
let myVariable = 5;
let myVaria ble = 5; // TS1003 — space in name

// Example 2: Missing catch variable
try { /* ... */ }
catch { /* ... */ } // TS1003 in older TS — need catch variable
catch (e) { /* ... */ } // Correct
```
