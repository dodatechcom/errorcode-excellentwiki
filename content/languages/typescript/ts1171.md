# TS1171 — Comma not allowed here

TS1171 occurs when a comma appears in a position where TypeScript does not allow it. This commonly happens after rest elements, in trailing comma positions in certain contexts, or between specific syntactic elements.

## Common Causes

```typescript
// Cause 1: Comma after rest element in destructuring
const [...rest,] = arr; // TS1171 — trailing comma after rest

// Cause 2: Comma after rest parameter
function f(...args,) {} // TS1171 — comma after rest param

// Cause 3: Trailing comma after rest element in array
const [...first, second] = arr; // TS1171 — ...rest must be last

// Cause 4: Comma in an import assertion or similar new syntax position
// In older TS versions, specific comma positions trigger this
```

## Solutions

### Fix 1: Remove trailing comma after rest element

```typescript
// Wrong
const [...rest,] = arr;
function f(...args,) {}

// Correct
const [...rest] = arr;
function f(...args) {}
```

### Fix 2: Move rest element to the end

```typescript
// Wrong — rest must be last
const [first, ...middle, last] = arr; // TS1171

// Correct
const [first, ...rest] = arr;
const last = arr[arr.length - 1];
```

### Fix 3: Check for misplaced commas in function calls

```typescript
// Wrong
console.log(, "hello"); // TS1171
console.log("hello",); // OK in modern TS

// Correct
console.log("hello");
```

## Examples

```typescript
// Example 1: Object destructuring with rest
const { a, ...rest, } = obj; // TS1171

// Correct
const { a, ...rest } = obj;

// Example 2: Multiple rest elements
const [a, ...b, ...c] = arr; // TS1171 — only one rest allowed

// Correct
const [a, ...b] = arr;
```
