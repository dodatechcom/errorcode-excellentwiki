# TS1056 — Accessors are only available when targeting ECMAScript 5 and higher

TypeScript's `get` and `set` accessors (getters and setters) are an ECMAScript 5 feature. When your `tsconfig.json` targets an older version (such as ES3), the compiler does not allow them.

## Common Causes

```typescript
// Cause 1: Targeting ES3 in tsconfig.json
// tsconfig.json: { "compilerOptions": { "target": "es3" } }
class Person {
  private _name: string = "";
  get name(): string {
    return this._name;
  }
  set name(value: string) {
    this._name = value;
  }
}

// Cause 2: Accessors in a file without a tsconfig targeting ES5+
// If no tsconfig exists, older TS defaults may apply

// Cause 3: Accessors combined with an outdated --target flag via CLI
// tsc --target es3 main.ts
class Counter {
  private _count: number = 0;
  get count(): number {
    return this._count;
  }
}

// Cause 4: Accessors in ambient declarations targeting ES3
declare class Logger {
  get level(): string;
  set level(value: string);
}
```

## Solutions

### Solution 1: Change the compilation target to ES5 or higher

```json
{
  "compilerOptions": {
    "target": "es5"
  }
}
```

### Solution 2: Remove getters and setters

```typescript
// Wrong — uses accessors with ES3 target
class User {
  private _name: string = "";
  get name(): string {
    return this._name;
  }
  set name(value: string) {
    this._name = value;
  }
}

// Correct — use plain methods instead
class User {
  private _name: string = "";
  getName(): string {
    return this._name;
  }
  setName(value: string): void {
    this._name = value;
  }
}
```

### Solution 3: Use a modern target like ES2015+

```json
{
  "compilerOptions": {
    "target": "es2015"
  }
}
```

## Examples

```typescript
// With target: "es5" this works correctly
class Temperature {
  private _celsius: number = 0;

  get fahrenheit(): number {
    return this._celsius * 9 / 5 + 32;
  }

  set celsius(value: number) {
    this._celsius = value;
  }
}

const temp = new Temperature();
temp.celsius = 100;
console.log(temp.fahrenheit); // 212
```
