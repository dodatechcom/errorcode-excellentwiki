# TS1161 — Unterminated string literal

TS1161 occurs when TypeScript reaches the end of a line (or the end of the file) without finding the closing quote of a string literal. This means a string was opened with `"`, `'`, or `` ` `` but never properly closed.

## Common Causes

```typescript
// Cause 1: Missing closing quote
const name = "Alice; // TS1161 — missing closing "

// Cause 2: Multiline string with single/double quotes
const message = "Hello
World"; // TS1161 — newline before closing quote

// Cause 3: Mismatched quotes — opened with ' closed with "
const text = 'hello"; // TS1161

// Cause 4: Escape sequence breaking the closing quote
const path = "C:\Users\name"; // TS1161 — \n and \U treated as escape

// Cause 5: Template literal missing closing backtick
const html = `<div></div>`; // correct
const html = `<div></div>; // TS1161 — missing backtick
```

## Solutions

### Fix 1: Add the missing closing quote

```typescript
// Wrong
const name = "Alice;

// Correct
const name = "Alice";
```

### Fix 2: Use template literals for multi-line strings

```typescript
// Wrong
const message = "Hello
World";

// Correct
const message = `Hello
World`;

// Or use explicit newline
const message = "Hello\nWorld";
```

### Fix 3: Fix escape sequences with raw string or double backslashes

```typescript
// Wrong
const path = "C:\Users\name";

// Correct — escape backslashes
const path = "C:\\Users\\name";

// Or use a raw string with String.raw
const path = String.raw`C:\Users\name`;
```

## Examples

```typescript
// Example 1: Accidentally using backtick as quote
const greeting = `Hello"; // TS1161 — mix of ` and "

// Example 2: Long string broken across lines
const json = '{"name": "Alice", "age": 30
}'; // TS1161 — newline in single-quoted string

// Correct — concatenation
const json = '{"name": "Alice", "age": 30\n}';
```
