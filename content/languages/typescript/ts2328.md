# TS2328 — Type is not comparable to type

TS2328 occurs when a value is compared (using `===`, `!==`, `==`, or `!=`) to a type that is not comparable. This happens when the types have no overlap or when comparing incompatible types like `never`, or when comparing types that structurally cannot be equal.

## Common Causes

```typescript
// Cause 1: Comparing a value to a type with no overlap
declare const x: string;
if (x === 42) {} // TS2328 — string and number have no overlap (strict comparison)

// Cause 2: Comparing to never
declare const x: string;
const y: never = x as never;
if (x === y) {} // TS2328 — never is not comparable

// Cause 3: Comparing to an empty object type
declare const x: string;
const empty: {} = {};
if (x === empty) {} // TS2328 — string is not comparable to {}

// Cause 4: Comparing enum values from different enums
enum Direction { Up, Down }
enum Color { Red, Blue }

const d: Direction = Direction.Up;
const c: Color = Color.Red;
if (d === c) {} // TS2328

// Cause 5: Comparing string literal types with no common values
type A = "hello";
type B = "world";
declare const a: A;
declare const b: B;
if (a === b) {} // TS2328 — no common values
```

## Solutions

### Solution 1: Ensure the comparison is between compatible types

```typescript
// Wrong
declare const x: string;
if (x === 42) {} // TS2328

// Correct
declare const x: string;
if (x === "42") {} // Comparing string to string
```

### Solution 2: Use type narrowing before comparison

```typescript
// Wrong
declare const value: string | number;
if (value === true) {} // TS2328

// Correct
declare const value: string | number;
if (value === 42) {} // Both are number-compatible
```

### Solution 3: Use type assertions when values are actually comparable

```typescript
// Wrong
declare const a: "hello";
declare const b: "world";
if (a === b) {} // TS2328

// Correct — if comparing runtime values
declare const a: string;
declare const b: string;
if (a === b) {} // OK
```

## Examples

```typescript
// Comparing string enums — valid
enum Status {
  Active = "ACTIVE",
  Inactive = "INACTIVE",
}

const s: Status = Status.Active;
if (s === "ACTIVE") {} // OK — string enum compares to string literal

// Comparing numeric enums — valid
enum Direction {
  Up = 0,
  Down = 1,
}

const d: Direction = Direction.Up;
if (d === 0) {} // OK
```
