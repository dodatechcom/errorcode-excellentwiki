# TS2304 — Cannot Find Name

TS2304 occurs when TypeScript encounters an identifier that has not been declared in the current scope. This typically means a variable, function, class, or type is being used without being imported or defined.

## Common Causes

```typescript
// Cause 1: Using a variable before declaring it
console.log(x); // TS2304: Cannot find name 'x'

// Cause 2: Missing import for a type or value
const data: User = {}; // TS2304: Cannot find name 'User'

// Cause 3: Typo in variable or function name
let userName = "Alice";
console.log(useName); // TS2304: Cannot find name 'useName'

// Cause 4: Using a global without the proper lib or type
const result = fetch("/api"); // TS2304: Cannot find name 'fetch' if lib not configured
```

## Solutions

### Fix 1: Declare the variable before using it

```typescript
// Wrong
console.log(x);

// Correct
let x = 10;
console.log(x);
```

### Fix 2: Import missing types and modules

```typescript
// Wrong — User type is not imported
const user: User = { name: "Alice" };

// Correct
import { User } from "./models";
const user: User = { name: "Alice" };
```

### Fix 3: Fix typos in identifiers

```typescript
// Wrong
let userName = "Alice";
console.log(useName); // TS2304

// Correct
console.log(userName);
```

### Fix 4: Add missing lib references in tsconfig

```json
{
  "compilerOptions": {
    "lib": ["ES2020", "DOM"]
  }
}
```

## Examples

```typescript
// Example 1: Forgetting to import
// file: app.ts
const arr = new Array(); // TS2304 if Array is not imported

// Example 2: Using a global type without @types
const el = document.getElementById("app"); // TS2304 if DOM types are missing

// Example 3: Circular dependency causing unresolved names
// a.ts
import { b } from "./b";
export const a = b + 1;

// b.ts
import { a } from "./a";
export const b = a + 1; // May produce TS2304
```
