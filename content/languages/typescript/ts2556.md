# TS2556 — A spread argument must have a tuple type or be passed to a rest parameter

TS2556 occurs when you use the spread operator (`...`) to pass an array as an argument to a function, but the function parameter is not a rest parameter and the spread value is not a tuple type with a known length.

## Common Causes

```typescript
// Cause 1: Spreading an array into non-rest parameters
function sum(a: number, b: number) { return a + b; }
const nums = [1, 2];
sum(...nums); // TS2556 — nums is number[], not a tuple

// Cause 2: Spreading an array into a function with fixed params
function logName(name: string, age: number) { }
const args = ["Alice", 30];
logName(...args); // TS2556

// Cause 3: Using spread on a Set or Map
const set = new Set([1, 2, 3]);
console.log(Math.max(...set)); // TS2556 — Set has no tuple type
```

## Solutions

### Fix 1: Convert to a rest parameter

```typescript
// Wrong
function sum(a: number, b: number) { return a + b; }
const nums = [1, 2];
sum(...nums);

// Correct — use a rest parameter
function sum(...nums: number[]) { return nums.reduce((a, b) => a + b, 0); }
sum(...nums);
```

### Fix 2: Use a tuple type for the spread value

```typescript
// Wrong
const args = ["Alice", 30];
logName(...args);

// Correct — use `as const` to make it a tuple
const args = ["Alice", 30] as const;
logName(...args);

// Or explicitly type as tuple
const args: [string, number] = ["Alice", 30];
logName(...args);
```

### Fix 3: Use Function.prototype.apply

```typescript
// Wrong
const nums = [1, 2];
sum(...nums);

// Correct — use apply
sum.apply(null, nums);
```

## Examples

```typescript
// Example 1: Spreading Math.max arguments
const values = [10, 20, 30];
Math.max(...values); // TS2556

// Fix 1: use Math.max.apply
Math.max.apply(null, values);

// Fix 2: use as const
Math.max(...values as const);
```
