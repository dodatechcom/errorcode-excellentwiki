# TS2312 — 'X' and 'Y' index signatures are incompatible

TS2312 occurs when two types have index signatures with incompatible value types. For one index signature to be assignable to another, the source's value type must be assignable to the target's value type.

## Common Causes

```typescript
// Cause 1: Incompatible index signature value types
interface A {
  [key: string]: string;
}

interface B {
  [key: string]: number;
}

const a: A = {};
const b: B = a; // TS2312 — string is not assignable to number

// Cause 2: Partial compatibility with union types
interface StringMap {
  [key: string]: string;
}

interface MixedMap {
  [key: string]: string | number;
}

const sm: StringMap = {};
const mm: MixedMap = sm; // OK — string is subtype of string | number
// But the reverse fails:
const sm2: StringMap = mm; // TS2312 — string | number is not assignable to string

// Cause 3: Index signatures with incompatible number keys
interface NumMap {
  [key: number]: string;
}

interface StrMap {
  [key: string]: string;
}

const nm: NumMap = {};
const sm3: StrMap = nm; // TS2312 — number index not assignable to string index

// Cause 4: Index signature vs named property conflict
interface WithIndex {
  [key: string]: string;
  name: string;
}

interface WithNumber {
  [key: string]: number;
}
```

## Solutions

### Solution 1: Align the value types

```typescript
// Wrong
interface A {
  [key: string]: string;
}
interface B {
  [key: string]: number;
}

// Correct — use compatible types
interface B {
  [key: string]: string;
}
```

### Solution 2: Use a common base type

```typescript
// Wrong
interface StringMap {
  [key: string]: string;
}
interface NumberMap {
  [key: string]: number;
}

// Correct — use a union type for both
interface StringOrNumberMap {
  [key: string]: string | number;
}
```

### Solution 3: Use explicit property types instead of index signatures

```typescript
// Wrong
interface A {
  [key: string]: string;
}
interface B {
  [key: string]: number;
}

// Correct — define explicit properties
interface A {
  name: string;
}
interface B {
  name: string;
}
```

## Examples

```typescript
// Compatible index signatures
interface Config {
  [key: string]: string | number | boolean;
}

const defaults: Config = {
  host: "localhost",
  port: 3000,
  debug: false,
};

// Extending with a compatible index signature
interface StrictConfig extends Config {
  [key: string]: string | number | boolean;
}
```
