# TS1435 — Unknown keyword or identifier

TS1435 occurs when TypeScript encounters an identifier that it doesn't recognize as a valid keyword or identifier in the current context. This often means a typo in a keyword, using a future/reserved word, or an unrecognized syntax.

## Common Causes

```typescript
// Cause 1: Typo in a keyword
const x = 10;
pritn(x); // TS1435: 'pritn' is not recognized (typo of 'print')

// Cause 2: Using a keyword from another language
def greet():  // TS1435: 'def' is not a TypeScript keyword (Python syntax)

// Cause 3: Unrecognized decorator or modifier
@sealed  // TS1435 if decorator not recognized or not enabled
class MyClass {}

// Cause 4: Misspelled type annotation keyword
let x: numberr = 10; // TS1435: 'numberr' is not a valid type
```

## Solutions

### Fix 1: Correct keyword typos

```typescript
// Wrong
pritn(x);

// Correct
console.log(x);
```

### Fix 2: Use valid TypeScript keywords

```typescript
// Wrong — Python syntax
def greet():

// Correct — TypeScript function
function greet(): void {}
```

### Fix 3: Enable decorator support if needed

```json
// tsconfig.json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

### Fix 4: Fix type annotation typos

```typescript
// Wrong
let x: numberr = 10;

// Correct
let x: number = 10;
```

## Examples

```typescript
// Example 1: Using a non-existent type
let data: Arrray<number> = []; // TS1435: 'Arrray' not recognized

// Fix:
let data: Array<number> = [];

// Example 2: Wrong loop syntax from another language
for i in range(10) {  // TS1435: not valid TypeScript
}

// Fix:
for (let i = 0; i < 10; i++) {}
```
