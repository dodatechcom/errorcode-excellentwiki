# TS2309 — An export assignment cannot be used in a module

TS2309 occurs when you use `export =` (CommonJS-style default export) in a file that TypeScript treats as an ES module. This happens when your tsconfig's `module` setting is `es2015`/`esnext` but you're using CommonJS export syntax.

## Common Causes

```typescript
// Cause 1: Using 'export =' in ES module mode
export = {
  name: "myModule",
  version: "1.0.0",
}; // TS2309 if module is ES2015+

// Cause 2: Mixing 'export =' with named exports
export = function () {};
export const other = 1; // TS2309 — can't mix with export =

// Cause 3: 'export =' in a declaration file designed for CommonJS
// types/my-lib.d.ts
declare function myLib(): void;
export = myLib; // TS2309 if consuming file uses ES modules

// Cause 4: Importing with 'import =' when file uses export default
import myLib = require("my-lib"); // TS2309 if module resolution is ES
```

## Solutions

### Fix 1: Use export default instead of export =

```typescript
// Wrong — CommonJS style
export = {
  name: "myModule",
};

// Correct — ES module style
export default {
  name: "myModule",
};
```

### Fix 2: Use named exports

```typescript
// Wrong
export = {
  name: "myModule",
  version: "1.0.0",
};

// Correct
export const name = "myModule";
export const version = "1.0.0";
```

### Fix 3: Switch to CommonJS module resolution

```json
// tsconfig.json
{
  "compilerOptions": {
    "module": "commonjs"
  }
}
```

### Fix 4: Use import/export syntax compatible with your module setting

```typescript
// For ES modules
import myLib from "my-lib";

// For CommonJS
import myLib = require("my-lib");
```

## Examples

```typescript
// Example 1: Declaration file with export =
// my-lib.d.ts
declare class MyClass {
  constructor();
  doSomething(): void;
}
export = MyClass; // Only works with "module": "commonjs"

// Example 2: Trying to use both export = and export default
export = something;
export default something; // TS2309 — can't have both
```
