# TS2782 — Type is too complex to represent

TS2782 occurs when TypeScript determines that a type is too complex to display in error messages or tooltips. This typically happens with deeply nested generics, large conditional types, or complex mapped types that produce unreadable representations.

## Common Causes

```typescript
// Cause 1: Deeply nested generic type
type DeepWrapper<T> = { data: T };
type Wrapped = DeepWrapper<DeepWrapper<DeepWrapper<...>>>;

// Cause 2: Complex mapped type with many keys
type Everything<T> = {
  [K in keyof T]-?: T[K] extends object
    ? { [P in keyof T[K]]: T[K][P] }
    : T[K];
};

// Cause 3: Conditional type with many branches
type Complex<T> = T extends string
  ? A : T extends number
    ? B : T extends boolean
      ? C : // ... many more branches

// Cause 4: Large recursive template literal types
type DeepPath<T, K extends string> = K extends `${infer P}.${infer R}`
  ? P extends keyof T ? DeepPath<T[P], R> : never
  : K extends keyof T ? T[K] : never;
```

## Solutions

### Fix 1: Break complex types into smaller pieces

```typescript
// Instead of one huge conditional type, use multiple smaller ones
type HandleString<T> = T extends string ? StringResult : never;
type HandleNumber<T> = T extends number ? NumberResult : never;
type HandleBoolean<T> = T extends boolean ? BoolResult : never;

type Complex<T> = HandleString<T> | HandleNumber<T> | HandleBoolean<T>;
```

### Fix 2: Use interfaces instead of complex mapped types

```typescript
// Instead of a mapped type that's hard to represent:
type ComplexMapped<T> = {
  [K in keyof T]: T[K] extends Function ? never : T[K];
};

// Use an interface with a simpler structure
interface Simplified {
  [key: string]: Exclude<unknown, Function>;
}
```

### Fix 3: Limit type recursion depth

```typescript
// Instead of unlimited recursion, limit depth explicitly
type DeepPartialOne<T> = T extends object
  ? { [K in keyof T]?: T[K] extends object ? SimplePartial<T[K]> : T[K] }
  : T;

type SimplePartial<T> = { [K in keyof T]?: T[K] };
```

## Examples

```typescript
// Example: API response type that's too complex
// Instead of deeply nested inferred types:
const result = await api.get<ComplexResponse>();

// Fix: explicitly type the parts
interface SimplifiedResponse {
  user: { id: number; name: string };
  meta: { page: number; total: number };
}
const result: SimplifiedResponse = await api.get();
```
