# TS2307 — Cannot Find Module

TS2307 occurs when TypeScript cannot resolve a module imported via `import` or `require()`. This can happen when the module path is incorrect, the module doesn't exist, or type declarations are missing.

## Common Causes

```typescript
// Cause 1: Typo in module path
import { helper } from "./helpr"; // TS2307: Cannot find module './helpr'

// Cause 2: Missing type declarations for a package
import _ from "lodash"; // TS2307 if @types/lodash is not installed

// Cause 3: Module not installed via npm
import express from "express"; // TS2307 if express is not in node_modules

// Cause 4: Incorrect relative path
import { utils } from "../utils"; // TS2307 if the path doesn't resolve
```

## Solutions

### Fix 1: Correct the import path

```typescript
// Wrong
import { helper } from "./helpr";

// Correct
import { helper } from "./helper";
```

### Fix 2: Install missing type declarations

```bash
npm install --save-dev @types/lodash
npm install --save-dev @types/express
```

### Fix 3: Install the missing package

```bash
npm install express
```

### Fix 4: Configure module resolution in tsconfig

```json
{
  "compilerOptions": {
    "moduleResolution": "node",
    "baseUrl": ".",
    "paths": {
      "@app/*": ["src/*"]
    }
  }
}
```

### Fix 5: Create a type declaration file for untyped modules

```typescript
// types/custom-module.d.ts
declare module "untyped-module" {
  const value: any;
  export default value;
}
```

## Examples

```typescript
// Example 1: Missing @types package
import jwt from "jsonwebtoken";
// TS2307: Cannot find module 'jsonwebtoken' or its corresponding type declarations
// Fix: npm install --save-dev @types/jsonwebtoken

// Example 2: Incorrect relative path
// src/utils.ts exists but you import from wrong path
import { formatDate } from "./util"; // TS2307 — file is "utils", not "util"
```
