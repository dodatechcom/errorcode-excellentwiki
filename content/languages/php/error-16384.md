# [Solution] PHP Error 16384 — E_USER_DEPRECATED Fix

PHP error code 16384 corresponds to `E_USER_DEPRECATED` (16384). This is a deprecation notice explicitly triggered by user code or a library via `trigger_error()`. The script continues executing, but the notice indicates that the function or pattern you are using is deprecated and should be replaced.

## Common Causes

- Calling deprecated methods in libraries or frameworks
- Using outdated API patterns flagged by maintainers
- Passing deprecated configuration options to functions
- Using a function marked for removal in a future version

## How to Fix

### 1. Replace Deprecated API Calls

```php
// WRONG — using deprecated method from a library
<?php
$cache->getRaw('key'); // triggers E_USER_DEPRECATED
?>

// CORRECT — use the current method
<?php
$cache->get('key');
?>
```

### 2. Suppress During Migration

```php
<?php
// Temporarily suppress while migrating to the new API
$value = @oldLibraryFunction($param); // suppresses E_USER_DEPRECATED
?>
```

### 3. Add a Custom Handler for Monitoring

```php
<?php
function user_deprecation_handler($errno, $errstr, $errfile, $errline) {
    if ($errno === E_USER_DEPRECATED) {
        error_log("USER_DEPRECATED: $errstr in $errfile on line $errline");
        return true;
    }
    return false;
}

set_error_handler('user_deprecation_handler');
error_reporting(E_ALL | E_USER_DEPRECATED);
?>
```

### 4. Mark Your Own Functions as Deprecated

```php
<?php
class APIv1 {
    /** @deprecated Use APIv2::fetch() instead */
    public function getData() {
        trigger_error(__METHOD__ . '() is deprecated, use APIv2::fetch()', E_USER_DEPRECATED);
        return $this->fetch();
    }
}

class APIv2 {
    public function fetch() {
        return ['data' => 'value'];
    }
}
?>
```

## Examples

```php
<?php
// Error 16384: deprecated method call
$lib->oldMethod(); // library triggers E_USER_DEPRECATED

// Error 16384: deprecated config option
config('legacy.option'); // framework triggers E_USER_DEPRECATED
?>
```
