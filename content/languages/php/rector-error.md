# Rector Code Migration Error

Rector errors occur during automated code refactoring when Rector cannot transform a file due to syntax issues, rule configuration problems, or unsupported code patterns. These errors typically happen during PHP version migrations or when applying automated code standards.

## Common Causes

```php
<?php
// Cause 1: Rector rule configuration issue
// rector.php has invalid rule configuration

// Cause 2: Syntax error in target file prevents transformation
function broken( { // Syntax error — Rector cannot parse

// Cause 3: Rule conflicts with existing code patterns
// Rector tries to apply a rule that conflicts with custom logic

// Cause 4: Missing required dependency
// Rector rule needs a package not installed

// Cause 5: Transformation creates invalid code
// Automated fix produces syntax errors
```

## How to Fix

### Fix 1: Always run Rector in dry-run mode first

```bash
# Dry-run shows changes without modifying files
vendor/bin/rector process src/ --dry-run

# For specific PHP version migration
vendor/bin/rector process src/ --set=php81 --dry-run

# Save dry-run output to diff file
vendor/bin/rector process src/ --dry-run --output-format=json > rector-report.json
```

### Fix 2: Check and fix rule configuration

```php
<?php
// rector.php
declare(strict_types=1);

use Rector\Config\RectorConfig;
use Rector\Set\ValueObject\LevelSetList;

return RectorConfig::configure()
    ->withPaths([
        __DIR__ . '/src',
    ])
    ->withSets([
        LevelSetList::UP_TO_PHP_81,
    ])
    ->withSkip([
        // Skip files that cause issues
        __DIR__ . '/src/Legacy/AutoGenerated.php',
        __DIR__ . '/src/Tests/',
    ]);
```

### Fix 3: Fix syntax errors before running Rector

```bash
# Check syntax of all PHP files first
find src/ -name "*.php" -exec php -l {} \;

# Or with parallel processing
find src/ -name "*.php" | xargs -P 4 -I {} php -l {}

# Fix any syntax errors before running Rector
```

### Fix 4: Exclude problematic files and directories

```php
<?php
// rector.php — skip problematic paths
return RectorConfig::configure()
    ->withSkip([
        // Files with complex patterns Rector can't handle
        __DIR__ . '/src/Generated/',
        __DIR__ . '/src/Legacy/',

        // Specific patterns
        __DIR__ . '/src/*Test.php',
        __DIR__ . '/src/*Factory.php',

        // Specific rules for specific paths
        \Rector\Privatization\Rector\Class_\PrivatizeFinalClassPropertyRector::class => [
            __DIR__ . '/src/Entity/',
        ],
    ]);
```

### Fix 5: Run specific rules individually

```bash
# Run a single rule to isolate issues
vendor/bin/rector process src/ \
    --rule=Rector\CodeQuality\Rector\Class_\InlineConstructorDefaultToPropertyRector \
    --dry-run

# Run specific set
vendor/bin/rector process src/ --set=code-quality --dry-run

# Run on a single file for debugging
vendor/bin/rector process src/Controller/UserController.php --dry-run
```

### Fix 6: Handle failed transformations

```php
<?php
// If Rector produces invalid code, revert and investigate

// 1. Use git to track changes
git stash          # before rector
vendor/bin/rector process src/
git diff           # review changes

// 2. If changes are bad, revert
git checkout src/

// 3. Configure Rector to skip the problematic rule
return RectorConfig::configure()
    ->withSkip([
        \Some\Problematic\Rector\RuleRector::class,
    ]);
```

## Examples

```php
<?php
// Example Rector migration workflow

// Step 1: Create rector config
// rector.php
return RectorConfig::configure()
    ->withPaths([__DIR__ . '/src'])
    ->withSets([LevelSetList::UP_TO_PHP_82]);

// Step 2: Dry-run to preview
vendor/bin/rector process src/ --dry-run

// Step 3: Review the output
// Applied rules:
// - Rector\Php81\Rector\Func_\MbStrtolowerRector
// - Rector\Php82\Rector\String_\DeprecatedMbStrFunctionsRector

// Step 4: Apply changes
vendor/bin/rector process src/

// Step 5: Run tests to verify
vendor/bin/phpunit

// Step 6: Commit the migration
git add src/
git commit -m "Migrate to PHP 8.2 with Rector"
```
