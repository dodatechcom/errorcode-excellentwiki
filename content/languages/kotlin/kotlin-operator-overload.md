## What This Error Means

Operator overloading ambiguity occurs when multiple operator functions are defined with conflicting signatures, or when the compiler cannot resolve which overload to use due to implicit conversions.

## Common Causes

- Defining both `plus` and `plusAssign` without clear semantics
- `compareTo` returning non-Int type
- Ambiguous overload between extension and member function
- Operator function with wrong return type

```kotlin
// Ambiguous: plus vs plusAssign
class Container {
    operator fun plus(other: Container): Container { ... }
    operator fun plusAssign(other: Container) { ... }
}
// val a = b + c   // plus
// a += c          // plusAssign — different semantics
```

## How to Fix

**1. Define operators with correct signatures**

```kotlin
data class Money(val amount: Double) {
    operator fun plus(other: Money) = Money(amount + other.amount)
    operator fun compareTo(other: Money) = amount.compareTo(other.amount)
    operator fun minus(other: Money) = Money(amount - other.amount)
}
```

**2. Follow the compareTo/equals contract**

```kotlin
data class Score(val value: Int) : Comparable<Score> {
    override fun compareTo(other: Score) = value.compareTo(other.value)
    // equals and hashCode auto-generated by data class
    // Consistent: a == b implies a.compareTo(b) == 0
}
```

**3. Avoid overloading conflicting operators**

```kotlin
// WRONG: Confusing semantics
class Weird {
    operator fun plus(other: Weird) = Weird()      // Returns new
    operator fun plusAssign(other: Weird) { }       // Mutates
}

// CORRECT: Choose one pattern
class Weird {
    operator fun plus(other: Weird) = Weird()       // Immutable pattern
}
```

**4. Use invoke operator for function-like objects**

```kotlin
class Validator(private val pattern: Regex) {
    operator fun invoke(input: String): Boolean = pattern.matches(input)
}

val emailValidator = Validator(Regex(".*@.*"))
emailValidator("test@example.com")  // true
```

        major != other.major -> major - other.minor
        else -> minor - other.minor
    }

// Example 2: Unary operators
data class Vector(val x: Double, val y: Double) {
    operator fun unaryMinus() = Vector(-x, -y)
    operator fun not() = Vector(y, x)  // Perpendicular
}

// Example 3: Get/set operators for indexed access
class Matrix(private val data: Array<DoubleArray>) {
    operator fun get(row: Int, col: Int) = data[row][col]
    operator fun set(row: Int, col: Int, value: Double) { data[row][col] = value }
}
