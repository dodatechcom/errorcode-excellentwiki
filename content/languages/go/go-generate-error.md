# Go generate Error

Fix Go generate errors. Handle code generation, tool installation, and output file management.

## Why It Happens

- go generate does not run because the directive is in the wrong format
- Generated files are not committed causing build failures
- go generate runs tools that are not installed
- go generate produces different output on different machines

## Common Error Messages

```
go generate: no such tool
```
```
go generate: command not found
```
```
go generate: pattern has no matches
```
```
go generate: error generating code
```

## How to Fix It

### Solution 1: Run go generate

```go
//go:generate stringer -type=Color
//go:generate mockgen -source=repo.go -destination=mocks/mock_repo.go
//go:generate go-bindata -pkg=assets -o=assets/assets.go static/...
//go:generate protoc --go_out=. proto/service.proto
// Run: go generate ./...
```

### Solution 2: Install required tools

```go
// Install all tools
go install golang.org/x/tools/cmd/stringer@latest
go install github.com/robfig/mockgen@latest
// Or use tools.go
//go:build tools
import _ "golang.org/x/tools/cmd/stringer"
```

### Solution 3: Commit generated files

```go
// Generated files should be committed
to your repository to ensure reproducible builds
// Add a comment at the top: // Code generated by ... DO NOT EDIT.
```

### Solution 4: Fix environment differences

```go
// Use GOFLAGS or go install to ensure consistent tool versions
// Or use a tools.go file to pin versions
go install golang.org/x/tools/cmd/stringer@v0.14.0
```

## Common Scenarios

- go generate produces no output because the command is not found
- Generated code differs between developers because of tool version differences
- go generate runs successfully but the generated file is empty

## Prevent It

- Use tools.go to pin code generation tool versions
- Always commit generated files to version control
- Run go generate after pulling changes
