# [Solution] SQLite nested JSON too deep

SQLite JSON1 extension produces **nested JSON too deep** when a json string has nesting depth that exceeds sqlite's internal limit. The JSON1 extension provides powerful JSON manipulation functions for SQLite.

## Common Causes

- The JSON is deeply nested (e.g., generated by a recursive process).
- The nesting exceeds SQLite's internal limit (default 1000).
- A malformed JSON string creates artificial nesting.

## How to Fix

### Flatten deeply nested JSON structures

```sql
-- Use json_tree() to extract values at specific depths
SELECT * FROM json_tree('{"a":{"b":{"c":1}}}', '$.a.b');
```

### Reduce nesting in the application layer

```python
# Flatten nested dicts before inserting into SQLite
```

### Use multiple rows instead of deep nesting

```sql
-- Instead of {"a":{"b":{"c":1}}}
-- Use: a.b.c = 1 (flat key-value pairs)
```

## Examples

```sql
SELECT json_extract('{"a":{"b":{"c":{"d":{...}}}}}', '$.a.b.c.d');
-- Error: JSON too deeply nested
```
