# MySQL Foreign Key Does Not Exist Error

DROP FOREIGN KEY or ALTER TABLE fails because the specified foreign key constraint name does not exist in the table's metadata.

## Common Causes

- Foreign key constraint name was auto-generated by MySQL
- The constraint was already dropped in a previous migration
- Typo in the constraint name
- Constraint was on a different table than expected
- Table was recreated without the foreign key

## How to Fix

### Find the Actual Constraint Name

```sql
-- List all foreign key constraints on a table
SELECT
  CONSTRAINT_NAME,
  COLUMN_NAME,
  REFERENCED_TABLE_NAME,
  REFERENCED_COLUMN_NAME
FROM INFORMATION_SCHEMA.KEY_COLUMN_USAGE
WHERE TABLE_NAME = 'order_items'
  AND REFERENCED_TABLE_NAME IS NOT NULL;
```

### Drop by Actual Name

```sql
-- First find the constraint name
SELECT CONSTRAINT_NAME
FROM INFORMATION_SCHEMA.TABLE_CONSTRAINTS
WHERE TABLE_NAME = 'order_items'
  AND CONSTRAINT_TYPE = 'FOREIGN KEY';

-- Then drop it
ALTER TABLE order_items DROP FOREIGN KEY fk_order_items_order;
```

### Drop All Foreign Keys Dynamically

```sql
-- Generate DROP statements for all foreign keys
SELECT CONCAT('ALTER TABLE ', TABLE_NAME, ' DROP FOREIGN KEY ', CONSTRAINT_NAME, ';')
FROM INFORMATION_SCHEMA.TABLE_CONSTRAINTS
WHERE TABLE_NAME = 'order_items'
  AND CONSTRAINT_TYPE = 'FOREIGN KEY';
```

### Add Foreign Key with Explicit Name

```sql
-- Always name your foreign keys for easier management
ALTER TABLE order_items
  ADD CONSTRAINT fk_order_items_order_id
  FOREIGN KEY (order_id) REFERENCES orders(id)
  ON DELETE CASCADE;
```

### Check for Similar Names

```sql
-- Check for constraint names that contain your search term
SELECT CONSTRAINT_NAME
FROM INFORMATION_SCHEMA.TABLE_CONSTRAINTS
WHERE TABLE_NAME = 'order_items'
  AND CONSTRAINT_NAME LIKE '%order%';
```

## Examples

```
ERROR 1091 (42000): Can't DROP 'fk_items_order'; check that
  column/key exists

ERROR 1826 (HY000): Failed to add the foreign key constraint:
  Duplicate key name 'fk_items_order'
```
