# ORA-01652: Unable to Extend Temp Segment

ORA-01652 occurs when Oracle cannot allocate space for a temporary segment in the temporary tablespace. This typically happens during large sorting, hashing, or index creation operations.

## Common Causes

- The temporary tablespace is too small for the operation
- A query requires a large sort area that exceeds temp space
- The temporary tablespace datafile cannot auto-extend
- A runaway query is consuming all available temp space

## How to Fix

### Check Temporary Tablespace Usage

```sql
SELECT
    tablespace_name,
    SUM(bytes_used) / 1024 / 1024 AS used_mb,
    SUM(bytes_free) / 1024 / 1024 AS free_mb
FROM v$temp_space_header
GROUP BY tablespace_name;
```

### Add Datafile to Temporary Tablespace

```sql
ALTER TABLESPACE temp ADD TEMPFILE '/u01/oradata/temp02.dbf'
    SIZE 10G AUTOEXTEND ON NEXT 1G MAXSIZE 30G;
```

### Resize Existing Tempfile

```sql
ALTER DATABASE TEMPFILE '/u01/oradata/temp01.dbf' RESIZE 20G;
```

### Kill Runaway Sessions

```sql
SELECT sid, serial#, sql_id, temp_space_allocated / 1024 / 1024 AS mb
FROM v$rsql
WHERE temp_space_allocated > 100 * 1024 * 1024;

ALTER SYSTEM KILL SESSION 'sid,serial' IMMEDIATE;
```

### Optimize the Query

```sql
-- Add indexes to reduce sort operations
CREATE INDEX idx_large ON large_table(sort_column);

-- Use hints to avoid sorting
SELECT /*+ NO_SORT */ * FROM large_table ORDER BY col;
```

## Examples

```sql
-- Large sort operation
SELECT * FROM million_row_table ORDER BY created_date;
-- ORA-01652: unable to extend temp segment by 128 in tablespace TEMP
-- Fix: ALTER TABLESPACE temp ADD TEMPFILE ...

-- Index creation
CREATE INDEX idx_big ON huge_table(col);
-- ORA-01652: unable to extend temp segment by 256 in tablespace TEMP
-- Fix: increase temp tablespace size
```
