The `ORA-01017: invalid username/password; logon denied` error means the Oracle database rejected the username or password supplied during login. The credentials do not match any valid database account, or the account is locked.

## Common Causes

- Incorrect username or password (case-sensitive in Oracle 11g+)
- The database account is locked due to too many failed attempts
- The password has expired per the profile's password policy
- Connecting to the wrong database instance or service name

## How to Fix

Reset the password for the affected user:

```sql
ALTER USER app_user IDENTIFIED BY new_password;
```

Check if the account is locked and unlock it:

```sql
SELECT username, account_status FROM dba_users WHERE username = 'APP_USER';

ALTER USER app_user ACCOUNT UNLOCK;
```

Verify the service name in your connection string:

```bash
lsnrctl status
```

## Examples

```sql
-- Connecting with wrong password
sqlplus app_user/wrongpass@ORCL
-- ERROR: ORA-01017: invalid username/password; logon denied

-- Account locked after failed attempts
sqlplus app_user/correctpass@ORCL
-- ERROR: ORA-01017: invalid username/password; logon denied
```

Check account status to distinguish between a bad password and a locked account:

```sql
SELECT username, account_status, profile
FROM dba_users
WHERE username = 'APP_USER';
```
