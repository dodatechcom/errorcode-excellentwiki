# ORA-04031: Unable to Allocate Shared Memory

ORA-04031 occurs when Oracle cannot allocate a contiguous block of shared memory from the shared pool. The SGA is fragmented or too small for the current workload.

## Common Causes

- The shared pool is too small for the application's cursor usage
- Excessive hard parsing due to lack of bind variables
- Shared pool fragmentation from many small allocations
- Large PL/SQL packages or cursor caching consuming shared memory

## How to Fix

### Check Shared Pool Size

```sql
SELECT name, value / 1024 / 1024 AS mb
FROM v$sga
WHERE name IN ('Shared Pool', 'Large Pool', 'Java Pool');
```

### Increase Shared Pool Size

```sql
ALTER SYSTEM SET shared_pool_size = 4G SCOPE=BOTH;
```

### Flush Shared Pool

```sql
-- Temporary fix: flush shared pool
ALTER SYSTEM FLUSH SHARED_POOL;
```

### Use Bind Variables to Reduce Parsing

```sql
-- WRONG: hard parse every time
EXECUTE IMMEDIATE 'SELECT * FROM users WHERE id = ' || 123;

-- CORRECT: bind variables reduce shared pool usage
EXECUTE IMMEDIATE 'SELECT * FROM users WHERE id = :1' USING 123;
```

### Enable Automatic Memory Management

```sql
ALTER SYSTEM SET sga_target = 16G SCOPE=BOTH;
ALTER SYSTEM SET sga_max_size = 16G SCOPE=SPFILE;
```

## Examples

```sql
-- Cursor exhaustion
BEGIN
    FOR i IN 1..100000 LOOP
        EXECUTE IMMEDIATE 'SELECT count(*) FROM tab' || i;
    END LOOP;
END;
-- ORA-04031: unable to allocate 4192 bytes of shared memory

-- Fix: use bind variables and increase shared_pool_size
```
