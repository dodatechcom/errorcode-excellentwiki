# CircleCI Test Result XML Parse Error

Test result XML parse errors occur when the JUnit XML output files generated by the test framework contain formatting issues that prevent CircleCI from processing them.

## Common Causes

- XML file is empty or contains only whitespace
- XML declaration is missing or malformed
- Test framework outputs non-standard XML elements
- Multiple XML files are concatenated without proper wrapping
- File encoding is not UTF-8

## How to Fix

### Solution 1: Verify XML output format

```yaml
steps:
  - run:
      name: Run tests
      command: |
        mkdir -p test-results
        pytest --junitxml=test-results/results.xml
  - run:
      name: Validate XML
      command: xmllint --noout test-results/results.xml
  - store_test_results:
      path: test-results
```

### Solution 2: Use standard JUnit reporters

```yaml
steps:
  - run:
      name: Run tests
      command: |
        npx jest --ci --reporters=default --reporters=jest-junit
      environment:
        JEST_JUNIT_OUTPUT_DIR: test-results
```

### Solution 3: Ensure directory contains valid XML

```yaml
steps:
  - store_test_results:
      path: test-results  # Directory must contain .xml files
```

## Examples

```
Error: Failed to parse test results: invalid XML
WARNING: No valid JUnit XML files found
```

## Prevent It

- Use standard test reporters that output JUnit XML
- Validate XML output before storing
- Ensure test results directory is not empty
