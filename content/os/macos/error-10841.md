# macOS Error -10841 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10841 (kLSDataUnavailable) indicates that Launch Services is unable to access required data during an application launch or file opening operation. This variant of the -1083x family typically occurs when there is a mismatch between the Launch Services database and the actual state of applications on disk. It often appears when applications are updated, moved, or removed while Launch Services is active.

## Common Causes

- Applications were updated while Launch Services was referencing old metadata
- The Launch Services database has references to moved or renamed applications
- Multiple users modified Launch Services data concurrently
- The application bundle identifier changed during an update
- System Integrity Protection blocked a modification to system-level UTI data
- The Launch Services cache is out of sync with the filesystem

## How to Fix Error -10841

### 1. Force a Complete Rebuild

Rebuild the Launch Services database from scratch:

```bash
# Kill existing processes
pkill -f lsregister

# Full rebuild with all domains
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder
```

### 2. Verify Application Bundles

Check for corrupted or moved applications:

```bash
# List all applications
ls /Applications/

# Check for missing Info.plist files
for app in /Applications/*.app; do
    if [ ! -f "$app/Contents/Info.plist" ]; then
        echo "Missing Info.plist: $app"
    fi
done

# Verify code signatures
for app in /Applications/*.app; do
    codesign --verify --deep --strict "$app" 2>&1 || echo "Invalid: $app"
done
```

### 3. Clear the Application State

Reset application state data:

```bash
# Clear all saved application states
rm -rf ~/Library/Saved\ Application\ State/*

# Clear the Launch Services user-level cache
rm -rf ~/Library/Caches/com.apple.LaunchServices*
rm -rf ~/Library/Preferences/com.apple.LaunchServices.plist

# Restart Finder
killall Finder
```

### 4. Check for Bundle Identifier Conflicts

Bundle identifier mismatches can cause data unavailability:

```bash
# List bundle identifiers of installed apps
for app in /Applications/*.app; do
    bid=$(defaults read "$app/Contents/Info.plist" CFBundleIdentifier 2>/dev/null)
    echo "$bid: $app"
done | sort

# Look for duplicate bundle identifiers
```

### 5. Reset the Dock

The Dock maintains application references:

```bash
# Restart the Dock
killall Dock

# Reset Dock preferences if needed
defaults delete com.apple.dock
killall Dock
```

### 6. Check System Logs for Details

Identify the specific data that is unavailable:

```bash
# Check recent Launch Services logs
log show --predicate 'subsystem == "com.appleLaunchServices"' --last 30m

# Look for specific error details
log show --predicate 'eventMessage contains "10841"' --last 1h
```

## Examples

This error commonly occurs when:

- After updating an application that was set as a default handler
- When the Mac has multiple user accounts modifying app preferences
- After migrating applications from one Mac to another
- When applications are installed to non-standard locations
- After a partial Time Machine restore that updated some apps but not others
