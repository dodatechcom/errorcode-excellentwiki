# macOS Error -10845 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10845 (kLSDataUnavailable) indicates that Launch Services experienced a data retrieval failure while attempting to resolve an application or file handler. This error in the -1083x family typically occurs when the Launch Services database is in a transitional state, such as during a rebuild, or when the system is under heavy load and cannot allocate resources to complete the data lookup.

## Common Causes

- The Launch Services database is being rebuilt while a request is made
- System is under heavy CPU or memory pressure
- The Launch Services daemon has crashed or been killed
- Temporary file locks prevent data access
- The system volume is remounting or repairing
- Network-related delays in accessing shared Launch Services data

## How to Fix Error -10845

### 1. Allow Launch Services to Complete Rebuild

If a rebuild is in progress, wait and retry:

```bash
# Check if lsregister is running
ps aux | grep lsregister

# Wait for it to complete, then restart Finder
sleep 10
killall Finder
```

### 2. Reduce System Load

Ensure the system has sufficient resources:

```bash
# Check CPU usage
top -l 1 -s 0 | head -5

# Check memory pressure
memory_pressure

# Check for resource-heavy processes
ps aux --sort=-%cpu | head -10
ps aux --sort=-%mem | head -10
```

### 3. Force Clean Rebuild

If the database is stuck in a transitional state:

```bash
# Kill all Launch Services related processes
pkill -f lsregister
pkill -f cfprefsd

# Clear all Launch Services data
rm -rf ~/Library/Caches/com.apple.LaunchServices*
rm -rf ~/Library/Preferences/com.apple.LaunchServices*
sudo rm -rf /Library/Caches/com.apple.LaunchServices*

# Force rebuild
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder
```

### 4. Check for Stuck File Locks

Release any file locks that may be blocking data access:

```bash
# Check for processes holding locks on Launch Services files
lsof +D /Library/LaunchServices/ 2>/dev/null

# Check for locked files in user cache
lsof ~/Library/Caches/com.apple.LaunchServices* 2>/dev/null

# Force release locks by restarting the preferences daemon
sudo launchctl kickstart -k system/com.apple.cfprefsd
```

### 5. Restart the System

A clean restart resolves most transient issues:

```bash
# Save work and restart
sudo reboot
```

### 6. Check System Logs for Root Cause

Investigate what triggered the data unavailability:

```bash
# Check for Launch Services errors
log show --predicate 'subsystem == "com.appleLaunchServices"' --last 1h --level error

# Check for system resource issues
log show --predicate 'eventMessage contains "jetsam"' --last 1h

# Check for memory pressure events
log show --predicate 'eventMessage contains "memory pressure"' --last 1h
```

## Examples

This error commonly occurs when:

- Opening many files at once in a script or automation
- After force-quitting an application during Launch Services operation
- When the system is performing Time Machine backups
- During heavy compilation or build processes
- When multiple network volumes are being mounted simultaneously
