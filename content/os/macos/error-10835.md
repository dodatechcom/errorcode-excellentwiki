# macOS Error -10835 (kLSUnrecognizedCert) — Unrecognized Certificate

Error -10835 (kLSUnrecognizedCert / errLSUnrecognizedCert) indicates that Launch Services has encountered a code signing certificate that it cannot verify or does not recognize. This error occurs when an application's signing certificate is issued by an authority that macOS does not trust, is self-signed without proper trust chain, or has been revoked by Apple. The system blocks the application from running until the trust issue is resolved.

## Common Causes

- The application is signed with a self-signed certificate
- The signing certificate was issued by an untrusted Certificate Authority
- The application was signed with an ad-hoc signature (no certificate)
- Apple has revoked the developer's certificate
- The certificate chain is incomplete or broken
- The certificate has expired
- macOS does not have the intermediate certificates needed to verify the chain

## How to Fix Error -10835

### 1. Verify the Certificate Chain

Check the full certificate chain of the application:

```bash
# Display the code signing information
codesign -dv /Applications/YourApp.app

# Verify with full details
codesign --verify --deep --strict --verbose=4 /Applications/YourApp.app

# Display the signing certificate chain
codesign -dvvv /Applications/YourApp.app
```

### 2. Install Missing Intermediate Certificates

If the certificate chain is incomplete:

```bash
# Check which certificates are in the system keychain
security find-identity -v -p codesigning

# List all certificates
security find-certificate -c "Developer ID" -a /Library/Keychains/System.keychain

# Import an intermediate certificate
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain /path/to/certificate.cer
```

### 3. Check Certificate Revocation

Verify if Apple has revoked the developer's certificate:

```bash
# Check revocation status
spctl --assess --verbose /Applications/YourApp.app 2>&1

# Check the certificate's OCSP responder
codesign --verify --verbose /Applications/YourApp.app
```

### 4. Remove and Reinstall the Application

If the certificate is invalid or revoked:

```bash
# Remove the application
rm -rf /Applications/YourApp.app

# Remove quarantine data
defaults delete com.apple.LaunchServices

# Download a fresh copy from the official source
# Ensure the developer's certificate is valid before reinstalling

# Remove quarantine from the new download
xattr -d com.apple.quarantine ~/Downloads/YourApp.app

# Move to Applications
mv ~/Downloads/YourApp.app /Applications/
```

### 5. Trust the Certificate Manually

For development or enterprise certificates:

```bash
# Add a certificate to the system keychain
sudo security add-cert -k /Library/Keychains/System.keychain /path/to/certificate.cer

# Or trust it for code signing
sudo security trust-cert -d -r trustAsRoot -k /Library/Keychains/System.keychain /path/to/certificate.cer

# Verify trust
security find-certificate -c "YourCertName" -a /Library/Keychains/System.keychain
```

### 6. Reset the Code Signing Cache

Clear cached signing information:

```bash
# Clear the code signing cache
sudo rm -rf /Library/Caches/com.apple.coresymbolicationd
sudo rm -rf ~/Library/Caches/com.apple.appstore

# Restart the system
sudo reboot
```

## Examples

This error commonly occurs when:

- Installing enterprise applications signed with internal certificates
- Running development builds signed with a personal certificate
- Opening apps from developers whose certificate was recently revoked
- Using apps signed with expired certificates
- Installing apps from sources that don't use Apple's notarization system
