# macOS Error -10840 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10840 (kLSDataUnavailable) indicates that Launch Services encountered a data unavailability error while performing an internal operation. This error in the -1083x family typically manifests when Launch Services tries to access its persistent store and finds the data in an unreadable or corrupted state. It is often seen after system crashes, power loss, or interrupted macOS updates.

## Common Causes

- The Launch Services persistent store was corrupted by a system crash
- An interrupted macOS update left Launch Services data incomplete
- Power loss during a write operation corrupted the database
- The APFS container has metadata inconsistencies
- Launch Services database file is locked by a hung process
- Insufficient permissions to read the Launch Services data directory

## How to Fix Error -10840

### 1. Force Rebuild Launch Services

Perform a complete rebuild of the Launch Services database:

```bash
# Kill all Launch Services related processes
pkill -f lsregister
pkill -f cfprefsd

# Force rebuild
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder and Dock
killall Finder
killall Dock
```

### 2. Check Filesystem Integrity

Verify the system volume is healthy:

```bash
# Check disk usage
df -h

# Verify the disk
diskutil verifyVolume disk1s1

# For deeper checks, boot into Recovery and run First Aid
```

### 3. Clear All System Caches

Remove corrupted cache files:

```bash
# Clear user caches
rm -rf ~/Library/Caches/*

# Clear system caches (requires sudo)
sudo rm -rf /Library/Caches/*

# Clear Launch Services specific data
rm -rf ~/Library/Caches/com.apple.LaunchServices*
sudo rm -rf /Library/Caches/com.apple.LaunchServices*

# Restart the system
sudo reboot
```

### 4. Check Permissions on Launch Services Directory

Ensure proper access:

```bash
# Check permissions on the Launch Services directory
ls -la /System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/

# Ensure the lsregister binary is executable
ls -la /System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister
```

### 5. Reset Preferences Daemon

The preferences daemon manages Launch Services data:

```bash
# Restart the preferences daemon
sudo launchctl kickstart -k system/com.apple.cfprefsd

# Restart Finder
killall Finder
```

### 6. Restore from Backup

If the issue persists after trying the above:

```bash
# Check if Time Machine backup is available
tmutil listbackups

# You can restore specific files from Time Machine
# Or boot into Recovery and restore the entire system
```

## Examples

This error commonly occurs when:

- After a sudden power loss or kernel panic
- When resuming from a failed hibernation
- After an interrupted macOS update
- When the system volume has filesystem corruption
- During startup when Launch Services initializes before system volumes are ready
