# macOS Error -10811 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10811 (kLSDataUnavailable) indicates that Launch Services cannot retrieve the data it needs to perform the requested operation. This error typically occurs when the system is unable to read application metadata, such as the UTI (Uniform Type Identifier) database or application registration information. It can affect file opening, URL handling, and application launching.

## Common Causes

- Launch Services database is corrupted or missing
- The application's Info.plist is missing or unreadable
- The system-wide UTI database has been damaged
- A macOS update or migration left Launch Services in an inconsistent state
- File permissions prevent Launch Services from reading required data
- The application bundle is incomplete or corrupted

## How to Fix Error -10811

### 1. Rebuild the Launch Services Database

Force a complete rebuild of the Launch Services database:

```bash
# Kill the Launch Services agent
sudo killall -9 lsregister

# Rebuild the database from scratch
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder
```

### 2. Repair Disk Permissions

Corrupted permissions can prevent Launch Services from reading data:

```bash
# Verify and repair disk permissions (macOS Mojave and earlier)
sudo diskutil resetUserPermissions / $(id -u)

# For macOS Catalina and later, use First Aid in Recovery mode
# Or run:
sudo /usr/libexec/repair_packages --verify --standard-pkgs /
```

### 3. Clear System Caches

Corrupted caches can cause data unavailability:

```bash
# Clear Launch Services cache
rm -rf ~/Library/Caches/com.apple.LaunchServices*
rm -rf /Library/Caches/com.apple.LaunchServices*

# Clear the font cache
sudo atsutil databases -remove

# Restart the Launch Services daemon
sudo launchctl kickstart -k system/com.apple.Finder
```

### 4. Reset the App State

Reset the application state data:

```bash
# Remove app state files
rm -f ~/Library/Saved\ Application\ State/com.apple.finder.savedState
rm -f ~/Library/Saved\ Application\ State/com.apple.dock.savedState

# Restart Finder and Dock
killall Finder
killall Dock
```

### 5. Re-register the Application

Force the application to re-register with Launch Services:

```bash
# Open the application once to register it
open -a "ApplicationName"

# Verify registration
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump | grep -A 5 "ApplicationName"
```

### 6. Run First Aid in Recovery Mode

If the issue persists, the system volume may need repair:

1. Shut down your Mac
2. Boot into Recovery mode (Command+R on Intel, power button hold on Apple Silicon)
3. Open Disk Utility
4. Select "Macintosh HD" and run First Aid
5. Restart after repair completes

## Examples

This error commonly occurs when:

- After a macOS major version upgrade
- When restoring from a Time Machine backup
- Opening files after resetting the user account
- Launching apps that were migrated from another Mac
- Running `open` commands in Terminal after system changes
