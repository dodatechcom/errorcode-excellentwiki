# macOS Error -10836 (kLSDataUnavailable) — Application Not Trusted

Error -10836 (kLSDataUnavailable / errLSAppNotTrusted) indicates that Launch Services cannot verify the trustworthiness of the application. Similar to error -10830, this error occurs when macOS security mechanisms flag an application as untrusted. Error -10836 specifically indicates that the trust verification process itself failed or returned an indeterminate result, often due to missing or expired certificates in the trust chain.

## Common Causes

- The application's trust chain cannot be fully verified
- Intermediate or root certificates are missing from the system keychain
- The application was signed but the signing certificate has expired
- The trust evaluation timed out due to network issues (OCSP/CRL check)
- The application has been modified after signing, invalidating the trust
- macOS security policy prevents the application from being trusted

## How to Fix Error -10836

### 1. Verify the Application Signature

Check the full signature and trust chain:

```bash
# Display code signing details
codesign -dv --verbose=4 /Applications/YourApp.app

# Verify the signature
codesign --verify --deep --strict /Applications/YourApp.app

# Check the certificate chain
security verify-cert -c /Applications/YourApp.app/Contents/Resources/embedded.provisionprofile
```

### 2. Update System Certificates

Ensure all root and intermediate certificates are current:

```bash
# Check for system updates
softwareupdate -l

# Install all available updates
softwareupdate -ia

# List installed certificates
security find-certificate -a /Library/Keychains/System.keychain
```

### 3. Clear the Trust Cache

Reset the system's trust evaluation cache:

```bash
# Clear trust cache
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder

# Clear the code signing cache
sudo rm -rf /Library/Caches/com.apple.coresymbolicationd

# Restart Finder
killall Finder
```

### 4. Remove and Reinstall the Application

A fresh installation with a valid signature resolves most trust issues:

```bash
# Remove the application
rm -rf /Applications/YourApp.app

# Remove application data
rm -rf ~/Library/Application\ Support/YourApp
defaults delete com.developer.YourApp

# Download fresh from the official source
# Verify the download
spctl --assess --verbose ~/Downloads/YourApp.app

# Remove quarantine and install
xattr -d com.apple.quarantine ~/Downloads/YourApp.app
mv ~/Downloads/YourApp.app /Applications/

# Launch once
open -a "YourApp"
```

### 5. Reset Gatekeeper

If Gatekeeper's trust database is corrupted:

```bash
# Reset Gatekeeper
sudo spctl --reset-default

# Clear Gatekeeper's cached assessments
sudo rm -rf /var/db/SystemPolicy-gkopaque.db
sudo rm -rf /var/db/SystemPolicypolicybundles

# Restart your Mac
sudo reboot
```

### 6. For Enterprise Applications

If using internally signed applications:

```bash
# Import the enterprise certificate
sudo security add-trusted-cert -d -r trustAsRoot -k /Library/Keychains/System.keychain /path/to/enterprise-cert.cer

# Or configure MDM to push the certificate
# Or create a configuration profile with the certificate payload
```

## Examples

This error commonly occurs when:

- Opening an app whose certificate chain includes an expired intermediate certificate
- Trying to run an app after a macOS update changed trust policies
- Launching an enterprise app without the proper root certificate installed
- Opening an app when the OCSP responder is unreachable
- Running a modified copy of a signed application
