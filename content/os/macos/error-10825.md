# macOS Error -10825 (kLSUnknownType) — Launch Services Unknown Type

Error -10825 (kLSUnknownType / errLSNoExtension) indicates that Launch Services could not determine the type of the file or resource being opened. This error occurs when the system cannot identify the UTI (Uniform Type Identifier), file extension, or MIME type of the item. Without knowing the type, Launch Services cannot find an appropriate application to handle it.

## Common Causes

- The file has no extension or has been renamed without an extension
- The file extension is non-standard or unrecognized by the system
- The UTI database does not contain an entry for this file type
- The file's type metadata has been stripped during transfer or copying
- The file was created by obscure or specialized software without proper UTI declaration
- The file's content does not match its declared extension

## How to Fix Error -10825

### 1. Identify the File Type

Determine what kind of file you're dealing with:

```bash
# Check the file type using the file command
file /path/to/file

# Check for a file extension
ls -la /path/to/file

# Check the UTI if any
mdls -name kMDItemContentType /path/to/file

# Use xxd to peek at file header bytes
xxd /path/to/file | head -5
```

### 2. Add or Correct the File Extension

If the file is missing an extension or has the wrong one:

```bash
# Rename to add the correct extension
mv /path/to/file /path/to/file.txt

# Rename to correct a wrong extension
mv /path/to/file.pdf /path/to/file.doc

# Check the file content first to determine the correct extension
file --extension /path/to/file
```

### 3. Open with a Specific Application

Bypass UTI detection by specifying the application directly:

```bash
# Open with a specific application
open -a "TextEdit" /path/to/file

# Open a disk image
open /path/to/file.dmg

# Open a PDF with Preview
open -a "Preview" /path/to/file.pdf
```

### 4. Set the UTI for a Custom File Type

For custom file types, register the UTI:

```bash
# Create a UTI declaration in your app's Info.plist
# Or use a Finder sync extension to register the type

# Check existing UTI registrations
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump | grep -A 10 "public.filename-extension"
```

### 5. Restore File Metadata

If metadata was stripped during transfer:

```bash
# Check extended attributes
ls -l@ /path/to/file

# Check the quarantine attribute
xattr -l /path/to/file

# For files received via AirDrop or network transfer
# Try re-downloading or re-copying the file
```

### 6. Use Terminal to View the File

Read the file content directly from Terminal:

```bash
# View text files
cat /path/to/file

# View the first few lines
head -20 /path/to/file

# Search for readable strings in binary files
strings /path/to/file | head -20
```

## Examples

This error commonly occurs when:

- Opening a file that was renamed and lost its extension
- Double-clicking a file with a custom or proprietary extension
- Opening files created by legacy software without proper UTI support
- Trying to open files downloaded from the internet without extensions
- Opening files from Windows that have different extension conventions
