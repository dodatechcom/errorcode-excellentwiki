# macOS Error -10827 (kLSNotEditable) — Launch Services Not Editable

Error -10827 (kLSNotEditable / errLSDataUnavailable) indicates that Launch Services cannot modify the requested data or registry entry. This error occurs when trying to update or change Launch Services information such as file type associations, URL scheme handlers, or application registration data. The system or permissions prevent the modification.

## Common Causes

- The Launch Services database is read-only or locked by the system
- File type associations are managed by a system policy that prevents changes
- SIP (System Integrity Protection) blocks modification of system-level handlers
- The application's registration data is stored in a read-only location
- The user account does not have sufficient privileges to modify the association
- The UTI registration is provided by a system framework and cannot be overridden

## How to Fix Error -10827

### 1. Use System Preferences to Change Associations

The GUI method for changing file associations:

1. Right-click the file in Finder
2. Select "Get Info"
3. Under "Open with," select the desired application
4. Click "Change All" to apply to all files of this type

### 2. Modify User-Level Associations

Use Terminal to set associations at the user level:

```bash
# Set default application for a file type (user level)
defaults write com.apple.LaunchServices/com.apple.launchservices.secure LSHandlers -array-add '{LSHandlerContentType = "public.plain-text"; LSHandlerRoleAll = "com.apple.TextEdit";}'

# Restart Finder to apply
killall Finder
```

### 3. Use the lsregister Tool

Force an update to the Launch Services database:

```bash
# Rebuild with user-level changes
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain user

# Restart Finder
killall Finder
```

### 4. Check and Modify with PlistBuddy

Use PlistBuddy to inspect and modify Launch Services preferences:

```bash
# View current handlers
/usr/libexec/PlistBuddy -c "Print LSHandlers" ~/Library/Preferences/com.apple.LaunchServices.plist

# Add a new handler entry
/usr/libexec/PlistBuddy -c "Add :LSHandlers:0 dict" ~/Library/Preferences/com.apple.LaunchServices.plist
/usr/libexec/PlistBuddy -c "Add :LSHandlers:0:LSHandlerContentType string public.plain-text" ~/Library/Preferences/com.apple.LaunchServices.plist
/usr/libexec/PlistBuddy -c "Add :LSHandlers:0:LSHandlerRoleAll string com.apple.TextEdit" ~/Library/Preferences/com.apple.LaunchServices.plist
```

### 5. Check SIP Status

System Integrity Protection may block changes:

```bash
# Check SIP status
csrutil status

# SIP cannot and should not be disabled for normal file association changes
# If you need system-level UTI changes, use a Configuration Profile instead
```

### 6. Use Third-Party Tools

For advanced UTI management:

```bash
# Install duti for command-line UTI management
brew install duti

# List current handlers
duti -x public.plain-text

# Set default handler
duti -s com.apple.TextEdit "public.plain-text" all
```

## Examples

This error commonly occurs when:

- Trying to change the default application for a system-protected file type
- Attempting to register a URL scheme that conflicts with a system scheme
- Modifying Launch Services data from a script with insufficient privileges
- Trying to override a UTI provided by a system framework
- Changing file associations managed by MDM or configuration profiles
