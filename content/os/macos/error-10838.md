# macOS Error -10838 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10838 (kLSDataUnavailable) indicates that Launch Services cannot access the data required to complete the requested operation. This variant of the -1083x family typically occurs when Launch Services tries to read application metadata that is temporarily unavailable or locked by another process. It may also occur during system startup when Launch Services initializes before all volumes are mounted.

## Common Causes

- Launch Services attempted to read data from a volume that is not yet mounted
- Another process has locked the Launch Services database
- The application's metadata is temporarily unavailable during an update
- Network volumes or external drives are disconnected during a Launch Services operation
- The Launch Services daemon encountered a race condition during initialization
- Filesystem journaling conflicts prevent data access

## How to Fix Error -10838

### 1. Restart the Launch Services Daemon

Restart the Launch Services subsystem:

```bash
# Restart Finder (which manages Launch Services UI)
killall Finder

# If the issue persists, restart the system
sudo reboot
```

### 2. Check Mounted Volumes

Ensure all expected volumes are mounted:

```bash
# List all mounted volumes
diskutil list

# Check mounted filesystems
mount

# Remount a volume if needed
diskutil mount disk2s1
```

### 3. Clear Launch Services Locks

Remove any stale lock files:

```bash
# Check for lock files
ls -la /var/db/lsregister*
ls -la /Library/LaunchServices/

# Remove stale locks (be careful)
sudo rm -f /var/db/lsregister.lock
```

### 4. Rebuild the Launch Services Database

Force a fresh rebuild:

```bash
# Kill existing processes
pkill -f lsregister

# Rebuild from scratch
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder
```

### 5. Check System Logs

Investigate what caused the data unavailability:

```bash
# Check system logs for Launch Services errors
log show --predicate 'subsystem == "com.appleLaunchServices"' --last 1h

# Check for related filesystem errors
log show --predicate 'eventMessage contains "lsregister"' --last 1h

# Or use Console app
open /Applications/Utilities/Console.app
```

### 6. Ensure Proper Shutdown

Prevent this error by shutting down properly:

```bash
# Check for improper shutdown indicators
ls -la /private/var/folders/.byf*

# Ensure the system shuts down cleanly
# Always use the Apple menu > Shut Down instead of force shutdown
```

## Examples

This error commonly occurs when:

- During system startup before all volumes are mounted
- When external drives are disconnected during file operations
- After a kernel panic or system freeze
- When launching apps from a network share that briefly lost connection
- During a macOS update when system files are being replaced
