# macOS Error -10839 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10839 (kLSDataUnavailable) indicates that Launch Services encountered a data availability issue during a registration or lookup operation. This error in the -1083x family typically occurs when Launch Services attempts to read or write its internal registry and encounters an inconsistency, such as a partially written record or a stale reference to a moved or deleted application.

## Common Causes

- Launch Services registry contains stale entries from deleted applications
- The database is in an inconsistent state due to concurrent access
- An application was moved or renamed while Launch Services was reading its data
- The system volume has minor filesystem corruption
- Launch Services cached data conflicts with the actual filesystem state
- Multiple applications are registering simultaneously, causing race conditions

## How to Fix Error -10839

### 1. Rebuild the Launch Services Registry

Force a complete rebuild to resolve stale entries:

```bash
# Kill existing Launch Services processes
pkill -f lsregister

# Rebuild the entire registry
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder
```

### 2. Clear Stale Application Entries

Remove references to deleted applications:

```bash
# Dump the registry and look for stale entries
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump | grep -B 2 -A 5 "path:"

# Force a clean rebuild to remove stale entries
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -R -domain local -domain system -domain user

# Reseed the registry
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -seed
```

### 3. Repair the System Volume

Filesystem corruption can cause data availability issues:

```bash
# Verify the system volume
diskutil verifyVolume disk1s1

# Check for errors
fsck_apfs -n /dev/disk1s1

# If errors are found, run First Aid in Recovery mode
```

### 4. Clear Application State Data

Remove stale application state that may reference deleted apps:

```bash
# Clear saved application states
rm -rf ~/Library/Saved\ Application\ State/*

# Clear recent items
defaults delete com.apple.recentitems

# Restart Finder
killall Finder
```

### 5. Restart the System

A full system restart clears all transient states:

```bash
# Save any open work, then restart
sudo reboot
```

### 6. Check for Problematic Applications

Some applications can corrupt Launch Services data:

```bash
# Check recently installed applications
ls -lt /Applications/ | head -20

# Verify recently installed apps are properly signed
codesign --verify --deep --strict /Applications/RecentlyInstalled.app
```

## Examples

This error commonly occurs when:

- After deleting an application that was set as a default handler
- When multiple apps try to register for the same UTI simultaneously
- After moving applications between volumes or drives
- When the system volume runs out of space during Launch Services updates
- After restoring individual applications from Time Machine without their dependencies
