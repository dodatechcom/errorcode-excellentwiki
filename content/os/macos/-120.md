# macOS Error -120 (tmfoErr) — Too Many Files Open

Error -120 (tmfoErr) means the system has exhausted its available file descriptors. On macOS, this appears when applications, servers, or scripts open more files than the per-process limit allows. Every open file, socket, pipe, and device counts toward this limit. The error is common in web servers, database engines, and applications that handle many concurrent connections or large numbers of files.

## Common Causes

- Per-process file descriptor limit reached (default is 256 on macOS)
- Application leaking file descriptors without closing them
- Too many concurrent network connections open
- Large directory listings or recursive file operations
- Docker or Node.js applications with high connection counts
- macOS system-wide file descriptor limit exceeded

## How to Fix Error -120

### 1. Check Current Limits

See what limits are set on your system:

```bash
# Check per-process soft and hard limits
ulimit -n -H    # Hard limit
ulimit -n -S    # Soft limit

# Check all limits
ulimit -a

# See the system-wide maximum
sysctl kern.maxfiles
sysctl kern.maxfilesperproc
```

### 2. Increase the Per-Process Limit

Raise the file descriptor limit for your current session:

```bash
# Temporary increase (current shell session only)
ulimit -n 10240

# Increase for a specific command
launchctl limit maxfiles 10240 20480
```

### 3. Make the Limit Permanent with launchd

macOS uses `launchctl` for persistent limits:

```bash
# Check current launchd limits
launchctl limit

# Set new limits (requires sudo and reboot)
sudo launchctl limit maxfiles 65536 200000
sudo launchctl limit maxproc 2048 4096

# Apply immediately without reboot
sudo sysctl -w kern.maxfiles=65536
sudo sysctl -w kern.maxfilesperproc=200000
```

### 4. Create a launchd Property List

Create a permanent configuration file:

```bash
# Create the launchd plist for file limits
sudo tee /Library/LaunchDaemons/limits.maxfiles.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>limits.maxfiles</string>
    <key>ProgramArguments</key>
    <array>
        <string>launchctl</string>
        <string>limit</string>
        <string>maxfiles</string>
        <string>65536</string>
        <string>200000</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
</dict>
</plist>
EOF
```

### 5. Find Processes Using Many File Descriptors

Identify which process is consuming file descriptors:

```bash
# Show file descriptor count for all processes
lsof | awk '{print $2}' | sort | uniq -c | sort -rn | head -20

# Show open files for a specific process
lsof -p <PID> | wc -l

# Monitor file descriptors in real time
lsof -r 2 | awk '{print $2}' | sort | uniq -c | sort -rn | head -10

# Use ps to show open file count
ps aux | awk '{print $2, $11}' | while read pid cmd; do
    count=$(lsof -p "$pid" 2>/dev/null | wc -l)
    echo "$count $pid $cmd"
done | sort -rn | head -10
```

### 6. Close Leaked File Descriptors

If an application is leaking file descriptors:

```bash
# List all open files for a process
lsof -p <PID> | head -50

# Check for many instances of the same file
lsof -p <PID> | awk '{print $4}' | sort | uniq -c | sort -rn

# Restart the leaking process
sudo launchctl stop com.example.myservice
sudo launchctl start com.example.myservice
```

### 7. Configure Node.js for High File Counts

Node.js applications often hit tmfoErr:

```bash
# Increase ulimit before starting Node
ulimit -n 65536 && node server.js

# Or set the environment variable
NODE_OPTIONS="--max-old-space-size=4096" node server.js

# Check how many file descriptors Node has open
node -e "console.log(process.resourceUsage().maxfds)"
```

### 8. Configure Docker Desktop Limits

Docker Desktop on macOS has its own limits:

```bash
# Check Docker's open file limit
docker run --rm alpine sh -c "ulimit -n"

# Run a container with increased limits
docker run --ulimit nofile=65536:200000 myimage

# Configure in Docker Desktop:
# Settings > Docker Engine > Add: "default-ulimits": {"nofile": {"Name": "nofile", "Hard": 65536, "Soft": 65536}}
```

### 9. Use sysctl to Raise System-Wide Limits

```bash
# View current system-wide limits
sysctl kern.maxfiles
sysctl kern.maxfilesperproc

# Set new limits
sudo sysctl -w kern.maxfiles=65536
sudo sysctl -w kern.maxfilesperproc=200000

# Make permanent by adding to /etc/sysctl.conf
echo "kern.maxfiles=65536" | sudo tee -a /etc/sysctl.conf
echo "kern.maxfilesperproc=200000" | sudo tee -a /etc/sysctl.conf
```

### 10. Monitor with Activity Monitor

Use the GUI to monitor file descriptor usage:

1. Open Activity Monitor (Spotlight → "Activity Monitor")
2. Go to the **Open Files and Ports** tab
3. Select a process and click **i** info button
4. Check the number of open files

Or use the command line:

```bash
# Show top processes by open file count
for pid in $(ps -eo pid=); do
    count=$(lsof -p "$pid" 2>/dev/null | wc -l)
    name=$(ps -p "$pid" -o comm= 2>/dev/null)
    [ "$count" -gt 50 ] && echo "$count $name (PID $pid)"
done | sort -rn | head -10
```
