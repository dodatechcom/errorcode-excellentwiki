# macOS Error -10820 (kLSApplicationNotRunning) — Application Not Running

Error -10820 (kLSApplicationNotRunning) indicates that Launch Services attempted to communicate with an application that is not currently running. This error typically occurs when trying to send Apple Events, URLs, or file open requests to an application that has crashed, been force-quit, or was never launched. It can also appear when the application quit unexpectedly during a previous operation.

## Common Causes

- The target application crashed or was force-quit before the operation completed
- Apple Events are being sent to an application that is not running
- The application was expected to be running but hasn't been launched yet
- The application quit unexpectedly during a previous interaction
- Launch Services cache still references a stale process ID
- The application is sandboxed and cannot receive inter-process communication

## How to Fix Error -10820

### 1. Launch the Application First

Ensure the target application is running before sending commands:

```bash
# Launch the application
open -a "ApplicationName"

# Wait for it to fully launch
sleep 3

# Then perform the operation
```

### 2. Check for Stale Process References

Clean up stale process references:

```bash
# Check if the application is running
pgrep -x "ApplicationName"

# Force kill if it's in a bad state
pkill -x "ApplicationName"

# Relaunch it
open -a "ApplicationName"
```

### 3. Use Apple Events with Error Handling

When using AppleScript, handle the not-running case:

```applescript
tell application "Finder"
    activate
end tell

-- Or with explicit launching
tell application "ApplicationName"
    activate
    -- your commands here
end tell
```

### 4. Reset Launch Services State

Clear stale Launch Services state:

```bash
# Remove saved application state
rm -rf ~/Library/Saved\ Application\ State/com.apple.*.savedState

# Restart Finder
killall Finder

# Restart the Dock
killall Dock
```

### 5. Check Console for Crash Logs

If the application is crashing:

```bash
# Check for crash logs
ls ~/Library/Logs/DiagnosticReports/ | grep "ApplicationName"

# View the most recent crash log
cat ~/Library/Logs/DiagnosticReports/ApplicationName_$(ls -t ~/Library/Logs/DiagnosticReports/ | grep "ApplicationName" | head -1 | cut -d. -f1).ips

# Or use the Console app
open /Applications/Utilities/Console.app
```

### 6. Reset the Application

If the application consistently fails to stay running:

```bash
# Remove application preferences
defaults delete com.developer.ApplicationName

# Remove application support files
rm -rf ~/Library/Application\ Support/ApplicationName

# Reinstall the application
rm -rf /Applications/ApplicationName.app
# Download and reinstall from App Store or developer website
```

## Examples

This error commonly occurs when:

- Running AppleScript that sends events to an application that isn't running
- Trying to open a URL in a browser that crashed
- Using Automator workflows that depend on a specific application
- Calling `open -a` on an app that crashes on launch
- Interacting with background apps that have been suspended
