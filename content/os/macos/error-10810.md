# macOS Error -10810 (kLSApplicationNotFound) — Launch Services Application Not Found

Error -10810 (kLSApplicationNotFound) is a Launch Services error indicating that the system could not find an application registered to handle the requested URL scheme, file type, or UTI (Uniform Type Identifier). This error typically occurs when you try to open a file or URL and no installed application is associated with that type, or the previously associated application has been moved or deleted.

## Common Causes

- The application that previously handled the file type or URL scheme has been deleted or moved
- No application is registered to open the specific file type or URL scheme
- The application's Info.plist does not declare support for the UTI or URL scheme
- Launch Services database is corrupted or out of date
- The application was downloaded from the internet and quarantine prevents registration
- The app is in the Applications folder but has not been opened at least once

## How to Fix Error -10810

### 1. Re-register Applications with Launch Services

Force Launch Services to rebuild its database:

```bash
# Reset Launch Services database (requires logout/restart)
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder to apply changes
killall Finder
```

### 2. Check Application Registration

Verify which applications are registered to handle a specific UTI or URL scheme:

```bash
# Find apps registered for a UTI
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump | grep -A 5 "public.url-type"

# Check for apps that handle a URL scheme
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump | grep -A 5 "x-apple:"
```

### 3. Manually Associate File Types

Set the default application for a file type:

```bash
# Open a file with a specific application (registers it)
open -a "ApplicationName" /path/to/file

# Or use duti to set default UTI handlers (install via brew install duti)
duti -s com.apple.Safari "public.url" all
```

### 4. Check the Quarantine Attribute

Quarantined applications may not register properly:

```bash
# Check quarantine attribute
ls -l@ /Applications/YourApp.app

# Remove quarantine attribute
xattr -d com.apple.quarantine /Applications/YourApp.app
```

### 5. Reset Launch Services via Finder

1. Open Finder
2. Navigate to a file of the affected type
3. Right-click and select "Get Info"
4. Under "Open with," select the correct application
5. Click "Change All"

### 6. Reinstall the Application

If the application was moved or corrupted, reinstall it from the App Store or the developer's website, then open it at least once to register it with Launch Services.

## Examples

This error commonly occurs when:

- Clicking a mailto: link when no email client is configured
- Opening a .psd file when Adobe Photoshop is not installed
- Trying to open a custom URL scheme (myapp://) for an app that has been deleted
- Double-clicking a file type that no application claims to handle
- Running `open` command on a file with no registered handler
