# macOS Error -10842 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10842 (kLSDataUnavailable) indicates that Launch Services encountered a data retrieval failure when attempting to process a request. This error in the -1083x family is typically triggered when the system tries to look up application registration information and the data store is temporarily inaccessible or contains invalid entries. It is common after system modifications or when Launch Services is under heavy load.

## Common Causes

- Launch Services is processing too many concurrent requests
- The internal data store has become fragmented or corrupted
- A system update modified Launch Services data structures
- Third-party cleanup utilities removed Launch Services data
- The system is low on memory, causing Launch Services to fail
- Multiple applications are being registered simultaneously

## How to Fix Error -10842

### 1. Rebuild the Launch Services Database

Force a complete rebuild:

```bash
# Kill existing Launch Services processes
pkill -f lsregister

# Rebuild from scratch
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder and Dock
killall Finder
killall Dock
```

### 2. Check System Memory

Low memory can cause Launch Services to fail:

```bash
# Check memory usage
top -l 1 -s 0 | head -10

# Check swap usage
sysctl vm.swapusage

# Free up memory by clearing caches
sudo purge
```

### 3. Clear Fragmented Cache Data

Remove fragmented Launch Services data:

```bash
# Clear user-level Launch Services cache
rm -rf ~/Library/Caches/com.apple.LaunchServices*
rm -rf ~/Library/Preferences/com.apple.LaunchServices.plist

# Clear system-level cache
sudo rm -rf /Library/Caches/com.apple.LaunchServices*

# Restart Finder
killall Finder
```

### 4. Check for Third-Party Cleanup Interference

Some utilities may have removed Launch Services data:

```bash
# Check if common cleanup utilities are installed
ls /Applications/ | grep -i "clean\|optim\|boost"

# If present, check their exclusion lists
# Ensure Launch Services data directories are excluded from cleanup

# Verify the lsregister binary exists
ls -la /System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister
```

### 5. Restart the System

A clean restart resolves most transient issues:

```bash
# Clear user-level caches before restarting
rm -rf ~/Library/Caches/*

# Restart
sudo reboot
```

### 6. Monitor Launch Services Activity

Check what's happening with Launch Services:

```bash
# Watch Launch Services activity
log stream --predicate 'subsystem == "com.appleLaunchServices"' --level debug

# Check for errors in the stream
log show --predicate 'subsystem == "com.appleLaunchServices" AND messageType == 16' --last 1h
```

## Examples

This error commonly occurs when:

- Opening many files simultaneously (batch operations)
- After installing or uninstalling multiple applications at once
- When using Automator workflows that trigger many app launches
- During system startup when multiple apps try to register
- When running build scripts that create many temporary files
