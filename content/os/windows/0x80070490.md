# [Solution] Error 0x80070490 Windows 11/10 — Element Not Found Fix

Error 0x80070490 is a Windows Update error with the message `Element not found`. It occurs when the Windows Update agent or servicing stack cannot locate a required component, preventing updates from being installed.

This error affects both Windows 10 and 11 and commonly blocks cumulative updates, security patches, and feature upgrades.

## Description

The full error message typically reads:

> "There were problems installing some updates, but we'll try again later. Error 0x80070490 — Element not found"

Or in Windows Update history:

> "Security Update for Windows (KBXXXXXXX) — Error 0x80070490"

Error 0x80070490 maps to `ERROR_NOT_FOUND` in the Windows error codes. The Windows Update agent or servicing stack is looking for a specific component, manifest, or file that doesn't exist in the expected location.

Common scenarios include:

- **Cumulative updates** failing during the install phase
- **Feature updates** refusing to start the installation
- **Servicing stack updates** (SSU) failing
- **.NET Framework updates** encountering missing components
- **Definition updates** for Windows Defender failing

## Common Causes

1. **Corrupted servicing stack** — The Windows servicing components are damaged or incomplete.
2. **Missing system files** — Required manifests or files have been deleted or corrupted.
3. **Corrupted SoftwareDistribution folder** — The update download cache is damaged.
4. **Registry corruption** — Windows Update registry entries are invalid.
5. **Third-party software conflicts** — System modification tools breaking Windows Update components.

## Solutions

### Solution 1: Run DISM to Repair the System Image

DISM is the most effective tool for fixing 0x80070490 because it repairs the servicing stack that Windows Update depends on.

```cmd
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth
```

If DISM cannot find a repair source, provide one using a Windows ISO:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth /Source:D:\Sources\install.wim
```

Replace `D:` with the drive letter of your mounted Windows ISO.

### Solution 2: Clear the SoftwareDistribution Folder

The download cache may contain corrupted manifests that cause the element-not-found error.

```cmd
net stop wuauserv
net stop bits
```

Rename the cache:

```cmd
ren "C:\Windows\SoftwareDistribution" "SoftwareDistribution.old"
```

Delete the old cache after confirming Windows Update works:

```cmd
rmdir /s /q "C:\Windows\SoftwareDistribution.old"
```

Restart services:

```cmd
net start wuauserv
net start bits
```

### Solution 3: Repair Windows Update Registry Keys

Corrupted registry entries can prevent Windows Update from finding required components.

**Export the registry first:**

```cmd
reg export "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" C:\WU_Backup.reg
```

**Reset Windows Update registry entries:**

```cmd
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /v "SusClientId" /t REG_SZ /d "00000000-0000-0000-0000-000000000000" /f
```

### Solution 4: Run the Windows Update Troubleshooter

The built-in troubleshooter can automatically detect and fix common update issues.

```powershell
msdt.exe /id WindowsUpdateDiagnostic
```

Or through the Settings app:

1. Open **Settings > System > Troubleshoot > Other troubleshooters**.
2. Click **Run** next to **Windows Update**.
3. Follow the prompts and apply any recommended fixes.

### Solution 6: Manually Install the Update

Download and install the update directly from the Microsoft Update Catalog:

1. Find the KB number in Windows Update history.
2. Visit [catalog.update.microsoft.com](https://www.catalog.update.microsoft.com/).
3. Search for the KB number.
4. Download the correct version for your architecture.
5. Run the `.msu` file as administrator.

### Solution 7: Check for Corrupted System Components

Use SFC to repair any remaining corrupted files:

```cmd
sfc /scannow
```

If issues persist, check the CBS log for specific failures:

```powershell
Get-Content "C:\Windows\Logs\CBS\CBS.log" | Select-String "Error" | Select-Object -Last 20
```
