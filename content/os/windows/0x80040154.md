# [Solution] Error 0x80040154 Windows 11/10 — Class Not Registered Fix

Error 0x80040154 — Class not registered — occurs when a COM component or DLL is not properly registered in the Windows Registry. This typically happens after a failed installation or system corruption.

## Description

The full error message reads:

> "Class not registered (0x80040154)"

This error appears when an application tries to instantiate a COM object whose CLSID is missing from the registry. Common scenarios include:

- **Running a 32-bit COM component on a 64-bit system** — The DLL is registered for the wrong architecture.
- **Missing DLL registration** — A required COM DLL has been unregistered or never registered.
- **Application reinstallation** — After uninstalling an older version, the new version fails to register its components.
- **Windows Update or upgrade** — System files are replaced but COM registrations are lost.

## Common Causes

1. **Unregistered COM DLL** — The required COM DLL was never registered or was unregistered by another process.
2. **32-bit vs 64-bit mismatch** — A 32-bit COM component is registered only in the 64-bit registry hive, or vice versa.
3. **Corrupted registry entries** — The CLSID key for the component is missing or damaged.
4. **Missing dependencies** — A required runtime DLL (like Visual C++ Redistributable) is not installed.
5. **Application partial uninstall** — The application was removed but left orphaned registry entries without the actual DLL.

## Solutions

### Solution 1: Register the DLL with regsvr32

Find the DLL that corresponds to the failing COM class and register it manually:

```cmd
regsvr32 "C:\Path\To\YourComponent.dll"
```

If you do not know which DLL is missing, check the error details in **Event Viewer** under **Windows Logs > Application**. Look for the CLSID and search the registry under `HKEY_CLASSES_ROOT\CLSID` for that GUID to find the DLL path.

### Solution 2: Check 32-bit vs 64-bit Registration

On a 64-bit system, 32-bit COM components must be registered using the 32-bit version of regsvr32:

```cmd
C:\Windows\SysWOW64\regsvr32 "C:\Path\To\32BitComponent.dll"
```

The 64-bit regsvr32 at `C:\Windows\System32\regsvr32.exe` will only register 64-bit DLLs. Use the SysWOW64 version for 32-bit DLLs.

### Solution 3: Reinstall the Application

The cleanest fix is to reinstall the application that produces the error:

1. Open **Settings > Apps > Installed apps**.
2. Find the application, click the three dots, and select **Uninstall**.
3. Restart your computer.
4. Download the latest version from the official source and reinstall.

### Solution 4: Repair Visual C++ Redistributables

Many COM components depend on Visual C++ runtimes. Repair all installed redistributables:

1. Open **Settings > Apps > Installed apps**.
2. Sort by publisher and locate all **Microsoft Visual C++ Redistributable** entries.
3. For each entry, click **Modify** and select **Repair**.
4. Restart your computer after all repairs complete.

### Solution 5: Re-register All COM Components in a Directory

If the error is widespread, re-register all DLLs in the application folder:

```cmd
for %i in ("C:\Program Files\YourApp\*.dll") do regsvr32 /s "%i"
```

Run this from an elevated Command Prompt, replacing the path with your application's directory.
