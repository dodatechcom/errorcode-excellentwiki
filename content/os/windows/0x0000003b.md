# [Solution] Error 0x0000003b Windows 11/10 — SYSTEM_SERVICE_EXCEPTION Fix

Error 0x0000003b is a BSOD STOP error with the bug check `SYSTEM_SERVICE_EXCEPTION`. It occurs when a system service routine encounters an unexpected exception, typically caused by a driver or system service making an illegal operation in kernel mode.

This error affects both Windows 10 and 11 and is commonly triggered by GPU drivers, network drivers, and third-party security software.

## Description

The full error message typically reads:

> "Your PC ran into a problem and needs to restart. Stop code: SYSTEM_SERVICE_EXCEPTION"

The BSOD may also reference a specific faulting file:

> "What failed: netio.sys" or "What failed: tcpip.sys" or "What failed: ntoskrnl.exe"

Error 0x0000003b means a system service (a kernel-mode routine) attempted an operation that resulted in an exception. This could be accessing invalid memory, performing an illegal instruction, or encountering a condition that the service wasn't designed to handle.

Common scenarios include:

- **During gaming** — GPU driver crash under heavy graphics load
- **Network operations** — Network driver or firewall driver failure
- **While browsing** — Browser-related driver conflict with security software
- **After Windows Update** — Updated system files conflicting with existing drivers
- **Random intervals** — Intermittent hardware or driver instability

## Common Causes

1. **Outdated or corrupted drivers** — Especially GPU, network, and storage drivers.
2. **Third-party antivirus conflicts** — Security software hooking into kernel-mode operations.
3. **Faulty RAM** — Memory corruption causing system service failures.
4. **Corrupted system files** — Damaged Windows system service routines.
5. **Incompatible software** — Programs that install kernel-mode components.

## Solutions

### Solution 1: Update or Reinstall GPU Drivers

GPU drivers are the most frequent trigger for this BSOD.

**Clean install of GPU drivers:**

1. Download the latest driver for your GPU from the manufacturer's website.
2. Download [DDU (Display Driver Uninstaller)](https://www.wagnardsoft.com/display-driver-uninstaller-DDU-).
3. Boot into **Safe Mode**.
4. Run DDU to completely remove existing GPU drivers.
5. Restart and install the fresh driver.

**Using Device Manager to rollback:**

```powershell
Get-PnpDevice -Class Display | Select-Object FriendlyName, InstanceId
```

```cmd
pnputil /delete-driver oemXX.inf /uninstall
```

### Solution 2: Disable Third-Party Antivirus

Antivirus software that hooks into the Windows kernel can cause system service exceptions.

**Temporarily disable your antivirus:**

1. Open your antivirus application.
2. Navigate to **Settings > Protection** or **Real-time scanning**.
3. Disable real-time protection.
4. Test if the BSOD stops occurring.

**Uninstall and use Windows Defender instead:**

```powershell
# Check which antivirus is active
Get-CimInstance -Namespace root/SecurityCenter2 -ClassName AntiVirusProduct | Select-Object displayName, productState
```

Windows Defender is built into Windows 10/11 and provides sufficient protection for most users.

### Solution 3: Check RAM for Errors

Memory corruption can cause system services to behave unpredictably.

**Run Windows Memory Diagnostic:**

1. Press `Win + R`, type `mdsched.exe`, press Enter.
2. Select **Restart now and check for problems**.

**Check recent memory diagnostic results:**

```powershell
Get-WinEvent -LogName System | Where-Object { $_.ProviderName -eq "Microsoft-Windows-MemoryDiagnostics-Results" } | Select-Object -First 3 Message
```

**For more thorough testing, use MemTest86:**

1. Download from [memtest86.com](https://www.memtest86.com/).
2. Create a bootable USB.
3. Boot from the USB and run for at least 4 complete passes.

### Solution 4: Run SFC and DISM

Repair corrupted system files that may be causing the exception:

```cmd
sfc /scannow
```

If SFC cannot fix files:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

### Solution 5: Check Event Viewer for the Faulting Driver

Identify the specific driver causing the crash:

1. Press `Win + X` and select **Event Viewer**.
2. Go to **Windows Logs > System**.
3. Filter for **Critical** and **Error** events around the time of the BSOD.
4. Look for bugcheck events (Event ID 1001) which contain the faulting module.

```powershell
Get-WinEvent -LogName System -FilterXPath "*[System[(EventID=1001)]]" -MaxEvents 5 | Select-Object TimeCreated, Message | Format-List
```

### Solution 6: Update Windows and All Drivers

Ensure all system files and drivers are current:

```powershell
# Check for pending Windows updates
$Updates = (New-Object -ComObject Microsoft.Update.Session).CreateUpdateSearcher().Search("IsInstalled=0")
$Updates.Updates | Select-Object Title, KBArticleIDs | Format-Table
```

Update all drivers through Device Manager:

```powershell
pnputil /scan-devices
```

Or use the manufacturer's driver update utility (Dell SupportAssist, HP Support Assistant, Lenovo Vantage).

### Solution 7: Perform Clean Boot

Isolate the problem by disabling all non-Microsoft services:

1. Press `Win + R`, type `msconfig`, press Enter.
2. Go to the **Services** tab.
3. Check **Hide all Microsoft services**.
4. Click **Disable all**.
5. Go to the **Startup** tab and click **Open Task Manager**.
6. Disable all startup items.
7. Restart and test.

If the BSOD stops, re-enable services one by one to find the culprit.
