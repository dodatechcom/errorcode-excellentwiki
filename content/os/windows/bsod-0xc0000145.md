# [Solution] BSOD 0xc0000145 STATUS_UNEXPECTED_IO_ERROR Windows 11/10 — Fixed

Error 0xc0000145 is a critical BSOD STOP error with the status code `STATUS_UNEXPECTED_IO_ERROR`. It occurs when Windows encounters an unexpected input/output error during a critical operation — typically reading from or writing to the system drive. The I/O subsystem cannot complete the requested operation, and the kernel halts to prevent data corruption.

This error affects both Windows 10 and 11 and is closely tied to storage hardware issues, driver problems, and disk corruption.

## Common Causes

- **Failing hard drive** — Physical degradation causes intermittent read/write failures that interrupt critical I/O operations.
- **Disk corruption** — Corrupted file system structures cause the I/O subsystem to encounter unexpected states.
- **Outdated storage drivers** — AHCI, NVMe, or SATA controller drivers with bugs in their I/O handling.
- **Cable or connection issues** — Loose or damaged SATA/NVMe cables cause intermittent I/O failures.

## How to Fix

### Check Disk Health

The first step is to determine if the drive is failing:

```powershell
Get-PhysicalDisk | Select-Object FriendlyName, HealthStatus, OperationalStatus
Get-StorageReliabilityCounter -PhysicalDisk (Get-PhysicalDisk) | Select-Object Temperature, Wear, ReadErrorsTotal, WriteErrorsTotal
```

A `HealthStatus` of `Unhealthy` or any `ReadErrorsTotal`/`WriteErrorsTotal` greater than 0 indicates a failing drive. Back up data immediately.

**Check SMART status:**

```powershell
Get-WmiObject -Namespace root\wmi -Class MSStorageDriver_FailurePredictStatus | Select-Object PredictFailure, Reason
```

### Run CHKDSK

```cmd
chkdsk C: /f /r /x
```

Press `Y` to schedule for next restart. The `/r` flag locates bad sectors and recovers readable data.

**If the system cannot boot**, boot from a Windows installation USB:

1. Select **Repair your computer**.
2. Go to **Troubleshoot > Advanced options > Command Prompt**.
3. Run:

```cmd
chkdsk C: /f /r /x
```

### Update Storage Drivers

```powershell
Get-WmiObject Win32_PnPSignedDriver | Where-Object { $_.DeviceClass -eq "SCSIAdapter" -or $_.DeviceClass -eq "HDC" } | Select-Object DeviceName, DriverVersion, DriverDate | Format-Table -AutoSize
```

Download the latest storage drivers from the motherboard or system manufacturer:
- **Intel**: Intel Rapid Storage Technology (IRST)
- **AMD**: AMD StoreMI or SATA/NVMe driver
- **NVMe**: Check the SSD manufacturer's website

### Check Physical Connections

If using a desktop PC:

1. Power off the computer and unplug it.
2. Open the case and check that SATA data and power cables are firmly connected.
3. Try a different SATA port on the motherboard.
4. Replace the SATA cable if it shows signs of damage.

For laptops, the storage connection is typically internal — have a technician inspect it if the error persists.

### Run System File Checker

```cmd
sfc /scannow
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

### Update BIOS/UEFI

An outdated BIOS can cause I/O handling bugs:

```cmd
wmic baseboard get product,Manufacturer,version
```

Visit your motherboard manufacturer's website and download the latest BIOS version.

### Monitor Drive Temperature

```powershell
Get-StorageReliabilityCounter -PhysicalDisk (Get-PhysicalDisk) | Select-Object Temperature
```

NVMe SSDs can thermal throttle and cause I/O errors. Ensure adequate cooling for M.2 drives. Consider adding a heatsink if the drive exceeds 70°C under load.

## Examples

This error commonly occurs in these scenarios:

- **With a failing hard drive** — Bad sectors cause random read failures that trigger I/O errors.
- **When the storage cable is loose** — A partially disconnected SATA cable causes intermittent I/O failures.
- **After a sudden power loss** — Interrupted write operations corrupt disk structures, causing subsequent I/O errors.
- **With an overheating NVMe SSD** — Thermal throttling causes the drive to stop responding to I/O requests.
