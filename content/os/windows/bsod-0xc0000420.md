# [Solution] BSOD 0xc0000420 STATUS_AUTHENTICATION_FIREWALL_FAILED Windows 11/10 — Fixed

Error 0xc0000420 is a critical BSOD STOP error with the status code `STATUS_AUTHENTICATION_FIREWALL_FAILED`. It occurs when the Windows authentication firewall encounters a fatal error while validating network authentication or filtering connections. This is typically triggered by third-party security software that hooks into the Windows filtering platform (WFP) or authentication subsystem.

This error affects both Windows 10 and 11 and is most commonly caused by conflicting antivirus or firewall products.

## Common Causes

- **Conflicting security software** — Two or more antivirus or firewall products running simultaneously and conflicting.
- **Corrupted Windows Filtering Platform** — The WFP driver or related components are damaged.
- **Incompatible VPN software** — VPN clients with network filter drivers that interfere with authentication.
- **Corrupted network drivers** — Network adapter drivers that interact with the authentication firewall incorrectly.

## How to Fix

### Remove Conflicting Security Software

The most common fix is removing the third-party security product causing the conflict:

1. Open **Settings > Apps > Installed apps**.
2. Uninstall any third-party antivirus or firewall software.
3. Restart your computer.
4. Windows Defender will activate automatically.

**Check if Windows Defender is active:**

```powershell
Get-MpComputerStatus | Select-Object RealTimeProtectionEnabled, AntivirusEnabled, AMServiceEnabled
```

If multiple security products are installed, only keep one active.

### Reinstall the Conflicting Security Product

If you need third-party security software:

1. Download the latest version from the vendor's website.
2. Uninstall the current version completely.
3. Restart your computer.
4. Install the fresh version.

### Disable and Re-enable Windows Firewall

If the Windows Filtering Platform is corrupted:

```cmd
net stop CryptSvc
net stop WinDefend
```

Wait 10 seconds, then restart:

```cmd
net start CryptSvc
net start WinDefend
```

**Or reset the Windows Firewall:**

```cmd
netsh advfirewall reset
```

### Update Network Drivers

```powershell
Get-WmiObject Win32_PnPSignedDriver | Where-Object { $_.DeviceClass -eq "NET" } | Select-Object DeviceName, DriverVersion, DriverDate | Format-Table -AutoSize
```

Download the latest network adapter drivers from the manufacturer's website.

### Update Windows

Windows updates include patches to the Windows Filtering Platform and authentication subsystem:

```powershell
Install-Module PSWindowsUpdate -Force
Get-WindowsUpdate -AcceptAll -Install -AutoReboot
```

### Run System File Checker

```cmd
sfc /scannow
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

### Check VPN Software

If you are using a VPN:

1. Uninstall the VPN client.
2. Restart your computer.
3. Check if the BSOD stops occurring.
4. Reinstall the latest version of the VPN client if needed.

## Examples

This error commonly occurs in these scenarios:

- **With multiple antivirus products installed** — Running Windows Defender alongside Norton, McAfee, or other security suites creates authentication filter conflicts.
- **After installing VPN software** — A VPN's network filter driver conflicts with the Windows Filtering Platform.
- **Following a Windows Update** — An update changes WFP behavior and breaks compatibility with installed security software.
- **With third-party firewalls** — Products like ZoneAlarm or Comodo Firewall hook deeply into the authentication subsystem.
