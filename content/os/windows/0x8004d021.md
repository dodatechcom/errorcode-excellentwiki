# [Solution] Error 0x8004D021 Windows 11/10 — COM+ Activation Failed Fix

Error 0x8004D021 — COM+ activation failed — occurs when the COM+ runtime cannot activate a COM+ application, typically due to service misconfiguration or component corruption.

## Description

The full message reads:

> "COM+ activation failed (0x8004D021)"

This COM+ error is closely related to 0x8004D003. Common scenarios include:

- **COM+ application launch failures** — A COM+ application fails to start in Component Services.
- **Windows component failures** — System features dependent on COM+ cannot initialize.
- **IIS application pool errors** — Web applications using COM+ components fail to load.
- **Enterprise application deployment** — Custom COM+ applications fail after server migration.

## Common Causes

1. **COM+ System Application service stopped** — The `COMSysApp` service needs to be restarted.
2. **Corrupted COM+ application configuration** — The application settings in the COM+ catalog are damaged.
3. **Missing or mismatched application roles** — Security roles are misconfigured or missing.
4. **Component registration issues** — The DLL or proxy/stub DLL is not properly registered.
5. **DCOM permissions insufficient** — The COM+ application identity lacks required security permissions.

## Solutions

### Solution 1: Restart the COM+ System Application Service

```cmd
net stop COMSysApp
net start COMSysApp
```

### Solution 2: Re-register COM+ Core Components

```cmd
regsvr32 comsvcs.dll
regsvr32 colbact.dll
```

### Solution 3: Check and Reset COM+ Application Identity

1. Open **Component Services** (`dcomcnfg`).
2. Navigate to **COM+ Applications**.
3. Right-click the failing application and select **Properties**.
4. On the **Identity** tab, note the current account.
5. Try changing to **Interactive user** to test, then switch back to a dedicated service account.

### Solution 4: Delete and Recreate the COM+ Application

If the application persists in a bad state, remove it and reinstall from its installer:

1. In Component Services, right-click the application and select **Delete**.
2. Run the application installer to recreate the COM+ configuration.
