# [Solution] Error 0x80070035 Windows 11/10 — Network Path Not Found Fix

Error 0x80070035 means Windows cannot locate the specified network path when trying to access a shared resource. This is a common networking error that occurs when attempting to connect to shared folders, network drives, or other computers on a local network.

## Description

The full error message reads: **"The network path was not found."** This error commonly appears when mapping network drives, accessing shared folders, or trying to connect to another computer using its IP address or hostname. It indicates a breakdown in the network communication chain between your computer and the target resource.

## Common Causes

1. Incorrect network path or share name
2. SMB (Server Message Block) protocol disabled or misconfigured
3. Firewall blocking network file sharing traffic
4. Network discovery turned off
5. Target computer or share is offline
6. Incorrect network credentials or permissions
7. Network profile set to Public instead of Private

## Solutions

### Solution 1: Verify Network Connectivity and Share Path

Confirm basic connectivity and the correct path:

1. Open **Command Prompt**
2. Ping the target computer:
   ```
   ping TARGET_IP
   ```
3. Replace **TARGET_IP** with the IP address of the target computer
4. Verify the share path is correct: `\\TARGET_IP\ShareName`
5. Check that the shared folder exists and sharing is enabled on the target

### Solution 2: Enable SMB 1.0/CIFS and Network Discovery

Ensure required network features are enabled:

1. Press **Win + R**, type `optionalfeatures`, and press Enter
2. Scroll down and find **SMB 1.0/CIFS File Sharing Support**
3. Check all boxes under it and click **OK**
4. Go to **Control Panel** > **Network and Sharing Center** > **Change advanced sharing settings**
5. Turn on **Network discovery** and **File and printer sharing** for your network profile
6. Restart your computer

### Solution 3: Check Firewall Settings

Ensure Windows Firewall is not blocking file sharing:

1. Open **Windows Defender Firewall** > **Allow an app through Windows Firewall**
2. Ensure **File and Printer Sharing** is allowed on your network profile
3. Alternatively, temporarily disable the firewall to test:
   ```
   netsh advfirewall set allprofiles state off
   ```
4. If the connection works, re-enable the firewall and configure exceptions properly
5. Re-enable the firewall:
   ```
   netsh advfirewall set allprofiles state on
   ```
