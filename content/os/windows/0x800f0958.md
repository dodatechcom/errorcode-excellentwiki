# [Solution] Error 0x800F0958 Windows 11/10 — DISM Source Download Failed Fix

Error 0x800F0958 — DISM source download failed — occurs when DISM cannot retrieve source files from Windows Update, either due to network issues or a corrupted component store.

## Description

The full message reads:

> "DISM source download failed (0x800F0958)"

This error is similar to 0x800F0956 and 0x800F0957, occurring during DISM repair operations. Common scenarios include:

- **DISM /RestoreHealth failures** — The tool cannot download repair files.
- **Windows Update connectivity problems** — Network restrictions prevent downloads.
- **Corporate restricted environments** — WSUS or Group Policy limits download sources.
- **Incomplete Windows installation** — Missing components from a failed upgrade.

## Common Causes

1. **Internet connectivity issues** — No internet access or unstable connection.
2. **Windows Update blocked by Group Policy** — Policy prevents direct downloads from Microsoft.
3. **Corrupted DISM cache** — Previously failed downloads corrupt the cache.
4. **Firewall blocking** — Windows Defender Firewall or third-party firewall blocks the connection.
5. **Incorrect proxy configuration** — Proxy settings prevent DISM from reaching update servers.

## Solutions

### Solution 1: Use Local Installation Media

Mount a Windows ISO and run DISM with the local source:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth /Source:D:\sources\install.wim /LimitAccess
```

### Solution 2: Check Internet and Proxy Configuration

```cmd
netsh winhttp show proxy
```

If behind a proxy, configure DISM to use it:

```cmd
netsh winhttp set proxy proxy-server="http://your-proxy:port"
```

### Solution 3: Clear DISM Logs and Retry

```cmd
del /f /s /q C:\Windows\Logs\DISM\*
```

Then retry the DISM operation without the `/LimitAccess` switch.

### Solution 4: Use a Network Share as Source

If you have another machine with the same Windows build, share its WinSxS folder:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth /Source:\\server\share\WinSxS /LimitAccess
```

### Solution 5: Run SFC First

```cmd
sfc /scannow
```

Then try DISM again.
