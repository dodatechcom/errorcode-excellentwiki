# [Solution] Error 0x800F081F Windows 11/10 — Source Files Not Found Fix

Error 0x800F081F — The source files could not be found — occurs when Windows cannot locate the files needed to enable a Windows feature or install an update. It most commonly appears when enabling .NET Framework 3.5.

## Description

The full message reads:

> "The source files could not be found (0x800F081F)"

This CBS (Component-Based Servicing) error means Windows Update cannot download the required feature files. Common scenarios include:

- **Installing .NET Framework 3.5** — The most common trigger for this error.
- **Enabling optional Windows features** — Turning on features like Hyper-V, IIS, or Telnet Client.
- **Application installation** — Programs that depend on .NET 3.5 fail their prerequisite check.
- **Enterprise environments** — Group Policy blocks direct access to Windows Update.

## Common Causes

1. **Windows Update blocked** — Group Policy or firewall prevents Windows from contacting Microsoft servers.
2. **No internet access** — The system cannot reach Windows Update to download source files.
3. **Corrupted component store** — The WinSxS folder has integrity issues.
4. **Missing installation media** — No local source for feature files is available.
5. **WSUS configuration** — Enterprise WSUS server does not have the feature files.

## Solutions

### Solution 1: Enable Feature Using DISM with Windows Installation Media

Mount a Windows ISO and use it as the source:

```cmd
DISM /Online /Enable-Feature /FeatureName:NetFx3 /All /Source:D:\sources\sxs /LimitAccess
```

Replace `D:` with the drive letter of your mounted ISO. The `/LimitAccess` switch prevents DISM from contacting Windows Update.

### Solution 2: Enable .NET Framework 3.5 via Windows Features

1. Press `Win + R`, type `optionalfeatures`, and press Enter.
2. Check **.NET Framework 3.5 (includes .NET 2.0 and 3.0)**.
3. Click **OK**.
4. Select **Let Windows Update download the files for you**.

### Solution 3: Configure Group Policy for Alternate Sources

```cmd
gpupdate /force
```

Then retry the feature installation.

### Solution 4: Repair the Component Store

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

Restart after both commands complete, then retry the installation.

### Solution 5: Enable via PowerShell

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName NetFx3 -All -Source "D:\sources\sxs" -LimitAccess
```
