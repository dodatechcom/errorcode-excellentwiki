# [Solution] Error 0x800F081F Windows 11/10 — Source Files Not Found Fix

Error 0x800F081F occurs when Windows cannot find the source files required to install a feature, most commonly the .NET Framework 3.5. It affects both Windows 10 and 11 when enabling legacy features.

This error typically blocks installation of .NET Framework 3.5, which many older applications and games require. The fix involves pointing Windows to valid installation media or enabling the feature through alternative methods.

## Description

The full error message typically reads:

> "Windows couldn't complete the requested changes. The changes couldn't be installed. The following error code might help you fix this problem: 0x800f081f — The source files could not be found."

Error 0x800F081F is a CBS (Component-Based Servicing) error that means Windows Update cannot locate the necessary files to enable a Windows feature. By default, Windows downloads these files from Windows Update, but if that fails, you must supply them from local media.

The error appears in these common scenarios:

- **Installing .NET Framework 3.5** — The most common trigger for this error
- **Enabling Windows features** — Turning on optional components like Hyper-V or IIS
- **Application installation** — Programs that depend on .NET 3.5 fail their prerequisites check
- **Group Policy configuration** — Enterprise policies restricting Windows Update access
- **Offline systems** — Computers without internet access cannot download feature files

## Common Causes

1. **No internet access or Windows Update blocked** — Windows cannot download the required source files from Microsoft servers.
2. **Group Policy restrictions** — Enterprise policies limit where Windows can obtain feature installation files.
3. **Corrupted Windows component store** — The WinSxS folder has integrity issues.
4. **WSUS or proxy configuration** — Corporate network routing prevents access to Microsoft download servers.
5. **Missing installation media** — The system lacks a local source for the feature files.
6. **Antivirus interference** — Security software blocks the download of component files.

## Solutions

### Solution 1: Enable .NET Framework 3.5 via DISM with Windows ISO

The most reliable fix uses a Windows installation ISO as the source. Mount the ISO and note its drive letter, then run:

```cmd
DISM /Online /Enable-Feature /FeatureName:NetFx3 /All /Source:D:\sources\sxs /LimitAccess
```

Replace `D:` with the drive letter of your mounted ISO. This tells DISM to pull the .NET 3.5 files directly from the installation media instead of Windows Update.

**To verify the feature is enabled:**

```powershell
Get-WindowsOptionalFeature -Online -FeatureName NetFx3 | Select-Object FeatureName, State
```

### Solution 2: Enable .NET Framework 3.5 via Windows Settings

If you have internet access, try enabling the feature through the GUI:

1. Press `Win + R`, type `optionalfeatures`, and press Enter.
2. Scroll down and check **.NET Framework 3.5 (includes .NET 2.0 and 3.0)**.
3. Click **OK**.
4. Select **Let Windows Update download the files for you**.
5. Wait for the download and installation to complete.

If this fails, proceed to the DISM method above.

### Solution 3: Configure Group Policy for Alternate Sources

Enterprise environments often block direct Windows Update access. Configure Group Policy to allow alternate sources:

1. Press `Win + R`, type `gpedit.msc`, and press Enter.
2. Navigate to **Computer Configuration > Administrative Templates > System**.
3. Double-click **Specify settings for optional component installation and component repair**.
4. Set it to **Enabled**.
5. Check **Contact Windows Update directly to download repair content instead of Windows Server Update Services (WSUS)**.
6. Click **OK** and run `gpupdate /force`.

```cmd
gpupdate /force
```

Then retry the .NET 3.5 installation.

### Solution 4: Repair Windows Component Store

If the component store is corrupted, repair it before attempting feature installation:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

After DISM completes, run System File Checker:

```cmd
sfc /scannow
```

Restart your computer after both commands complete, then retry the .NET 3.5 installation.

### Solution 5: Enable via PowerShell with Source

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName NetFx3 -All -Source "D:\sources\sxs" -LimitAccess
```

Replace `D:` with your mounted ISO drive letter. To download from Windows Update instead of restricting to local source, remove the `-LimitAccess` switch:

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName NetFx3 -All
```

### Solution 6: Download .NET 3.5 Installer Directly

Microsoft provides a standalone installer for .NET Framework 3.5:

1. Visit the [Microsoft .NET download page](https://dotnet.microsoft.com/download/dotnet-framework/net35-sp1).
2. Download the offline installer.
3. Run the installer and follow the prompts.
4. If prompted, allow Windows Update to download required files.
