# [Solution] Error 0x80070043 Windows 11/10 — Network Name Not Found Fix

Error 0x80070043 indicates that Windows cannot resolve the network name when attempting to connect to a remote resource. This DNS or NetBIOS resolution failure prevents your computer from locating the target device on the network.

## Description

The full error message reads: **"The network name cannot be found."** This error typically appears when trying to access a network share, remote computer, or mapped drive using a hostname that cannot be resolved. It differs from error 0x80070035 in that the issue is specifically with name resolution rather than path availability.

## Common Causes

1. Incorrect network name or hostname entered
2. DNS resolution failure on the local network
3. NetBIOS name resolution not working
4. Hosts file containing an outdated entry for the target
5. Target computer is offline or renamed
6. DNS server not responding or misconfigured

## Solutions

### Solution 1: Verify the Network Name and Try IP Address

Confirm the correct name and try alternative connection methods:

1. Verify the exact hostname of the target computer
2. Try connecting using the IP address instead:
   ```
   \\192.168.1.100\ShareName
   ```
3. Check the hosts file for incorrect entries:
   ```
   notepad C:\Windows\System32\drivers\etc\hosts
   ```
4. Remove or update any outdated entries for the target

### Solution 2: Check DNS Configuration

Ensure DNS is properly configured:

1. Open **Command Prompt as Administrator**
2. Flush the DNS cache:
   ```
   ipconfig /flushdns
   ```
3. Test DNS resolution for the target hostname:
   ```
   nslookup TARGET_HOSTNAME
   ```
4. If resolution fails, try using your router's IP as the DNS server:
   ```
   ipconfig /setdns DHCP
   ```

### Solution 3: Ping the Server and Verify Connectivity

Test basic network connectivity:

1. Open **Command Prompt**
2. Ping the target by hostname:
   ```
   ping TARGET_HOSTNAME
   ```
3. If hostname fails, ping by IP:
   ```
   ping TARGET_IP
   ```
4. If IP works but hostname fails, it is a DNS/name resolution issue
5. Restart the **DNS Client** service:
   ```
   net stop dnscache && net start dnscache
   ```
