# [Solution] BSOD 0xc000021a STATUS_SYSTEM_PROCESS_TERMINATED Windows 11/10 — Fixed

Error 0xc000021a is a critical BSOD STOP error with the status code `STATUS_SYSTEM_PROCESS_TERMINATED`. It occurs when critical Windows subsystem processes — specifically `winlogon.exe` (handles user logons) or `csrss.exe` (Client/Server Runtime Subsystem) — terminate unexpectedly. The system halts immediately because these processes are essential for Windows operation.

This error affects both Windows 10 and 11 and often results in a boot loop or a system that cannot start normally.

## Common Causes

- **Corrupted system files** — `winlogon.exe`, `csrss.exe`, or their dependent DLLs are damaged.
- **Failed Windows Update** — An incomplete or corrupted update replaces protected system files.
- **Incompatible drivers** — A recently installed driver conflicts with the Windows subsystem.
- **Registry hive corruption** — The SYSTEM or SOFTWARE registry hives are unreadable.

## How to Fix

### Run SFC from Recovery Environment

If you can boot into Safe Mode:

1. Restart your PC and press `F8` or hold `Shift` while clicking Restart.
2. Go to **Troubleshoot > Advanced options > Startup Settings > Restart**.
3. Press `4` or `F4` for Safe Mode.

```cmd
sfc /scannow
```

**If you cannot boot at all:**

1. Boot from a Windows installation USB or recovery drive.
2. Select **Repair your computer**.
3. Go to **Troubleshoot > Advanced options > Command Prompt**.
4. Run:

```cmd
sfc /scannow /offbootdir=C:\ /offwindir=C:\Windows
```

### Run DISM to Repair the System Image

From the recovery Command Prompt:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

If running from recovery media:

```cmd
DISM /Image:C:\ /Cleanup-Image /RestoreHealth
```

### Disable Driver Signature Enforcement

1. Boot into **Advanced Startup Options** (hold Shift + Restart, or use recovery media).
2. Go to **Troubleshoot > Advanced options > Startup Settings > Restart**.
3. Press `7` or `F7` to select **Disable driver signature enforcement**.
4. Uninstall the problematic driver:

```cmd
pnputil /enum-devices /driver oem*.inf
pnputil /delete-driver oemXX.inf /uninstall
```

### Check and Repair Disk Errors

```cmd
chkdsk C: /f /r /x
```

The `/r` flag locates bad sectors and recovers readable data.

### Rebuild BCD and Repair Boot Records

```cmd
bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd
```

If `bootrec /fixboot` fails with access denied:

```cmd
bootsect /nt60 sys
bootrec /fixboot
```

### Perform a System Restore

1. Boot into **Advanced Startup Options**.
2. Go to **Troubleshoot > Advanced options > System Restore**.
3. Select a restore point dated before the error started.

### Reset Windows as Last Resort

From the recovery environment:

1. Go to **Troubleshoot > Reset this PC**.
2. Choose **Keep my files** to preserve personal data.
3. Choose **Cloud download** for a fresh copy of Windows.

## Examples

This error commonly occurs in these scenarios:

- **After a Windows Update** — A corrupted update replaces critical subsystem files.
- **After installing new software** — Third-party software modifies protected system files.
- **With corrupted boot configuration** — Damaged BCD prevents subsystem processes from loading.
- **After disk corruption** — Bad sectors prevent `winlogon.exe` or `csrss.exe` from loading.
