# [Solution] Error 0x80070091 Windows 11/10 — System Restore Failed Fix

Error 0x80070091 — System Restore failed — occurs when Windows cannot complete a system restore operation. This typically happens due to file or registry key exclusions, permission issues, or disk problems.

## Description

The full message reads:

> "System Restore failed (0x80070091)"

This error appears when System Restore encounters a file or registry key it cannot process. Common scenarios include:

- **System Restore point application** — Rolling back to a previous restore point fails partway through.
- **Automatic restore failures** — Windows fails to apply a restore point during startup repair.
- **Manual restore attempts** — Using System Restore from the GUI returns this error.
- **Safe Mode restore** — Attempting to restore in Safe Mode with limited file access.

## Common Causes

1. **Antivirus interference** — Security software locks files that System Restore needs to modify.
2. **Third-party backup software** — Other backup tools create restore conflicts.
3. **Corrupted restore point** — The restore point data is damaged or incomplete.
4. **Insufficient disk space** — Not enough space to complete the restoration.
5. **File system errors** — Disk corruption prevents file operations.

## Solutions

### Solution 1: Run System Restore in Safe Mode

Boot into Safe Mode and retry the restore:

1. Press `Win + R`, type `msconfig`, and press Enter.
2. Go to the **Boot** tab and check **Safe boot > Minimal**.
3. Click **OK** and restart.
4. In Safe Mode, search for **System Restore** and attempt the restore again.
5. After completion, open `msconfig` again and uncheck Safe boot.

### Solution 2: Disable Antivirus Temporarily

Disable any third-party antivirus or security software before running System Restore. Re-enable it after the restore completes.

### Solution 3: Run Check Disk

```cmd
chkdsk C: /f /r
```

You will be prompted to schedule the scan on next restart. Restart your computer and allow the scan to complete.

### Solution 4: Free Up Disk Space

```cmd
cleanmgr /sageset:1
cleanmgr /sagerun:1
```

Ensure the system drive has sufficient free space before retrying.

### Solution 5: Try a Different Restore Point

If one restore point fails, try an earlier restore point:

1. Open **System Restore**.
2. Click **Choose a different restore point**.
3. Select a restore point from a date before the problematic point.
