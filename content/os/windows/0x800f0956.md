# [Solution] Error 0x800F0956 Windows 11/10 — DISM Source Download Failed Fix

Error 0x800F0956 — DISM source download failed — occurs when DISM cannot download the required source files from Windows Update to repair the component store.

## Description

The full message reads:

> "DISM source download failed (0x800F0956)"

This error indicates that DISM attempted to download repair files from Windows Update but the download failed. Common scenarios include:

- **Running DISM RestoreHealth** — The repair operation cannot download files from Windows Update.
- **Network connectivity issues** — Intermittent internet connection causes partial downloads.
- **Windows Update server problems** — Microsoft servers are unreachable or overloaded.
- **Corporate proxy blocking** — Enterprise network filters block the download.

## Common Causes

1. **Unstable internet connection** — The connection drops during the download.
2. **Windows Update blocked** — Firewall or proxy blocks access to Windows Update servers.
3. **Server-side issues** — Microsoft servers are temporarily unavailable.
4. **Corrupted download cache** — Partial or corrupted download files are cached locally.
5. **Incorrect time or date** — System clock out of sync causes SSL certificate validation failures.

## Solutions

### Solution 1: Use a Local Source Instead of Windows Update

Mount a Windows ISO and use it as the source:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth /Source:D:\sources\install.wim /LimitAccess
```

Replace `D:` with your mounted ISO drive letter.

### Solution 2: Check Internet Connectivity and Proxy

Verify internet connectivity:

```cmd
ping www.microsoft.com
```

If you use a proxy, configure DISM to use it:

```cmd
netsh winhttp set proxy proxy-server="http://your-proxy:port"
```

### Solution 3: Clear DISM Download Cache

Delete any partially downloaded files:

```cmd
del /f /s /q C:\Windows\Logs\DISM\
```

Then retry the operation.

### Solution 4: Sync System Clock

```cmd
w32tm /resync
```

Ensure the date and time are correct, then retry the DISM operation.

### Solution 5: Try a Different Source

If one ISO does not work, try a Windows ISO matching your exact build version, or use a network share with the required files.
