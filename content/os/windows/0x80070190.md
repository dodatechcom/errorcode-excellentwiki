# [Solution] Error 0x80070190 Windows 11/10 — Digital Signature Not Verified Fix

Error 0x80070190 means Windows cannot verify the digital signature of a file or driver. This security-related error occurs when a driver or system file lacks a valid signature from a trusted certificate authority.

## Description

The full error message reads: **"Windows cannot verify the digital signature for this file."** This error commonly appears when loading unsigned or improperly signed drivers, during system boot when critical drivers fail signature verification, or when installing software with an expired or revoked certificate. Windows enforces driver signature verification to protect against malware.

## Common Causes

1. Driver does not have a valid digital signature
2. Digital signature certificate has expired
3. Certificate was revoked by the issuing authority
4. Driver was modified after signing (integrity check failed)
5. Boot Configuration Data pointing to unsigned drivers
6. System clock set to incorrect date affecting certificate validation

## Solutions

### Solution 1: Disable Driver Signature Enforcement Temporarily

Allow unsigned drivers to load (for testing only):

1. Open **Settings** > **Update & Security** > **Recovery**
2. Under **Advanced startup**, click **Restart now**
3. Go to **Troubleshoot** > **Advanced options** > **Startup Settings**
4. Click **Restart**
5. Press **7** or **F7** to select **Disable driver signature enforcement**
6. Windows will boot with signature enforcement disabled
7. Install the necessary driver while enforcement is disabled

### Solution 2: Update Drivers from Official Source

Obtain properly signed drivers:

1. Identify the device causing the error in **Device Manager**
2. Visit the **manufacturer's official website**
3. Download the latest driver for your specific device model and Windows version
4. Install the driver as Administrator
5. Verify the driver is now properly signed in Device Manager properties

### Solution 3: Check System Date and Verify Certificates

Ensure your system clock is correct:

1. Right-click the system clock in the taskbar and select **Adjust date/time**
2. Enable **Set time automatically**
3. Click **Sync now** to synchronize with the time server
4. If the issue persists with a specific driver, check its certificate:
   - Right-click the driver file > **Properties** > **Digital Signatures** tab
   - Select the signature and click **Details**
   - Verify the certificate chain is valid and not expired
