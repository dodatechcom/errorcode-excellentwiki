# [Solution] Error 0x8024a105 Windows 11/10 — Post-Reboot Still Pending Fix

Error 0x8024a105 is a Windows Update error code `WU_E_UH_POSTREBOOTSTILLPENDING`. It occurs when an update that was supposed to complete after a system restart is still pending — the reboot didn't finalize the installation as expected.

This error affects both Windows 10 and 11 and typically appears when Windows Update attempts to install an update but finds that a previous update is still stuck in a pending-reboot state.

## Description

The full error message typically reads:

> "WU_E_UH_POSTREBOOTSTILLPENDING — The update could not be installed because a restart is still pending."

Or in Windows Update history:

> "A quality update is available — Error 0x8024a105"

Error 0x8024a105 means the Windows Update agent detected that a previous update operation has a pending reboot that hasn't been completed. Windows cannot proceed with new updates until the pending operation finishes. This often happens when:

- The user clicked "Restart" but the system didn't fully process the pending updates during boot
- A power loss or forced shutdown interrupted the post-reboot update processing
- A service that handles post-reboot operations is stuck or crashed

Common scenarios include:

- **Cumulative updates** stuck in a pending-reboot loop
- **Feature updates** unable to proceed because previous updates are pending
- **Driver updates** blocked by incomplete prior installations
- **Servicing stack updates** that require a clean reboot cycle

## Common Causes

1. **Incomplete reboot processing** — The system didn't fully apply pending updates during restart.
2. **Pending.xml file stuck** — The Windows servicing stack's pending operations file hasn't been cleared.
3. **Forced shutdown during update** — Power loss or holding the power button during update processing.
4. **Corrupted servicing stack** — The Windows servicing components are damaged.
5. **Windows Update services hung** — The TrustedInstaller or Windows Module Installer service is stuck.
6. **Registry pending entries** — The registry contains stale pending update entries.

## Solutions

### Solution 1: Restart Your PC Properly

The simplest fix is performing a complete restart (not shutdown + power on).

1. Save all open work.
2. Open the **Start menu**.
3. Click **Power > Restart**.
4. Wait for Windows to fully boot and process any pending operations.

**Using command line for a clean restart:**

```cmd
shutdown /r /t 0 /f
```

The `/f` flag forces all applications to close, ensuring a clean restart.

**Using PowerShell:**

```powershell
Restart-Computer -Force
```

### Solution 2: Clear Pending Update Operations

If a proper restart didn't fix it, manually clear the pending operations.

**Stop Windows Update services:**

```cmd
net stop wuauserv
net stop trustedinstaller
net stop bits
net stop cryptsvc
```

**Delete the pending.xml file:**

```cmd
del /f /q "C:\Windows\WinSxS\pending.xml"
del /f /q "C:\Windows\WinSxS\poqexec.log"
```

**If files are locked, use this approach:**

```cmd
takeown /f "C:\Windows\WinSxS\pending.xml" /a
icacls "C:\Windows\WinSxS\pending.xml" /grant administrators:F
del /f /q "C:\Windows\WinSxS\pending.xml"
```

**Restart services:**

```cmd
net start wuauserv
net start bits
net start cryptsvc
```

### Solution 3: Use DISM to Clear Pending Actions

DISM can directly manage pending operations:

```cmd
DISM /Online /Cleanup-Image /RevertPendingActions
```

This command rolls back any pending servicing operations and clears the pending state.

**Follow up with a full system repair:**

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

### Solution 4: Clear the SoftwareDistribution Cache

Corrupted download cache can maintain false pending states.

```cmd
net stop wuauserv
net stop bits
```

Clear the cache contents:

```cmd
del /f /q /s "C:\Windows\SoftwareDistribution\Download\*.*"
del /f /q /s "C:\Windows\SoftwareDistribution\DataStore\*.*"
```

Restart services:

```cmd
net start wuauserv
net start bits
```

Force a fresh update check:

```powershell
UsoClient StartScan
```

### Solution 5: Reset Registry Pending Entries

Stale registry entries can maintain false pending states.

**Export the registry first:**

```cmd
reg export "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing" C:\CBS_Backup.reg
```

**Check for pending operations in the registry:**

```cmd
reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\Pending" /s
```

**Remove pending package entries (carefully):**

```cmd
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\PackagesPending" /f
```

**Warning:** Only modify registry entries if you're comfortable doing so. Incorrect changes can cause system instability.

### Solution 6: Run the Windows Update Troubleshooter

The troubleshooter can detect and clear stuck pending operations.

```powershell
msdt.exe /id WindowsUpdateDiagnostic
```

Through Settings:

1. Open **Settings > System > Troubleshoot > Other troubleshooters**.
2. Click **Run** next to **Windows Update**.
3. Apply all recommended fixes.
4. Restart your PC.
