# [Solution] Error 0x80070020 Windows 11/10 — File in Use Fix

Error 0x80070020 occurs when a file or resource you are trying to access is locked by another running process. Windows prevents concurrent access to protect file integrity, and this error means another program has exclusive control over the file.

## Description

The full error message reads: **"The process cannot access the file because it is being used by another process."** This error commonly appears when trying to delete, move, rename, or overwrite a file that is currently open in another application. It frequently occurs during file operations, software installations, or system updates.

## Common Causes

1. The target file is open in another application
2. A background process has locked the file
3. Antivirus software is scanning the file in real-time
4. Windows Explorer has the file locked
5. A previous operation on the file did not release the file handle
6. System processes accessing critical files during updates

## Solutions

### Solution 1: Close Programs Using the File

Identify and close the program that has the file open:

1. Try closing all open applications that might use the file
2. Check if the file is open in any document viewer or editor
3. If a specific application is listed in the error, close that application
4. Use **Resource Monitor** to identify which process holds the lock (see Solution 2)

### Solution 2: Use Resource Monitor to Find the Locking Process

Locate the specific process holding the file:

1. Press **Ctrl + Shift + Esc** to open Task Manager
2. Click **Performance** tab and then **Open Resource Monitor** at the bottom
3. Go to the **CPU** tab
4. In the **Associated Handles** section, search for the file name
5. Identify the process holding the handle
6. Right-click the process and select **End Process** or **End Process Tree**
7. Retry the file operation

### Solution 3: Restart Windows Explorer

Release Explorer-locked files by restarting the shell:

1. Press **Ctrl + Shift + Esc** to open Task Manager
2. Find **Windows Explorer** in the Processes tab
3. Right-click it and select **Restart**
4. Alternatively, run this in Command Prompt:
   ```
   taskkill /f /im explorer.exe && start explorer.exe
   ```
5. Retry the file operation after Explorer restarts
