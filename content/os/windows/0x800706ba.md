# [Solution] Error 0x800706BA Windows 11/10 — RPC Server Unavailable Fix

Error 0x800706BA — The RPC server is unavailable — occurs when a client cannot connect to the Remote Procedure Call service on a target machine. This is one of the most common network-related COM errors.

## Description

The full message reads:

> "The RPC server is unavailable (0x800706BA)"

This error means the RPC endpoint mapper on the remote (or local) machine is not reachable. Common scenarios include:

- **Network file sharing** — Accessing shared folders or printers on another computer.
- **Remote administration** — Using MMC, WMI, or PowerShell remoting to manage remote systems.
- **DCOM applications** — Distributed COM components cannot communicate across machines.
- **Windows Update** — The update client cannot reach the update server via RPC.

## Common Causes

1. **RPC service not running** — The Remote Procedure Call service is stopped or disabled.
2. **Network connectivity problems** — The target machine is offline or unreachable.
3. **Firewall blocking RPC** — Windows Firewall or third-party firewall blocks TCP port 135.
4. **DNS resolution failure** — The target hostname cannot be resolved to an IP address.
5. **RPC port range blocked** — Dynamic RPC ports (49152–65535) are blocked by firewall.
6. **Target machine RPC service stopped** — The RPC service on the remote computer is not running.

## Solutions

### Solution 1: Check RPC Service on Local Machine

Verify the RPC service is running on your machine:

```cmd
sc query RpcSs
```

If stopped, start it:

```cmd
net start RpcSs
```

### Solution 2: Check Network Connectivity

Test basic connectivity to the target machine:

```cmd
ping <target-computer-name>
```

If ping fails, check that the target computer is powered on and connected to the network. Try using the IP address instead of the hostname to rule out DNS issues.

### Solution 3: Check RPC Service on the Remote Machine

If you have access to the remote machine, verify the RPC service is running there as well. The RPC service must be running on both client and server.

### Solution 4: Configure Firewall to Allow RPC

Allow RPC traffic through Windows Defender Firewall:

1. Press `Win + R`, type `wf.msc`, and press Enter.
2. Click **Inbound Rules**.
3. Find and enable **Remote Procedure Call (RPC)** and **Remote Event Log Management**.
4. If these do not exist, add a new rule:
   - Rule Type: **Port**
   - Protocol: **TCP**
   - Port: **135**
   - Action: **Allow the connection**
   - Profile: **Domain, Private**
   - Name: **RPC Endpoint Mapper**

### Solution 5: Enable RPC Dynamic Port Range

RPC uses dynamic ports for communication. Ensure the dynamic port range is open:

```cmd
netsh int ipv4 show dynamicport tcp
```

Windows default dynamic port range is 49152–65535. If the range has been modified, reset it:

```cmd
netsh int ipv4 set dynamicport tcp start=49152 num=16384
```

### Solution 6: Flush DNS and Reset Network Stack

```cmd
ipconfig /flushdns
ipconfig /registerdns
netsh int ip reset
```

Restart your computer after running these commands.
