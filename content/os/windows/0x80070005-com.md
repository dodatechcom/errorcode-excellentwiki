# [Solution] Error 0x80070005 COM Access Denied Windows 11/10 — COM/DCOM Fix

Error 0x80070005 — Access Denied — occurs in COM/DCOM scenarios when a component lacks permission to launch, activate, or access a COM object. This is the COM-specific variant of the access denied error.

## Description

The full message reads:

> "Access Denied (0x80070005)"

In the COM context, this error means the caller does not have sufficient DCOM launch or activation permissions. Common scenarios include:

- **DCOM application launch** — A client process cannot launch a COM server on the local or remote machine.
- **COM+ activation** — The COM+ runtime denies activation due to insufficient permissions.
- **Component Services console** — Access denied when modifying COM+ applications.
- **Cross-user COM calls** — A service running under one account cannot access a COM object configured for another account.

## Common Causes

1. **Insufficient DCOM launch permissions** — The calling user lacks the **Remote Launch** or **Remote Activation** permission.
2. **Incorrect DCOM security settings** — The machine-wide or application-specific DCOM security limits are misconfigured.
3. **Interactive Window Station restriction** — COM objects configured to run in an interactive window station fail when called from a service.
4. **User account mismatch** — The COM server runs under an account different from the client, with no delegation trust.
5. **Corrupted COM security registry** — Registry keys for COM security are damaged.

## Solutions

### Solution 1: Adjust DCOM Launch and Activation Permissions

1. Press `Win + R`, type `dcomcnfg`, and press Enter.
2. Expand **Component Services > Computers > My Computer**.
3. Right-click **My Computer** and select **Properties**.
4. Go to the **COM Security** tab.
5. Under **Launch and Activation Permissions**, click **Edit Limits**.
6. Add the user or group experiencing the error.
7. Grant **Local Launch**, **Remote Launch**, **Local Activation**, and **Remote Activation** permissions.
8. Click **OK** and restart the failing application.

### Solution 2: Configure Application-Specific DCOM Permissions

1. Open **Component Services**.
2. Navigate to **Computers > My Computer > DCOM Config**.
3. Find the specific COM application causing the error.
4. Right-click and select **Properties**.
5. Go to the **Security** tab.
6. Configure **Launch and Activation Permissions** to **Customize**.
7. Add the required users with appropriate permissions.

### Solution 3: Launch in Interactive Window Station

If a COM object is running as a service and requires UI interaction:

1. Open **Component Services > DCOM Config**.
2. Right-click the COM application and select **Properties**.
3. Go to the **Identity** tab.
4. Select **Interactive user** instead of **This user** or **Launching user**.

### Solution 4: Reset COM Security to Defaults

Reset COM security settings to their defaults:

```cmd
reg delete HKLM\SOFTWARE\Microsoft\Ole /v DefaultLaunchPermission /f
reg delete HKLM\SOFTWARE\Microsoft\Ole /v MachineLaunchRestriction /f
reg delete HKLM\SOFTWARE\Microsoft\Ole /v MachineAccessRestriction /f
```

Restart your computer after running these commands.

### Solution 5: Re-register the COM Component

```cmd
regsvr32 "C:\Path\To\Component.dll"
```
