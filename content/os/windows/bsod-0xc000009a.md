# [Solution] BSOD 0xc000009a STATUS_INSUFFICIENT_RESOURCES Windows 11/10 — Fixed

Error 0xc000009a is a critical BSOD STOP error with the status code `STATUS_INSUFFICIENT_RESOURCES`. It occurs when Windows or a driver cannot allocate the system resources it needs — typically kernel pool memory, nonpaged pool, or system PTEs (Page Table Entries). The kernel halts because it cannot continue operating without these essential resources.

This error affects both Windows 10 and 11 and is commonly caused by memory leaks in drivers, excessive system resource consumption, or faulty RAM.

## Common Causes

- **Driver memory leaks** — A driver allocates kernel pool memory but never frees it, eventually exhausting all available resources.
- **Too many open handles** — Applications or drivers hold excessive file, registry, or object handles.
- **Faulty RAM** — Physical memory errors reduce the available pool and cause allocation failures.
- **Excessive nonpaged pool usage** — Drivers that must remain in physical memory consume all available nonpaged pool.

## How to Fix

### Identify the Resource-Hungry Driver

**Check nonpaged pool usage:**

```powershell
Get-Process | Sort-Object WorkingSet64 -Descending | Select-Object -First 20 Name, WorkingSet64, HandleCount | Format-Table -AutoSize
```

**Use Task Manager to find memory-consuming processes:**

1. Open **Task Manager** (`Ctrl + Shift + Esc`).
2. Go to the **Processes** tab.
3. Sort by **Memory** to find processes consuming excessive RAM.

**Check pool usage with poolmon (from Windows SDK):**

```cmd
poolmon -b
```

This sorts by bytes and shows which pool tags are consuming the most memory. The tag with the highest byte count is the likely leak source.

### Run Driver Verifier

Driver Verifier monitors drivers for memory leaks:

```cmd
verifier /standard /all
```

Restart your computer. When a driver leaks memory, Driver Verifier will flag it. Reset when done:

```cmd
verifier /reset
```

### Restart Memory-Consuming Applications

1. Open **Task Manager** (`Ctrl + Shift + Esc`).
2. Sort by **Memory**.
3. Right-click applications with excessive memory usage and select **End task**.
4. Restart the application to clear the leak.

### Check RAM for Errors

```cmd
mdsched.exe
```

Select **Restart now and check for problems**. Use MemTest86 from a bootable USB for thorough testing. Faulty RAM can reduce available resources and cause allocation failures.

### Update Drivers

Update all drivers, especially those recently installed:

```powershell
Get-WmiObject Win32_PnPSignedDriver | Sort-Object DriverDate -Descending | Select-Object -First 20 DeviceName, DriverVersion, DriverDate | Format-Table -AutoSize
```

Download the latest drivers from the manufacturer's website.

### Increase Available Memory

If the system is consistently running low on resources:

1. Close unnecessary startup programs:
   - Open **Task Manager** > **Startup** tab.
   - Right-click unnecessary items and select **Disable**.

2. Reduce the number of running services:
   - Open **Services** (`Win + R` > `services.msc`).
   - Set unnecessary services to **Disabled** or **Manual**.

### Repair System Files

```cmd
sfc /scannow
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

## Examples

This error commonly occurs in these scenarios:

- **After running the system for days without rebooting** — A driver's memory leak gradually consumes all available resources.
- **When running many applications simultaneously** — Excessive resource consumption exhausts the kernel pool.
- **With a buggy network or antivirus driver** — The driver allocates memory for packet processing but never frees it.
- **On systems with limited RAM** — Insufficient physical memory for the workload causes resource exhaustion.
