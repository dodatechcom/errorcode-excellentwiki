# [Solution] Error 0x800F0954 Windows 11/10 — DISM Source Not Found Fix

Error 0x800F0954 — DISM source not found — occurs when DISM cannot locate the required source files for image repair or feature installation, often due to a misconfigured Group Policy blocking Windows Update access.

## Description

The full message reads:

> "DISM source not found (0x800F0954)"

This error is common in enterprise environments where Windows Update is redirected through WSUS. Common scenarios include:

- **Running DISM in a corporate environment** — WSUS blocks direct access to Microsoft servers.
- **Offline systems** — The computer has no internet access for downloading source files.
- **Misconfigured Group Policy** — Settings prevent DISM from contacting Windows Update.
- **Missing installation media** — No local WinSxS or ISO source is available.

## Common Causes

1. **WSUS restrictions** — Group Policy forces DISM to use WSUS, which does not have the required files.
2. **No internet access** — The machine cannot reach Windows Update servers.
3. **Corrupted local cache** — The local component store is damaged.
4. **Missing Windows installation media** — No ISO or DVD is available for source files.
5. **Registry misconfiguration** — The Source policy registry key is set incorrectly.

## Solutions

### Solution 1: Use Local Installation Media

Mount a Windows ISO and specify it as the source:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth /Source:D:\sources\install.wim /LimitAccess
```

Replace `D:` with your mounted ISO drive letter.

### Solution 2: Modify Group Policy to Allow Windows Update

1. Press `Win + R`, type `gpedit.msc`, and press Enter.
2. Navigate to **Computer Configuration > Administrative Templates > System**.
3. Open **Specify settings for optional component installation and component repair**.
4. Set it to **Enabled**.
5. Check **Contact Windows Update directly to download repair content instead of Windows Server Update Services (WSUS)**.
6. Click **OK** and run:

```cmd
gpupdate /force
```

### Solution 3: Use Windows Update as Source with LimitAccess

If Group Policy cannot be changed, specify an alternate source file:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
```

### Solution 4: Repair Component Store with SFC

```cmd
sfc /scannow
```

Then retry the DISM operation.
