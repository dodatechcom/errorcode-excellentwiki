# [Solution] Error 0x800F0922 Windows 11/10 — Installation Failed Fix

Error 0x800F0922 — The installation failed — occurs when DISM or Windows Update cannot complete a feature installation. This is commonly caused by insufficient disk space, VPN interference, or a corrupted servicing stack.

## Description

The full message reads:

> "The installation failed (0x800F0922)"

This CBS error appears when the component-based servicing stack encounters a blocking condition. Common scenarios include:

- **Windows feature installation** — Adding or removing Windows features fails partway through.
- **Windows Update** — Cumulative or feature updates fail to install.
- **DISM operations** — Servicing commands return this error code.
- **.NET Framework installation** — Enabling legacy frameworks stops unexpectedly.

## Common Causes

1. **Insufficient disk space** — The system drive does not have enough free space for the installation.
2. **VPN or proxy interference** — Network filters block the servicing stack from downloading files.
3. **Corrupted servicing store** — The WinSxS component store has integrity issues.
4. **Outdated servicing stack** — The CBS servicing stack update is missing.
5. **Third-party antivirus blocking** — Security software interferes with the installation process.

## Solutions

### Solution 1: Check and Free Disk Space

Ensure the system drive has at least 20 GB of free space:

```cmd
wmic logicaldisk where caption="C:" get FreeSpace /value
```

If space is low, run Disk Cleanup:

```cmd
cleanmgr /sageset:1
cleanmgr /sagerun:1
```

### Solution 2: Disable VPN and Proxy

Temporarily disable any VPN client or proxy software, then retry the installation. Corporate networks with WSUS may also cause this error.

### Solution 3: Run DISM to Repair the Component Store

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

Restart your computer after the scans complete.

### Solution 4: Install the Latest Servicing Stack Update

Download and install the latest servicing stack update (SSU) for your Windows version from the Microsoft Update Catalog.

### Solution 5: Temporarily Disable Antivirus

Disable third-party antivirus software temporarily and retry the installation. Re-enable it immediately after the installation completes.
