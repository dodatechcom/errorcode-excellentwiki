# [Solution] Error 0x0000007e Windows 11/10 — SYSTEM_THREAD_EXCEPTION_NOT_HANDLED Fix

Error 0x0000007e is a BSOD STOP error with the bug check `SYSTEM_THREAD_EXCEPTION_NOT_HANDLED`. It occurs when a system thread generates an exception that the error handler doesn't catch, most commonly due to faulty or incompatible drivers.

This error affects both Windows 10 and 11 and typically points to a specific driver file that caused the crash, which is usually named in the blue screen message.

## Description

The full error message typically reads:

> "Your PC ran into a problem and needs to restart. STOP: 0x0000007E — SYSTEM_THREAD_EXCEPTION_NOT_HANDLED"

The BSOD may also reference a specific file, such as:

> "What failed: nvlddmkm.sys" (NVIDIA driver) or "igdkmd64.sys" (Intel graphics)

Error 0x0000007e means a system thread (a kernel-mode thread) attempted to execute an instruction that generated an exception, and the system's exception handler could not resolve it. The system halts to prevent further corruption.

Common scenarios include:

- **After a driver update** — The new driver version contains a bug or is incompatible
- **After connecting new hardware** — The system driver for the hardware conflicts with existing components
- **During gaming or GPU-intensive tasks** — Graphics driver instability under load
- **After a Windows Update** — A cumulative update modified driver behavior

## Common Causes

1. **Faulty or incompatible device drivers** — The most common cause, especially GPU, network, and storage drivers.
2. **Corrupted system files** — Windows kernel or HAL files are damaged.
3. **Faulty RAM** — Memory errors cause unpredictable system thread behavior.
4. **Hard disk errors** — Corrupted sectors prevent system files from loading correctly.
5. **Incompatible third-party software** — Kernel-mode software (antivirus, virtualization) conflicting with Windows.

## Solutions

### Solution 1: Boot into Safe Mode

Safe Mode loads Windows with minimal drivers, which helps identify if a driver is causing the crash.

1. Restart your PC and press `F8` or hold `Shift` while clicking Restart.
2. Go to **Troubleshoot > Advanced options > Startup Settings > Restart**.
3. Press `4` or `F4` for **Safe Mode** or `5` / `F5` for **Safe Mode with Networking**.
4. If the system boots successfully in Safe Mode, the issue is likely a third-party driver or software.

To identify the problematic driver from Safe Mode:

```powershell
Get-WmiObject Win32_PnPEntity | Where-Object { $_.ConfigManagerErrorCode -ne 0 } | Select-Object Name, DeviceID, ConfigManagerErrorCode | Format-Table
```

### Solution 2: Update or Roll Back Graphics Drivers

GPU drivers are the most frequent cause of this error. If you recently updated your graphics driver:

**Roll back the driver:**

1. Press `Win + X` and select **Device Manager**.
2. Expand **Display adapters**.
3. Right-click your GPU and select **Properties**.
4. Go to the **Driver** tab and click **Roll Back Driver**.
5. Select a reason and confirm.

**Or update to the latest stable driver:**

- **NVIDIA**: Visit [nvidia.com/drivers](https://www.nvidia.com/drivers) and download the latest Game Ready or Studio driver.
- **AMD**: Visit [amd.com/support](https://www.amd.com/support) and download the latest Adrenalin driver.
- **Intel**: Use [Intel Driver & Support Assistant](https://www.intel.com/content/www/us/en/support/detect.html).

Using Device Manager from Safe Mode:

```powershell
pnputil /scan-devices
```

### Solution 3: Check RAM for Errors

Faulty memory can cause system threads to crash unpredictably.

**Using Windows Memory Diagnostic:**

1. Press `Win + R`, type `mdsched.exe`, press Enter.
2. Select **Restart now and check for problems**.
3. The system will reboot and run a memory test.
4. Results appear in the Event Viewer after logging in.

```powershell
Get-WinEvent -LogName System | Where-Object { $_.ProviderName -eq "Microsoft-Windows-MemoryDiagnostics-Results" } | Select-Object -First 5 Message
```

**Using a bootable MemTest86 USB for more thorough testing:**

1. Download MemTest86 from [memtest86.com](https://www.memtest86.com/).
2. Create a bootable USB drive.
3. Boot from the USB and let it run for at least 4 passes.
4. Any errors indicate faulty RAM that should be replaced.

### Solution 4: Run System File Checker

Corrupted Windows system files can cause this exception. From Safe Mode or recovery:

```cmd
sfc /scannow
```

If SFC reports it cannot fix files, run DISM first:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

### Solution 5: Check Disk Health

Disk errors can corrupt the system files that system threads depend on:

```cmd
chkdsk C: /f /r
```

Type `Y` to schedule the check on next restart, then reboot.

For more detailed disk health information:

```powershell
Get-PhysicalDisk | Select-Object FriendlyName, HealthStatus, OperationalStatus
Get-StorageReliabilityCounter -PhysicalDisk (Get-PhysicalDisk) | Select-Object Temperature, Wear, ReadErrorsTotal, WriteErrorsTotal
```

### Solution 6: Uninstall Recently Installed Software

If the error started after installing new software, particularly antivirus, virtualization tools, or system utilities:

1. Boot into Safe Mode.
2. Open **Settings > Apps > Installed apps**.
3. Sort by **Install date** and uninstall recently added programs.
4. Restart and test.

Alternatively, from Safe Mode Command Prompt:

```powershell
Get-WmiObject Win32_Product | Sort-Object InstallDate -Descending | Select-Object -First 10 Name, Version, InstallDate
```

### Solution 7: Check the Event Viewer for the Faulting Module

The Event Viewer often records the specific module that caused the crash:

1. Press `Win + X` and select **Event Viewer**.
2. Go to **Windows Logs > System**.
3. Look for **Critical** or **Error** events around the time of the BSOD.
4. The event description often includes the faulting driver name.

```powershell
Get-WinEvent -LogName System -MaxEvents 50 | Where-Object { $_.Level -le 2 } | Select-Object TimeCreated, Id, Message | Format-List
```
