TS2762 — JSX element ‘X’ has no corresponding closing tag

TS2762 occurs when a JSX element is opened but never closed. JSX requires all elements to either be self-closing (<Component />) or have a matching closing tag (</Component>).

Common Causes

// Cause 1: Missing closing tag
function App() {
  return <div>Hello; // TS2762: no closing </div>
}

// Cause 2: Missing self-closing slash
function App() {
  return <img src="photo.jpg">; // TS2762: should be <img ... />
}

// Cause 3: Mismatched tags
function App() {
  return (
    <div>
      <span>Hello</div> // TS2762: </span> missing
    </div>
  );
}

// Cause 4: Unclosed component
function App() {
  return <MyComponent props={value}; // TS2762: missing /> or </MyComponent>
}

Solutions

Fix 1: Add the closing tag

// Wrong
return <div>Hello;

// Correct
return <div>Hello</div>;

Fix 2: Use self-closing syntax for void elements

// Wrong
return <img src="photo.jpg">;

// Correct
return <img src="photo.jpg" />;

Fix 3: Ensure all nested tags are properly closed

// Wrong
return (
  <div>
    <span>Hello</div>
  </div>
);

// Correct
return (
  <div>
    <span>Hello</span>
  </div>
);

Fix 4: Close custom components properly

// Wrong
return <MyComponent props={value}>;

// Correct — self-closing
return <MyComponent props={value} />;

// Or with children
return (
  <MyComponent props={value}>
    <p>Child content</p>
  </MyComponent>
);

Examples

// Example 1: Conditional rendering without closing
function App({ show }: { show: boolean }) {
  return (
    <div>
      {show && <Alert message="Warning" // TS2762
    }
  </div>
  );
}

// Fix:
return (
  <div>
    {show && <Alert message="Warning" />}
  </div>
);

// Example 2: Fragment without closing
function App() {
  return (
    <>
      <p>First</p>
      <p>Second</p>
    // TS2762: missing </>
  );
}
