TS2554 — Expected X arguments, but got Y

TS2554 occurs when you call a function with a different number of arguments than its signature expects. Unlike JavaScript, TypeScript enforces exact argument counts based on the function’s type declaration.

Common Causes

// Cause 1: Too few arguments
function greet(name: string, greeting: string) {}
greet("Alice"); // TS2554: Expected 2 arguments, but got 1

// Cause 2: Too many arguments
function add(a: number, b: number) {}
add(1, 2, 3); // TS2554: Expected 2 arguments, but got 3

// Cause 3: Calling without required arguments
function fetchData(url: string, callback: Function) {}
fetchData(); // TS2554: Expected 2 arguments, but got 0

// Cause 4: Spread operator passing wrong count
function log(a: string, b: string) {}
const args = ["hello"];
log(...args); // TS2554: Expected 2 arguments, but got 1

Solutions

Fix 1: Provide the correct number of arguments

// Wrong
function greet(name: string, greeting: string) {}
greet("Alice");

// Correct
greet("Alice", "Hello");

Fix 2: Make parameters optional if they aren’t always needed

// Wrong — greeting should be optional
function greet(name: string, greeting: string) {}

// Correct
function greet(name: string, greeting?: string) {
  const g = greeting ?? "Hello";
  return `${g}, ${name}!`;
}
greet("Alice"); // OK
greet("Alice", "Hi"); // OK

Fix 3: Use rest parameters for variable argument count

// Wrong — fixed 2 args
function log(a: string, b: string) {}

// Correct — variable args
function log(...args: string[]) {}
log("hello"); // OK
log("hello", "world"); // OK

Fix 4: Remove extra arguments

// Wrong
add(1, 2, 3);

// Correct
add(1, 2);

Examples

// Example 1: Callback with wrong arity
function onComplete(success: boolean, message: string) {}
onComplete(true); // TS2554: Missing second argument

// Example 2: Spread with wrong count
const coords = [10, 20, 30] as const;
function point2D(x: number, y: number) {}
point2D(...coords); // TS2554: Expected 2, but got 3
