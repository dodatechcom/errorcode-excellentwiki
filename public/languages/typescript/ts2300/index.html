TS2300 — Duplicate identifier

TS2300 occurs when the same identifier is declared more than once in the same scope. TypeScript doesn’t allow redeclaring a variable, parameter, or type name that already exists in the current block or file.

Common Causes

// Cause 1: Redeclaring a variable in the same scope
let x = 10;
let x = 20; // TS2300: Duplicate identifier 'x'

// Cause 2: Parameter name conflicts with variable
let name = "Alice";
function greet(name: string) {} // TS2304 if 'name' is in outer scope with let/const

// Cause 3: Duplicate interface declarations
interface Config {
  host: string;
}
interface Config { // TS2300: Duplicate identifier
  port: number;
}

// Cause 4: Importing the same name from different modules
import { helper } from "./a";
import { helper } from "./b"; // TS2304 if both export 'helper'

Solutions

Fix 1: Rename the duplicate identifier

// Wrong
let x = 10;
let x = 20;

// Correct
let x = 10;
let x2 = 20;

Fix 2: Use block scoping to avoid conflicts

let name = "Alice";

function greet() {
  let name = "Bob"; // OK — different scope
  console.log(name);
}

Fix 3: Merge interface declarations

// Wrong
interface Config {
  host: string;
}
interface Config {
  port: number;
}

// Correct — merge into one
interface Config {
  host: string;
  port: number;
}

Fix 4: Rename imports to avoid conflicts

import { helper as helperA } from "./a";
import { helper as helperB } from "./b";

Examples

// Example 1: Redeclaring in for loop
for (let i = 0; i < 5; i++) {}
for (let i = 0; i < 5; i++) {} // TS2300: 'i' already declared

// Fix: use different variable names
for (let i = 0; i < 5; i++) {}
for (let j = 0; j < 5; j++) {}

// Example 2: Conflicting class and namespace
class MyClass {}
namespace MyClass {} // TS2300 in some configurations
