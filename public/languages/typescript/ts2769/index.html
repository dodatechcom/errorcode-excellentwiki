TS2769 — No overload matches this call

TS2769 occurs when a function has multiple overload signatures but none of them match the arguments you provided. TypeScript checks each overload and if none are compatible, it reports this error.

Common Causes

// Cause 1: Arguments don't match any overload
function process(value: string): void;
function process(value: number): void;
function process(value: boolean): void;
function process(value: any) {}

process({ key: "value" }); // TS2769: No overload matches this call

// Cause 2: Wrong argument types in overloaded function
function createElement(tag: "div"): HTMLDivElement;
function createElement(tag: "span"): HTMLSpanElement;
function createElement(tag: string): HTMLElement;
function createElement(tag: string) {}

createElement("canvas"); // TS2769: 'canvas' doesn't match specific overloads

// Cause 3: Missing arguments in some overloads
function fetch(url: string): Promise<Response>;
function fetch(url: string, init: RequestInit): Promise<Response>;
function fetch(url: string, init?: RequestInit) {}

fetch(); // TS2769: No overload matches

// Cause 4: Type mismatch with generic constraints
function merge<T extends object>(a: T, b: T): T & T {
  return { ...a, ...b };
}
merge("a", "b"); // TS2769: string is not an object

Solutions

Fix 1: Match one of the defined overload signatures

// Wrong
function process(value: string): void;
function process(value: number): void;
function process(value: any) {}
process({ key: "value" });

// Correct — use a valid type
process("hello"); // matches string overload
process(42); // matches number overload

Fix 2: Add a catch-all overload at the end

// Add an overload that accepts any type
function createElement(tag: "div"): HTMLDivElement;
function createElement(tag: "span"): HTMLSpanElement;
function createElement(tag: string): HTMLElement;
function createElement(tag: string) {} // catch-all

Fix 3: Use type assertions to satisfy an overload

const input = document.getElementById("canvas") as HTMLCanvasElement;

Fix 4: Restructure overloads to cover more cases

// Instead of specific overloads, use union types
function process(value: string | number | boolean) {}

Examples

// Example 1: React setState overload mismatch
interface State {
  count: number;
}
class Component {
  setState(state: Partial<State>) {}
}
this.setState({ count: "5" }); // TS2769

// Example 2: Array.reduce with wrong initial value
const nums = [1, 2, 3];
nums.reduce((acc, val) => acc + val); // TS2769 in some TS versions
