TS2551 — Property ‘X’ does not exist (did you mean…)

TS2551 is a variant of TS2339 where TypeScript detects that the property doesn’t exist but finds a similarly-named property on the type. It usually includes a “did you mean…” suggestion, making it easier to fix typos.

Common Causes

// Cause 1: Typo in property name
interface User {
  name: string;
  age: number;
}
const user: User = { name: "Alice", age: 30 };
console.log(user.nme); // TS2551: Property 'nme' does not exist. Did you mean 'name'?

// Cause 2: Typo in method name
const arr = [1, 2, 3];
arr.filer(x => x > 1); // TS2551: Did you mean 'filter'?

// Cause 3: Wrong casing (camelCase vs PascalCase)
interface HttpResponse {
  statusCode: number;
}
const res: HttpResponse = { statusCode: 200 };
console.log(res.StatusCode); // TS2551: Did you mean 'statusCode'?

// Cause 4: Accessing property on wrong object
interface Car { make: string; }
interface Truck { bed: boolean; }
function processVehicle(v: Car) {
  console.log(v.bed); // TS2551: Property 'bed' does not exist on type 'Car'
}

Solutions

Fix 1: Use the suggested property name

// Wrong
console.log(user.nme);

// Correct — follow the "did you mean" suggestion
console.log(user.name);

Fix 2: Check casing conventions

// Wrong — PascalCase
console.log(res.StatusCode);

// Correct — camelCase
console.log(res.statusCode);

Fix 3: Verify you’re using the right object type

interface Car { make: string; }
interface Truck { bed: boolean; }

// If you need truck properties, accept Truck type
function processVehicle(v: Car | Truck) {
  if ("bed" in v) {
    console.log(v.bed); // OK — v is Truck
  }
}

Fix 4: Use IDE autocompletion to discover correct names

Most modern IDEs show available properties as you type, preventing these typos before compilation.

Examples

// Example 1: Misspelled DOM method
const canvas = document.getElementById("canvas") as HTMLCanvasElement;
const ctx = canvas.getContex("2d"); // TS2551: Did you mean 'getContext'?

// Example 2: Wrong property on array
const nums = [1, 2, 3];
const result = nums.includs(2); // TS2551: Did you mean 'includes'?
