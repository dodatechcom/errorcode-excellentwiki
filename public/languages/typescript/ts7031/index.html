TS7031 — Binding element ‘X’ implicitly has an ‘any’ type

TS7031 occurs when destructured parameters in a function don’t have type annotations and noImplicitAny is enabled. This is similar to TS7006 but specifically targets destructured binding elements.

Common Causes

// Cause 1: Destructured function parameter without types
function greet({ name, age }) { // TS7031: 'name' implicitly has 'any' type
  return `Hello ${name}, you are ${age}`;
}

// Cause 2: Destructured in arrow functions
const process = ({ id, label }) => { // TS7031
  console.log(id, label);
};

// Cause 3: Destructured in method parameters
class Service {
  handle({ data, status }) { // TS7031
    return data;
  }
}

// Cause 4: Destructured with defaults but no type
function configure({ host = "localhost", port = 3000 }) { // TS7031
  console.log(host, port);
}

Solutions

Fix 1: Annotate the destructured parameter with an interface

interface User {
  name: string;
  age: number;
}

// Wrong
function greet({ name, age }) {}

// Correct
function greet({ name, age }: User) {}

Fix 2: Use inline type annotations

// Wrong
function greet({ name, age }) {}

// Correct
function greet({ name, age }: { name: string; age: number }) {}

Fix 3: Type destructured defaults

// Wrong
function configure({ host = "localhost", port = 3000 }) {}

// Correct
function configure({ host = "localhost", port = 3000 }: { host?: string; port?: number }) {}

Fix 4: Type arrow function parameters

// Wrong
const process = ({ id, label }) => {};

// Correct
const process = ({ id, label }: { id: number; label: string }) => {};

Examples

// Example 1: React component props
function Button({ onClick, label, disabled }) { // TS7031
  return <button onClick={onClick} disabled={disabled}>{label}</button>;
}

// Fix: type the props
interface ButtonProps {
  onClick: () => void;
  label: string;
  disabled?: boolean;
}
function Button({ onClick, label, disabled }: ButtonProps) {}
