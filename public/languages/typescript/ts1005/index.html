TS1005 — ‘;’ expected

TS1005 is a parse error indicating TypeScript encountered a token where it expected a semicolon. This usually means there’s a syntax issue with how statements, declarations, or expressions are structured.

Common Causes

// Cause 1: Missing semicolon after variable declaration
let x = 10
let y = 20  // TS1005 on this line — expected ';' after declaration

// Cause 2: Missing semicolon in for loop
for (let i = 0; i < 10; i++)
{  // TS1005 — expected ';' after for statement

// Cause 3: Missing semicolon after interface property
interface User {
  name: string
  age: number  // TS1005 if strict semicolon enforcement
}

// Cause 4: Missing semicolon in import statement
import { helper } from "./utils"
import { other } from "./other"  // TS1005

Solutions

Fix 1: Add missing semicolons

// Wrong
let x = 10
let y = 20

// Correct
let x = 10;
let y = 20;

Fix 2: Use a linter to enforce semicolons

// .eslintrc.json
{
  "rules": {
    "semi": ["error", "always"]
  }
}

Fix 3: Check for missing closing brackets or braces

// Wrong — missing closing parenthesis
const result = add(1, 2
// TS1005: expected ';' — really means missing ')'

// Correct
const result = add(1, 2);

Fix 4: Fix malformed expressions

// Wrong — double operator
let x = 10 ++ 5; // TS1005

// Correct
let x = 10 + 5;

Examples

// Example 1: Accidentally using a keyword
let class = "myClass"; // TS1005: ';' expected (class is reserved)

// Example 2: Forgetting semicolon in chained method
const result = items
  .filter(x => x.active)
  .map(x => x.name)  // OK with ASI
  .join(", ");        // TS1005 if context requires semicolon

// Fix: add semicolon at end
const result = items
  .filter(x => x.active)
  .map(x => x.name)
  .join(", ");
