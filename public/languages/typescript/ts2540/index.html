TS2540 — Cannot assign to ‘X’ because it is a read-only property

TS2540 occurs when you try to modify a value that’s marked as readonly, const, or comes from a Readonly<T> type. TypeScript prevents mutation of these values at compile time.

Common Causes

// Cause 1: Assigning to a const variable
const x = 10;
x = 20; // TS2540: Cannot assign to 'x' because it is a constant

// Cause 2: Modifying a readonly property
interface Config {
  readonly host: string;
}
const config: Config = { host: "localhost" };
config.host = "other.com"; // TS2540

// Cause 3: Modifying a readonly array
const arr: readonly number[] = [1, 2, 3];
arr.push(4); // TS2540

// Cause 4: Modifying a property in Readonly<T>
interface User {
  name: string;
}
const user: Readonly<User> = { name: "Alice" };
user.name = "Bob"; // TS2540

Solutions

Fix 1: Use ’let’ instead of ‘const’ for mutable variables

// Wrong
const x = 10;
x = 20; // TS2540

// Correct
let x = 10;
x = 20; // OK

Fix 2: Remove ‘readonly’ from the property

// Wrong
interface Config {
  readonly host: string;
}

// Correct — if mutation is intended
interface Config {
  host: string;
}

Fix 3: Create a mutable copy

// Wrong
const arr: readonly number[] = [1, 2, 3];
arr.push(4); // TS2540

// Correct
const arr: readonly number[] = [1, 2, 3];
const mutableArr = [...arr]; // Spread into mutable array
mutableArr.push(4);

Fix 4: Use a type assertion to bypass readonly (use carefully)

// Use only when you're certain mutation is safe
const config: Readonly<Config> = { host: "localhost" };
(config as Config).host = "other.com"; // Bypasses readonly

Examples

// Example 1: Readonly class property
class ApiClient {
  readonly baseUrl = "https://api.example.com";
}
const client = new ApiClient();
client.baseUrl = "https://other.com"; // TS2540

// Example 2: Object.freeze returns Readonly
const frozen = Object.freeze({ x: 1, y: 2 });
frozen.x = 10; // TS2540

// Example 3: Tuple assignment
const tuple: readonly [string, number] = ["hello", 42];
tuple[0] = "world"; // TS2540

  • TS2322: Type is not assignable — broader type issues.
