TS2820 — Action ‘X’ is not a recognized action

TS2820 occurs when you use an unrecognized action property in a RequestInit or similar fetch-related configuration. The action property expects specific HTTP method strings, and TypeScript validates that the value is one of the recognized methods.

Common Causes

// Cause 1: Typo in HTTP method
fetch("/api", { action: "get" }); // TS2820: 'get' is not a recognized action

// Cause 2: Using lowercase instead of uppercase
fetch("/api", { method: "get" }); // TS2820 if type expects uppercase

// Cause 3: Custom action string not in the union
fetch("/api", { action: "CUSTOM_ACTION" }); // TS2820

// Cause 4: Wrong property name
fetch("/api", { action: "GET" }); // TS2820 if 'action' isn't a valid property

Solutions

Fix 1: Use the correct HTTP method

// Wrong
fetch("/api", { action: "get" });

// Correct — use 'method' instead of 'action'
fetch("/api", { method: "GET" });

Fix 2: Use valid HTTP methods

// Wrong
fetch("/api", { method: "get" }); // lowercase

// Correct — uppercase
fetch("/api", { method: "GET" });

Fix 3: Use the correct RequestInit property

// Wrong
fetch("/api", { action: "POST" });

// Correct
fetch("/api", { method: "POST" });

Fix 4: Type the request options properly

const options: RequestInit = {
  method: "GET",
  headers: {
    "Content-Type": "application/json",
  },
};
fetch("/api", options);

Examples

// Example 1: Confusing 'action' with 'method'
fetch("/api/users", {
  action: "POST", // TS2820
  body: JSON.stringify({ name: "Alice" }),
});

// Fix:
fetch("/api/users", {
  method: "POST",
  body: JSON.stringify({ name: "Alice" }),
});

// Example 2: GraphQL fetch with custom options
fetch("/graphql", {
  method: "POST", // Not 'action'
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ query: "{ user { name } }" }),
});
