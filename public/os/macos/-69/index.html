macOS Error -69 (dskFulErr) — Disk Full

Error -69 (dskFulErr) indicates the disk is full and cannot accept new data. On macOS, this appears when writing files, installing applications, saving documents, or when system processes need temporary space. The error can come from Finder, Terminal, or any application performing file I/O. macOS may also show a system-level “Your disk is almost full” warning before reaching this point.

Common Causes

  • Startup disk has less than 1 GB of free space
  • User Downloads or Documents folder is very large
  • Application caches (~/Library/Caches) consuming space
  • Time Machine local snapshots using disk space
  • Docker Desktop images and containers
  • Xcode derived data and build artifacts
  • Old iOS device backups in ~/Library/Application Support/MobileSync
  • System logs and analytics data accumulating

How to Fix Error -69

1. Check Available Disk Space

Determine how much space is free:

# Check disk usage in human-readable format
df -h /

# Show disk usage by directory
du -sh ~/Desktop ~/Downloads ~/Documents ~/Movies

# Use diskutil for a detailed breakdown
diskutil info /

2. Use the macOS Storage Management Tool

macOS has a built-in storage management GUI:

# Open the storage management window
open "x-apple.systempreferences:com.apple.settings.Storage"

This tool shows recommendations including:

  • Store in iCloud
  • Optimize Storage
  • Reduce Clutter
  • Empty Trash automatically

3. Clear User Caches

Caches are safe to delete and often large:

# Check cache size
du -sh ~/Library/Caches

# Remove all user caches
rm -rf ~/Library/Caches/*

# Remove specific application caches
rm -rf ~/Library/Caches/com.apple.Safari
rm -rf ~/Library/Caches/com.spotify.client

4. Empty the Trash

Trash may still hold disk space:

# Check Trash size
du -sh ~/.Trash

# Empty Trash
rm -rf ~/.Trash/*

# Also check for trashed items in other volumes
ls -la /Volumes/*/.Trashes/

5. Clear Time Machine Local Snapshots

Local snapshots can consume gigabytes:

# List local snapshots
tmutil listlocalsnapshots /

# Delete all local snapshots
for snapshot in $(tmutil listlocalsnapshots / | grep "com.apple"); do
    tmutil deletelocalsnapshots "$snapshot"
done

6. Remove Old iOS Backups

iOS backups in ~/Library/Application Support/MobileSync/Backup/ can be very large:

# Check backup size
du -sh ~/Library/Application\ Support/MobileSync/Backup/*

# Remove old backups
rm -rf ~/Library/Application\ Support/MobileSync/Backup/*

7. Clean Up Xcode Derived Data

If you use Xcode, derived data accumulates quickly:

# Check Xcode derived data size
du -sh ~/Library/Developer/Xcode/DerivedData

# Remove all derived data
rm -rf ~/Library/Developer/Xcode/DerivedData/*

# Remove old device support files
rm -rf ~/Library/Developer/Xcode/iOS\ DeviceSupport/*

# Remove archived builds
rm -rf ~/Library/Developer/Xcode/Archives/*

8. Clear System Logs and Analytics

# Check log sizes
du -sh /var/log
du -sh ~/Library/Logs

# Remove old system logs
sudo find /var/log -name "*.gz" -delete
sudo find /var/log -name "*.log" -size +100M -exec truncate -s 0 {} \;

# Remove user logs
find ~/Library/Logs -type f -delete

# Remove analytics data
rm -rf ~/Library/Application\ Support/CrashReporter/*

9. Use diskutil for Advanced Storage Management

# List all disks and volumes
diskutil list

# Verify disk for errors
diskutil verifyVolume /

# Repair if needed
diskutil repairVolume /

# Check APFS container space
diskutil apfs list

10. Remove Unused Docker Resources

Docker Desktop can consume significant space:

# Check Docker disk usage
docker system df

# Prune all unused resources
docker system prune -a --volumes

# Remove all unused images
docker image prune -a
