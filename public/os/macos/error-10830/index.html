macOS Error -10830 (kLSDataUnavailable) — Application Not Trusted

Error -10830 (kLSDataUnavailable / errLSAppNotTrusted) indicates that Launch Services has determined the application is not trusted by the system. This error occurs when macOS security mechanisms, such as Gatekeeper or code signing verification, have flagged an application as untrusted. The application may fail to launch, open files, or register with Launch Services until the trust issue is resolved.

Common Causes

  • The application is not signed by a recognized developer certificate
  • Gatekeeper has blocked the application from running
  • The application’s code signature is invalid or expired
  • The application was downloaded from the internet and hasn’t been approved by the user
  • The developer certificate has been revoked by Apple
  • The application has been modified after signing, invalidating its signature

How to Fix Error -10830

1. Override Gatekeeper for the Application

Approve the application through System Preferences or Terminal:

# Open System Preferences > Security & Privacy > General
# Click "Open Anyway" next to the blocked application message

# Or use Terminal to remove the quarantine attribute
xattr -d com.apple.quarantine /Applications/YourApp.app

# For an entire app bundle
xattr -rd com.apple.quarantine /Applications/YourApp.app

2. Verify the Code Signature

Check the application’s code signing status:

# Check if the app is signed
codesign -dv /Applications/YourApp.app

# Verify the signature in detail
codesign --verify --deep --strict /Applications/YourApp.app

# Check if the signature is valid
codesign --verify --verbose /Applications/YourApp.app

3. Check Gatekeeper Settings

Verify and adjust Gatekeeper settings:

# Check Gatekeeper status
spctl --status

# Assess a specific application
spctl --assess --verbose /Applications/YourApp.app

# Temporarily allow apps from anywhere (not recommended for production)
sudo spctl --master-disable

# Re-enable after installing
sudo spctl --master-enable

4. Reinstall from a Trusted Source

Download a fresh copy from a trusted source:

# Remove the current version
rm -rf /Applications/YourApp.app

# Download from the Mac App Store or the developer's official website
# After downloading, remove quarantine before first launch
xattr -d com.apple.quarantine ~/Downloads/YourApp.app

# Move to Applications
mv ~/Downloads/YourApp.app /Applications/

# Launch once to register
open -a "YourApp"

5. Check if the Developer Certificate is Revoked

Apple may revoke certificates for malicious software:

# Check certificate status (requires the certificate's serial number)
# Use Apple's revocation check via spctl
spctl --assess --verbose /Applications/YourApp.app 2>&1

# If revoked, you must uninstall the application immediately

6. Reset Security Settings

If Gatekeeper is behaving incorrectly:

# Clear Gatekeeper's approval database
sudo spctl --reset-default

# Clear the system policy database
sudo rm -rf /var/db/SystemPolicy

# Restart your Mac to apply changes
sudo reboot

Examples

This error commonly occurs when:

  • Installing apps from developers who don’t have an Apple Developer ID
  • Opening apps that have been modified by antivirus software
  • Running apps downloaded from non-App Store sources
  • Opening an app whose developer certificate has expired
  • Trying to use an app after macOS update changed security policies
