macOS Error -10844 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10844 (kLSDataUnavailable) indicates that Launch Services encountered a data access failure while processing a request. This error in the -1083x family typically occurs when the system cannot read the application registration database, often due to permission issues, filesystem corruption, or concurrent modification. It is commonly seen when multiple processes try to update Launch Services data simultaneously.

Common Causes

  • Concurrent processes modifying Launch Services data
  • Permission issues on the Launch Services data directory
  • Filesystem corruption preventing data access
  • The Launch Services daemon is in an inconsistent state
  • A system update is in progress and files are locked
  • Third-party software interfering with Launch Services operations

How to Fix Error -10844

1. Rebuild Launch Services

Force a clean rebuild:

# Kill existing processes
pkill -f lsregister

# Rebuild from scratch
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder and Dock
killall Finder
killall Dock

2. Check and Fix Permissions

Ensure proper access to Launch Services data:

# Check permissions on the support directory
ls -la /System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/

# Ensure lsregister is executable
chmod +x /System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister

# Check home directory permissions
ls -la ~ | head -5

3. Clear System Preferences Cache

Corrupted preferences can cause data access issues:

# Restart the preferences daemon
sudo launchctl kickstart -k system/com.apple.cfprefsd

# Clear user preferences cache
rm -rf ~/Library/Preferences/com.apple.LaunchServices*

# Restart Finder
killall Finder

4. Wait for System Updates to Complete

If a system update is in progress:

# Check if Software Update is running
ps aux | grep -i softwareupdate

# Wait for it to complete, or check status
softwareupdate -l

# If stuck, try:
sudo killall softwareupdated

5. Check for Conflicting Processes

Identify processes that may be interfering:

# List processes that might access Launch Services
ps aux | grep -i "launch\|register\|lsregister"

# Check for malware or unwanted software
# Use Activity Monitor or:
ps aux | grep -v "$(whoami)" | grep -i "system"

6. Restart in Safe Mode

Safe mode clears caches and verifies the system:

  1. Shut down your Mac
  2. Press and hold the power button until startup options appear
  3. Select your startup disk
  4. Hold the Shift key and click “Continue in Safe Mode”
  5. Once logged in, restart normally

Examples

This error commonly occurs when:

  • Running batch file operations that trigger many app registrations
  • When multiple terminal windows are opening files simultaneously
  • During a macOS update that modifies system frameworks
  • When using automation tools that interact with Launch Services
  • After installing applications while other apps are being updated
