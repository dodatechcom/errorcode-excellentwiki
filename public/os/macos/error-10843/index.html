macOS Error -10843 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10843 (kLSDataUnavailable) indicates that Launch Services is unable to access or process its internal data during a request. This error in the -1083x family typically occurs when Launch Services encounters a read error while accessing its persistent storage, or when the data has been corrupted by filesystem issues. It is often seen on systems with failing storage hardware.

Common Causes

  • Failing storage hardware causing read errors
  • Filesystem corruption on the system volume
  • The Launch Services data file is partially corrupted
  • Disk I/O errors during Launch Services operations
  • The APFS snapshot containing Launch Services data is damaged
  • Insufficient disk space for Launch Services to write data

How to Fix Error -10843

1. Check Disk Health

Verify the storage hardware is healthy:

# Check S.M.A.R.T. status
diskutil info disk0 | grep SMART

# For detailed health information
# Install smartmontools: brew install smartmontools
sudo smartctl -a disk0

# Check for I/O errors in system logs
log show --predicate 'eventMessage contains "I/O error"' --last 24h

2. Run Disk Utility First Aid

Repair filesystem corruption:

# Verify the system volume
diskutil verifyVolume disk1s1

# Check for APFS container issues
diskutil apfs list
diskutil apfs verifyContainer disk1

# For repair, boot into Recovery mode and run First Aid

3. Check Available Disk Space

Ensure there’s sufficient space:

# Check disk space
df -h

# If low on space, clear unnecessary files
sudo rm -rf /private/var/log/asl/*.asl
rm -rf ~/Library/Caches/*
sudo rm -rf /Library/Caches/*

4. Rebuild Launch Services

Force a complete rebuild:

# Kill existing processes
pkill -f lsregister

# Full rebuild
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder

5. Check for Disk Errors

Run a deeper disk check:

# Check filesystem journal
fsck_apfs -n /dev/disk1s1

# If errors found, boot into Recovery and run:
# fsck_apfs -fy /dev/disk1s1

# Or use Disk Utility First Aid

6. Monitor Storage Performance

Check if storage is causing the issue:

# Check disk I/O statistics
iostat -c 3

# Check for disk errors in kernel logs
log show --predicate 'eventMessage contains "disk"' --last 1h | grep -i error

Examples

This error commonly occurs when:

  • On Macs with failing SSDs or hard drives
  • After a sudden power loss during disk operations
  • When the system volume has filesystem corruption
  • On systems with very low disk space
  • After disconnecting a drive while writing data
