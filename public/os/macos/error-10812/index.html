macOS Error -10812 (kLSApplicationNotFound) — Launch Services No Handler

Error -10812 (kLSApplicationNotFound / errLSNoHandlers) indicates that Launch Services found no registered handler (application) for the requested operation. Unlike error -10810, which means the application itself is missing, error -10812 specifically means no application has been registered to handle the URL scheme, file type, or UTI at all. This often occurs with custom or rare file types.

Common Causes

  • No application on the system claims to handle the specific UTI or URL scheme
  • The file type is not registered in the system’s UTI database
  • The application was installed but never opened (required for registration)
  • A custom application does not properly declare its supported file types in Info.plist
  • The URL scheme was never registered by any application
  • Application was installed via drag-and-drop but not moved to /Applications

How to Fix Error -10812

1. Identify the File Type or URL Scheme

Determine what type of file or URL is triggering the error:

# Check the UTI of a file
mdls -name kMDItemContentType /path/to/file

# Check the file extension
file /path/to/file

# For URL schemes, check what you're trying to open
# Example: myapp://something

2. Register an Application for the File Type

Open the file with an application that can handle it:

# Open with a specific application (registers the association)
open -a "TextEdit" /path/to/file

# For URL schemes, use open command
open "myapp://some-path"

3. Open the Application at Least Once

Some applications require being opened once to register with Launch Services:

# Open the application
open -a "ApplicationName"

# Verify it's registered
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump | grep "ApplicationName"

4. Set Default Application via Get Info

  1. Right-click the file in Finder
  2. Select “Get Info”
  3. Under “Open with,” choose an application
  4. Click “Change All” to apply to all files of this type

5. Install a Compatible Application

For common file types without handlers:

# For PDF files, macOS should have Preview installed
open -a "Preview" /path/to/file.pdf

# For plain text, use TextEdit
open -a "TextEdit" /path/to/file.txt

# Install Homebrew Cask applications for other types
brew install --cask vlc        # for media files
brew install --cask gimp       # for image files

6. Rebuild Launch Services

If an application should be registered but isn’t:

# Force Launch Services rebuild
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder

Examples

This error commonly occurs when:

  • Opening a file with a custom extension (.myapp, .custom, etc.)
  • Clicking a URL with a custom scheme (myapp://) when no app handles it
  • Trying to open a file type from a newly installed application that hasn’t been opened yet
  • Opening files created by specialized software that isn’t installed
  • Clicking links in documents that use non-standard URL schemes
