macOS Error -10813 (kLSDataUnavailable) — Launch Services No Suitable Handler

Error -10813 (kLSDataUnavailable / errLSNoSuitableHandlers) indicates that Launch Services could not find a suitable handler for the requested operation. While error -10812 means no handler exists at all, error -10813 means a handler may exist but it is not suitable for the current context — for example, the application exists but is not compatible with the current macOS version, or the application is restricted from handling the request.

Common Causes

  • The registered handler application is incompatible with the current macOS version
  • The application is restricted by sandbox or security policies
  • The application requires a specific version not present on the system
  • App Store apps have been restricted from handling certain file types
  • The handler application exists but has been quarantined or blocked by Gatekeeper
  • Launch Services has stale data about available handlers

How to Fix Error -10813

1. Clear and Rebuild Launch Services

Force Launch Services to re-evaluate available handlers:

# Kill any existing Launch Services processes
pkill -f lsregister

# Force complete rebuild
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder

2. Remove Quarantine from the Application

Gatekeeper may be blocking the handler:

# Check quarantine status
ls -l@ /Applications/YourApp.app

# Remove quarantine attribute
xattr -d com.apple.quarantine /Applications/YourApp.app

# For entire app bundle
xattr -rd com.apple.quarantine /Applications/YourApp.app

3. Check Application Compatibility

Verify the application is compatible with your macOS version:

# Check the app's minimum system version
defaults read /Applications/YourApp.app/Contents/Info.plist LSMinimumSystemVersion

# Check your macOS version
sw_vers

# Check the app's architecture (Apple Silicon vs Intel)
file /Applications/YourApp.app/Contents/MacOS/*

4. Reinstall the Application

A fresh installation may resolve handler registration issues:

# Remove the application
rm -rf /Applications/YourApp.app

# Reinstall from App Store or developer website
# Then open the app once to register it
open -a "YourApp"

5. Manually Set the Default Application

Override Launch Services by manually setting the default:

# Use duti to set default UTI handlers (install via: brew install duti)
duti -s com.yourapp.bundleid "public.filename-extension" all

# Or use the Finder Get Info panel
# Right-click file > Get Info > Open with > Change All

6. Reset Security & Privacy Settings

If Gatekeeper is blocking the handler:

# Check Gatekeeper status
spctl --status

# Allow app from anywhere (use with caution)
sudo spctl --master-disable

# After installing, re-enable Gatekeeper
sudo spctl --master-enable

Examples

This error commonly occurs when:

  • Opening a file with an app that requires a newer macOS version
  • Trying to use a newly installed app as a handler before it’s fully registered
  • Opening files with apps that have been moved but not reinstalled
  • Clicking URLs that should open in an App Store app that has been restricted
  • Opening files created by beta software that is no longer available
