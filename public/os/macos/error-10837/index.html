macOS Error -10837 (kLSDataUnavailable) — Launch Services Data Unavailable

Error -10837 (kLSDataUnavailable) indicates that Launch Services encountered a general data unavailability condition. This error occurs when the system cannot retrieve or process the metadata needed to perform a Launch Services operation, such as opening a file, launching an app, or resolving a URL scheme. It is a general-purpose error within the -1083x family that signals internal Launch Services data problems.

Common Causes

  • Launch Services internal database is corrupted or incomplete
  • Required metadata files are missing from the application bundle
  • The system volume has filesystem corruption affecting Launch Services
  • A macOS update left Launch Services in an inconsistent state
  • The Launch Services daemon has crashed or is unresponsive
  • Insufficient disk space prevents Launch Services from writing data

How to Fix Error -10837

1. Rebuild Launch Services Database

Force a complete rebuild of the Launch Services database:

# Kill any existing Launch Services processes
pkill -f lsregister

# Force complete rebuild
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder

2. Check Available Disk Space

Launch Services needs disk space to write its database:

# Check disk space
df -h

# Clear disk space if needed
# Remove old log files
sudo rm -rf /private/var/log/asl/*.asl

# Clear system caches
sudo rm -rf /Library/Caches/*
rm -rf ~/Library/Caches/*

3. Repair System Permissions

Corrupted system permissions can affect Launch Services:

# Reset ACLs on the home directory
chmod -N ~

# Reset extended attributes
xattr -c ~

# For system files, use Recovery mode Disk Utility First Aid

4. Clear System Caches

Remove corrupted cache files:

# Clear Launch Services specific cache
rm -rf ~/Library/Caches/com.apple.LaunchServices*
rm -rf /Library/Caches/com.apple.LaunchServices*

# Clear font cache
sudo atsutil databases -remove

# Restart the system
sudo reboot

5. Check the Launch Services Daemon

Verify the Launch Services daemon is running:

# Check if lsregister is responsive
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump > /dev/null

# If the command hangs or fails, restart the system
sudo reboot

6. Run First Aid in Recovery Mode

If the issue persists, the system volume may need repair:

  1. Shut down your Mac
  2. Boot into Recovery mode (Command+R on Intel, power button hold on Apple Silicon)
  3. Open Disk Utility
  4. Select “Macintosh HD” and run First Aid
  5. Restart after repair completes

Examples

This error commonly occurs when:

  • After a major macOS version upgrade
  • When the system volume is nearly full
  • After a system crash or power loss
  • When restoring from a Time Machine backup
  • Running multiple concurrent file operations
