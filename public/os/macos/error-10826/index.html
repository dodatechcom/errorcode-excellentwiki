macOS Error -10826 (kLSNotReadable) — Launch Services Not Readable

Error -10826 (kLSNotReadable / errLSAppInUnknownState) indicates that Launch Services cannot read the application or its metadata in its current state. This error typically occurs when the application bundle is corrupted, incomplete, or in a state that prevents Launch Services from parsing its Info.plist and other required files. It can also occur when the application is partially installed or damaged.

Common Causes

  • The application bundle is corrupted or incomplete
  • The Info.plist file is missing or unreadable
  • The application was partially overwritten during an update
  • File permissions prevent Launch Services from reading the app bundle
  • The application is in an unknown state after a crash or forced termination
  • The app was downloaded from the internet and is partially quarantined

How to Fix Error -10826

1. Verify the Application Bundle

Check if the application bundle is intact:

# Check the application's structure
ls -la /Applications/ApplicationName.app/Contents/

# Verify the Info.plist exists and is valid
plutil -lint /Applications/ApplicationName.app/Contents/Info.plist

# Check the application's code signature
codesign -dv /Applications/ApplicationName.app

2. Repair Application Permissions

Ensure Launch Services can read the application:

# Check current permissions
ls -la /Applications/ApplicationName.app

# Fix ownership
sudo chown -R root:wheel /Applications/ApplicationName.app

# Fix permissions
sudo chmod -R 755 /Applications/ApplicationName.app

# Remove quarantine attribute
xattr -rd com.apple.quarantine /Applications/ApplicationName.app

3. Reinstall the Application

A fresh copy resolves most bundle corruption issues:

# Remove the corrupted application
rm -rf /Applications/ApplicationName.app

# Remove application preferences
defaults delete com.developer.ApplicationName

# Remove application support files
rm -rf ~/Library/Application\ Support/ApplicationName

# Reinstall from App Store or developer website

4. Clear Launch Services Cache

Force Launch Services to re-evaluate the application:

# Clear the Launch Services cache
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -seed -R -domain local -domain system -domain user

# Restart Finder
killall Finder

5. Check Disk for Errors

Filesystem corruption can damage application bundles:

# Verify the disk
diskutil verifyVolume disk1s1

# If errors are found, boot into Recovery and run First Aid
# Or for the boot volume:
sudo fsck_apfs -fy /dev/disk1s1

6. Boot into Safe Mode

Safe mode verifies the system volume and rebuilds caches:

  1. Shut down your Mac
  2. Press and hold the power button until startup options appear
  3. Select your startup disk
  4. Hold the Shift key and click “Continue in Safe Mode”
  5. Once logged in, restart normally

Examples

This error commonly occurs when:

  • Opening an application that was interrupted during installation
  • Launching an app after a failed update
  • Opening an app that was partially deleted or moved
  • Running an application from a corrupted disk image
  • Trying to launch an app that crashed during its first launch
