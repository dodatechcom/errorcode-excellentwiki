[Solution] Error 0x000000ef Windows 11/10 — CRITICAL_PROCESS_DIED Fix

Error 0x000000ef is a BSOD STOP error with the bug check CRITICAL_PROCESS_DIED. It occurs when a critical system process terminates unexpectedly, and Windows cannot recover. The process most commonly involved is csrss.exe, wininit.exe, or smss.exe.

This error affects both Windows 10 and 11 and can occur during boot, during use, or in a recurring loop that prevents normal operation.

Description

The full error message typically reads:

“Your PC ran into a problem and needs to restart. We’re just collecting some error info, and then we’ll restart for you. Stop code: CRITICAL_PROCESS_DIED”

Or with the numeric code:

“STOP: 0x000000EF — A critical system process died”

This error means a process that Windows depends on for core functionality terminated. Unlike user-mode crashes, this cannot be recovered from, so Windows halts immediately.

Common scenarios include:

  • Boot loop — Windows starts, crashes with this error, restarts, and repeats
  • During heavy usage — A critical process crashes under system load
  • After a Windows Update — Corrupted system files from a failed update
  • After disk changes — Modifying partitions or dual-boot configurations

Common Causes

  1. Corrupted system files — Critical Windows binaries or their dependencies are damaged.
  2. Disk errors — Bad sectors prevent system processes from reading required files.
  3. Failed Windows Update — An incomplete update modified critical system components.
  4. Driver corruption — A faulty driver caused a critical process to crash.
  5. Malware — Malicious software modifying or replacing system processes.
  6. Improper shutdown — Power loss during critical write operations corrupted system files.

Solutions

Solution 1: Run SFC and DISM in Recovery Mode

Since CRITICAL_PROCESS_DIED often prevents normal booting, use the Windows Recovery Environment.

Boot from recovery media:

  1. Insert a Windows installation USB or recovery drive.
  2. Boot from the USB (change boot order in BIOS/UEFI).
  3. Select your language and click Repair your computer.
  4. Go to Troubleshoot > Advanced options > Command Prompt.

Run DISM first to repair the system image:

DISM /Image:C:\ /Cleanup-Image /RestoreHealth

Then run SFC:

sfc /scannow /offbootdir=C:\ /offwindir=C:\Windows

If you can access Safe Mode, run these directly:

DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow

Solution 2: Check Disk Health and Repair Errors

Disk corruption is a frequent cause of this error. Run Check Disk from the recovery Command Prompt:

chkdsk C: /f /r /x

The /r flag recovers readable data from bad sectors. This can take 30–90 minutes on large drives.

To check disk health status without repairing:

Get-PhysicalDisk | Select-Object FriendlyName, HealthStatus, OperationalStatus

For SSD health:

Get-StorageReliabilityCounter -PhysicalDisk (Get-PhysicalDisk) | Select-Object Temperature, Wear, ReadErrorsTotal

Replace any disk showing Unhealthy status.

Solution 3: Boot into Safe Mode

If you can reach the Advanced Startup screen:

  1. Restart and hold Shift while clicking Restart.
  2. Go to Troubleshoot > Advanced options > Startup Settings > Restart.
  3. Press 4 or F4 to enter Safe Mode.

If Safe Mode works, the issue is likely a driver or software conflict. In Safe Mode:

  1. Uninstall recently installed drivers via Device Manager.
  2. Uninstall recently installed applications.
  3. Run a full antivirus scan.

Solution 4: Use System Restore

Revert the system to a point before the error started:

  1. Boot into Advanced Startup Options.
  2. Go to Troubleshoot > Advanced options > System Restore.
  3. Select a restore point from before the crashes began.
  4. Follow the prompts and allow the restoration to complete.

From recovery Command Prompt:

rstrui.exe

List available restore points:

Get-ComputerRestorePoint | Select-Object SequenceNumber, CreationTime, Description | Format-Table

Solution 5: Rebuild Boot Configuration

Corrupted BCD can cause critical processes to fail during boot:

From the recovery Command Prompt:

bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd

If the BCD store is severely corrupted:

bcdboot C:\Windows /s C: /f ALL

Solution 6: Check for and Remove Malware

Malware that modifies system processes can cause this error. From Safe Mode, run Windows Defender offline scan:

Start-MpScan -ScanType QuickScan
Start-MpScan -ScanType FullScan

For a boot-time scan that runs before Windows fully loads:

  1. Open Windows Security > Virus & threat protection.
  2. Click Scan options > Microsoft Defender Offline scan.
  3. Click Scan now — the PC will restart and scan before booting.

Solution 7: Reset Windows as a Last Resort

If the system remains in a boot loop:

  1. Boot from Windows installation media.
  2. Select Repair your computer.
  3. Go to Troubleshoot > Reset this PC.
  4. Choose Keep my files to preserve personal data.
  5. Select Cloud download for a clean Windows image.
