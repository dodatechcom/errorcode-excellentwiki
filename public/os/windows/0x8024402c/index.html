[Solution] Error 0x8024402C Windows 11/10 — Windows Update Fix

Error 0x8024402C is a Windows Update connection error that prevents your computer from reaching Microsoft’s update servers. It affects both Windows 10 and 11 systems with network or proxy configuration issues.

This error typically means Windows Update cannot establish a connection to download or install updates. The root cause is almost always network-related — proxy settings, firewall rules, or internet connectivity problems.

Description

The full error message typically reads:

“There were problems installing some updates, but we’ll try again later. Error 0x8024402C”

Or in the Windows Update log:

“WU_E_PT_WINHTTP_CANNOT_CONNECT — WinHTTP SendRequest/ReceiveResponse failed.”

Error 0x8024402C maps to WU_E_PT_HTTP_REDIRECT_FAILED, which means Windows Update could not complete an HTTP connection to the update servers. This is typically caused by misconfigured proxy settings, network firewalls, or DNS resolution failures.

The error appears in these common scenarios:

  • Corporate networks — Proxy servers or firewalls block Windows Update traffic
  • VPN connections — VPN software redirects update traffic incorrectly
  • Home networks — Router or ISP configuration issues
  • After network changes — Switching between networks or changing DNS settings
  • Third-party firewalls — Security software blocking Windows Update ports

Common Causes

  1. Misconfigured proxy settings — A proxy server is set but not functioning or needed.
  2. Firewall blocking update ports — Windows Update requires ports 80 and 443 to be open.
  3. DNS resolution failure — The system cannot resolve Microsoft’s update server hostnames.
  4. VPN interference — VPN software reroutes traffic away from update servers.
  5. Third-party antivirus/firewall — Security programs intercept HTTP/HTTPS traffic.
  6. Corrupted Windows Update cache — Damaged cached files prevent successful connections.

Solutions

Solution 1: Reset Proxy Settings

Incorrect proxy settings are the most common cause of this error. Reset them to defaults:

netsh winhttp reset proxy

If you are behind a legitimate proxy server, import the settings from Internet Explorer:

netsh winhttp import proxy source=ie

Check current proxy configuration:

netsh winhttp show proxy

If the output shows a proxy that you do not need, reset it. If you are on a corporate network, contact your IT department before changing proxy settings.

Solution 2: Reset Windows Update Components

Clear the Windows Update cache and restart all related services:

net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver

Rename the cache folders so Windows recreates them:

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old
ren C:\Windows\System32\catroot2 catroot2.old

Restart the services:

net start wuauserv
net start cryptSvc
net start bits
net start msiserver

Try running Windows Update again after the restart.

Solution 3: Check Internet Connectivity

Verify that your system can reach Microsoft’s update servers:

ping download.windowsupdate.com

nslookup download.windowsupdate.com

If DNS resolution fails, try switching to a public DNS server:

netsh interface ip set dns "Wi-Fi" static 8.8.8.8
netsh interface ip add dns "Wi-Fi" 8.8.4.4 index=2

Replace "Wi-Fi" with your network interface name. Check your interface name with:

netsh interface show interface

Solution 4: Configure Firewall Rules

Windows Update requires access to specific Microsoft endpoints. Ensure your firewall allows them:

netsh advfirewall firewall add rule name="Windows Update (HTTP)" dir=out action=allow protocol=TCP remoteport=80
netsh advfirewall firewall add rule name="Windows Update (HTTPS)" dir=out action=allow protocol=TCP remoteport=443

If you use a third-party firewall, add exceptions for:

  • wuauclt.exe (Windows Update client)
  • svchost.exe (Windows Update service host)
  • Microsoft update domains: *.windowsupdate.com, *.update.microsoft.com, download.microsoft.com

Solution 5: Run Windows Update Troubleshooter

Windows includes a built-in troubleshooter for update issues:

  1. Open Settings (Win + I).
  2. Go to System > Troubleshoot > Other troubleshooters.
  3. Click Run next to Windows Update.
  4. Follow the on-screen prompts.

Or run it from the command line:

Get-TroubleshootingPack -Path "C:\Windows\diagnostics\WindowsUpdate" | Invoke-TroubleshootingPack

Solution 6: Temporarily Disable VPN and Third-Party Firewall

VPNs and third-party firewalls can redirect or block Windows Update traffic. Temporarily disable them:

  1. Disconnect your VPN connection from the system tray.
  2. Open your third-party firewall or antivirus and disable the firewall component.
  3. Run Windows Update.
  4. Re-enable your VPN and firewall immediately after.

For Windows Defender Firewall:

Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False

Re-enable it after testing:

Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True
