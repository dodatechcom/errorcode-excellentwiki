[Solution] Error 0x80070020 Windows 11/10 — Sharing Violation Fix

Error 0x80070020 is a Sharing Violation error that occurs when a process cannot access a file because another process is already using it. The file is locked, and Windows prevents simultaneous access to avoid data corruption.

This error affects both Windows 10 and 11 and commonly appears during file copy, move, delete, or rename operations, as well as during software installation and Windows Update.

Description

The full error message typically reads:

“The process cannot access the file because it is being used by another process.”

Or during file operations:

“Error 0x80070020 — The requested resource is in use.”

Error 0x80070020 maps to ERROR_SHARING_VIOLATION in Windows. When a file is opened with an exclusive lock by one process, other processes receive this error when attempting to access it. Windows uses file locks to maintain data integrity, but sometimes locks are not released properly.

Common scenarios include:

  • File copy/move operations — Cannot copy files that are currently open
  • File deletion — Unable to delete files being used by an application
  • Software installation — Installers failing to overwrite in-use files
  • Windows Update — Updates failing because system files are locked
  • Antivirus scanning — Security software locking files during real-time scanning

Common Causes

  1. Application file locks — An application has a file open and hasn’t released the lock.
  2. Background processes — Services or processes accessing the file in the background.
  3. Antivirus real-time scanning — Security software temporarily locking files during scans.
  4. Windows Explorer — File Explorer holding file handles due to preview or thumbnail generation.
  5. Improper shutdown — File locks not released after an unexpected power loss.
  6. Network file sharing — Remote users having files open on shared network drives.

Solutions

Solution 1: Close All Programs Using the File

Identify which process has the file locked and close it.

Using Resource Monitor:

  1. Press Ctrl + Shift + Esc to open Task Manager.
  2. Go to the Performance tab and click Resource Monitor at the bottom.
  3. Go to the CPU tab.
  4. In the Associated Handles section at the bottom, search for the filename.
  5. The process holding the lock will be listed. Right-click and End Process.

Using PowerShell to find the locking process:

# Replace with your file path
$FilePath = "C:\Path\To\File.ext"
Get-Process | ForEach-Object {
    $proc = $_
    try {
        $handles = $_ | Get-HandleCount
        $proc.Modules | ForEach-Object {
            if ($_.FileName -eq $FilePath) {
                Write-Output "Process: $($proc.Name) (PID: $($proc.Id))"
            }
        }
    } catch {}
}

Using Handle.exe from Sysinternals:

Download from Microsoft Sysinternals and run:

handle.exe "C:\Path\To\File.ext"

Solution 2: Restart Windows Explorer

Windows Explorer often holds file locks due to thumbnail previews or file indexing.

  1. Open Task Manager (Ctrl + Shift + Esc).
  2. Find Windows Explorer in the Processes tab.
  3. Right-click and select Restart.

Or via command line:

taskkill /f /im explorer.exe
start explorer.exe

Using PowerShell:

Stop-Process -Name explorer -Force
Start-Sleep -Seconds 2
Start-Process explorer

Solution 3: Restart Your Computer

A simple restart releases all file locks and clears stuck file handles.

shutdown /r /t 0

If the file is locked by a process that won’t terminate:

shutdown /r /f /t 0

The /f flag forces all running applications to close.

Solution 4: Disable Antivirus Temporarily

Antivirus real-time scanning can temporarily lock files during access.

Windows Defender:

Set-MpPreference -DisableRealtimeMonitoring $true

Re-enable after the operation:

Set-MpPreference -DisableRealtimeMonitoring $false

For third-party antivirus:

  1. Right-click the antivirus icon in the system tray.
  2. Select Disable or Pause protection for 15 minutes.
  3. Perform the file operation.
  4. Re-enable protection immediately.

Add the file or folder to the exclusion list to prevent future issues:

Add-MpExclusion -Path "C:\Path\To\Folder"

Solution 5: Use Safe Mode to Access Locked Files

Safe Mode loads minimal drivers and services, releasing most file locks.

  1. Press Win + R, type msconfig, press Enter.
  2. Go to the Boot tab.
  3. Check Safe boot and select Minimal.
  4. Click OK and restart.
  5. Perform the file operation in Safe Mode.
  6. Restart normally when done.

Solution 6: End Specific Background Processes

Common processes that hold file locks:

:: Stop Windows Search indexing
net stop WSearch

:: Stop Superfetch/SysMain
net stop SysMain

:: Stop Windows Update (if updating)
net stop wuauserv

After completing the file operation, restart the services:

net start WSearch
net start SysMain
net start wuauserv

Solution 7: Use Command Line to Force Delete

If you need to delete a locked file:

Using del with /F flag:

del /f "C:\Path\To\Locked\File.ext"

Using PowerShell Remove-Item:

Remove-Item -Path "C:\Path\To\Locked\File.ext" -Force

For directories with locked files:

Remove-Item -Path "C:\Path\To\Directory" -Recurse -Force

If standard methods fail, schedule deletion for next boot:

movefile /rename "C:\Path\To\File.ext" "C:\Path\To\PendingDelete"

Download movefile.exe from Sysinternals.
