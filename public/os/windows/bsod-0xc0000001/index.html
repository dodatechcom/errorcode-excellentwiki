[Solution] BSOD 0xc0000001 STATUS_UNSUCCESSFUL Windows 11/10 — Fixed

Error 0xc0000001 is a critical BSOD STOP error with the status code STATUS_UNSUCCESSFUL. It is a generic error code indicating that a requested operation failed. In the context of a BSOD, it typically means that a critical kernel-mode operation — such as a driver initialization, system service call, or boot process step — has failed without providing a more specific error code.

This error affects both Windows 10 and 11 and is most commonly encountered during boot, preventing Windows from starting.

Common Causes

  • Corrupted boot configuration — The Boot Configuration Data (BCD) is damaged or misconfigured, causing the boot process to fail.
  • Failed driver initialization — A critical driver fails to start during the boot sequence.
  • Corrupted system files — Essential Windows files are damaged and cannot be loaded.
  • Hardware failure — A failing storage device or other hardware component prevents the boot process from completing.

How to Fix

Repair Boot Configuration

Boot from a Windows installation USB or recovery drive:

  1. Select Repair your computer.
  2. Go to Troubleshoot > Advanced options > Command Prompt.
  3. Run:
bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd

If bootrec /fixboot fails:

bootsect /nt60 sys
bootrec /fixboot

Rebuild BCD from Scratch

If bootrec /rebuildbcd finds no installations:

bcdboot C:\Windows /s C: /f ALL

Replace C: with the correct drive letter if Windows is on a different partition.

Run System File Checker

From recovery Command Prompt:

sfc /scannow /offbootdir=C:\ /offwindir=C:\Windows

Run DISM

DISM /Image:C:\ /Cleanup-Image /RestoreHealth

Check Disk Health

chkdsk C: /f /r /x

Disk corruption is a common cause of boot failures with generic error codes.

Boot into Safe Mode

  1. Hold Shift and click Restart.
  2. Go to Troubleshoot > Advanced options > Startup Settings > Restart.
  3. Press 4 or F4 for Safe Mode.

If Safe Mode works, uninstall recently installed drivers or software.

Check for Device Errors

Get-WmiObject Win32_PnPEntity | Where-Object { $_.ConfigManagerErrorCode -ne 0 } | Select-Object Name, DeviceID, ConfigManagerErrorCode | Format-Table -AutoSize

Perform System Restore

rstrui.exe

Select a restore point dated before the error started.

Examples

This error commonly occurs in these scenarios:

  • During boot after a power loss — An unexpected shutdown corrupts the BCD or boot files.
  • After installing new hardware — A new device driver fails to initialize during boot.
  • Following a failed Windows Update — A corrupted update leaves the system in an unbootable state.
  • With a failing hard drive — Bad sectors prevent critical boot files from loading.
