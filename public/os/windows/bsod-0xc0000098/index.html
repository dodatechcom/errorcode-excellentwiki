[Solution] BSOD 0xc0000098 STATUS_INCONSISTENT_XRB Windows 11/10 — Fixed

Error 0xc0000098 is a critical BSOD STOP error with the status code STATUS_INCONSISTENT_XRB. It occurs when the boot loader encounters an inconsistent or corrupted Extended Root Boot (XRB) structure during the boot process. The boot configuration data is invalid, and Windows cannot continue loading.

This error affects both Windows 10 and 11 and almost always appears during boot, preventing Windows from starting.

Common Causes

  • Corrupted Boot Configuration Data (BCD) — The BCD store is damaged, misconfigured, or points to missing boot files.
  • Failed Windows Update — An interrupted update corrupts the boot configuration.
  • Disk corruption — Bad sectors on the system drive damage boot-related data structures.
  • Improper shutdown — A forced shutdown or power loss corrupts the boot volume metadata.

How to Fix

Rebuild BCD

Boot from a Windows installation USB or recovery drive:

  1. Select Repair your computer.
  2. Go to Troubleshoot > Advanced options > Command Prompt.
  3. Run the following commands in order:
bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd

If bootrec /rebuildbcd finds no Windows installations, rebuild the BCD manually:

bcdboot C:\Windows /s C: /f ALL

Replace C: with the correct drive letter.

Check and Repair Disk Errors

chkdsk C: /f /r /x

The /r flag locates bad sectors and recovers readable data. Boot-related structures stored on corrupted sectors will be repaired.

Run SFC from Recovery

sfc /scannow /offbootdir=C:\ /offwindir=C:\Windows

Run DISM from Recovery

DISM /Image:C:\ /Cleanup-Image /RestoreHealth

Fix Boot Sector Issues

If the boot sector is corrupted:

bootsect /nt60 sys
bootrec /fixboot

Restore the System from a Restore Point

From the recovery environment:

rstrui.exe

Select a restore point dated before the error started.

Verify BCD Integrity

bcdedit /enum all

Check the output for any entries that reference missing files or invalid paths. Remove invalid entries:

bcdedit /deletevalue {identifier} description

Examples

This error commonly occurs in these scenarios:

  • After a forced shutdown — Holding the power button during an update corrupts the boot configuration.
  • Following a failed Windows Update — An interrupted update leaves the BCD in an inconsistent state.
  • With a failing hard drive — Bad sectors corrupt the boot volume metadata.
  • After dual-boot configuration changes — Modifying Linux partitions or boot managers corrupts the Windows BCD.
