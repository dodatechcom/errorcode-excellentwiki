[Solution] Error 0x800b0101 Windows 11/10 — Certificate Expired Fix

Error 0x800b0101 is a Windows Update certificate error that occurs when the system cannot verify the digital signature of an update package because the certificate has expired or is not trusted. The update agent refuses to install unverified content.

This error affects both Windows 10 and 11 and commonly blocks Windows Defender definition updates, cumulative updates, and driver updates that use time-sensitive certificates.

Description

The full error message typically reads:

“Windows Update cannot currently check for updates, because the service is not running.”

Or more specifically:

“Error 0x800b0101 — A certificate chain could not be built to a trusted root authority.”

Or:

“Error 0x800b0101 — The required certificate is not within its validity period when verifying against the current system clock.”

Error 0x800b0101 maps to CRYPT_E_EXPIRED in Windows, indicating a certificate’s validity period doesn’t match the system’s current time. This can be caused by an expired root certificate, an incorrect system clock, or a missing intermediate certificate in the chain of trust.

Common scenarios include:

  • Windows Update failures — Updates rejected due to expired signing certificates
  • Windows Defender updates — Definition updates failing certificate validation
  • Microsoft Store — App installations blocked by certificate issues
  • SSL/TLS errors — Secure connections failing in browsers and apps
  • Code signing failures — Software installation blocked due to expired certificates

Common Causes

  1. Incorrect system time/date — The system clock is wrong, making valid certificates appear expired.
  2. Expired root certificates — The Windows root certificate store contains outdated certificates.
  3. Missing intermediate certificates — The certificate chain is incomplete.
  4. Corrupted certificate store — The Windows certificate database is damaged.
  5. Third-party software interference — Security tools modifying the certificate store.
  6. Enterprise certificate policies — Group Policy blocking certain certificate authorities.

Solutions

Solution 1: Sync System Time

An incorrect system clock is the most common cause. If the clock is ahead or behind, valid certificates may appear expired.

Sync time via Settings:

  1. Open Settings > Time & language > Date & time.
  2. Enable Set time automatically.
  3. Click Sync now under Synchronize your clock.

Sync time via command line:

w32tm /resync /force

Force a time sync with the internet time server:

w32tm /config /manualpeerlist:"time.windows.com" /syncfromflags:manual /reliable:YES /update
w32tm /resync /force

Using PowerShell:

# Check current time source
w32tm /query /status

# Force resync
w32tm /resync /rediscover

# Verify time difference
w32tm /stripchart /computer:time.windows.com /samples:5

Solution 2: Update Root Certificates

Outdated root certificates can prevent the system from building a valid certificate chain.

Allow Windows to automatically update certificates:

  1. Open Settings > Privacy & security > Windows Security.
  2. Go to Device security > Core isolation.
  3. Ensure Memory integrity is enabled (may require restart).

Manually update certificates via Windows Update:

wuauclt /detectnow /resetauthorization

Download the latest root certificate update from Microsoft:

Visit the Microsoft Root Certificate Program page and download the latest roots update.

Solution 3: Clear the Certificate Cache

Corrupted certificate cache entries can cause validation failures.

certutil -urlcache * delete
certutil -setreg CA\CRLPeriodUnits 1
certutil -setreg CA\CRLDeltaPeriodUnits 0

Reset the SSL state:

  1. Press Win + R, type inetcpl.cpl, press Enter.
  2. Go to the Content tab.
  3. Click Clear SSL State.
  4. You’ll see a confirmation that the SSL cache was cleared.

Using command line:

netsh sslcache flush

Solution 4: Repair the Certificate Store

If the certificate store itself is corrupted:

certutil -verifyStore

Re-register Windows Update DLLs:

regsvr32.exe /s cryptdlg.dll
regsvr32.exe /s initpki.dll
regsvr32.exe /s msasn1.dll
regsvr32.exe /s wintrust.dll
regsvr32.exe /s softpub.dll
regsvr32.exe /s dssenh.dll
regsvr32.exe /s rsaenh.dll

Reset the cryptographic services:

net stop cryptsvc
rd /s /q "C:\Windows\System32\catroot2"
net start cryptsvc

Solution 5: Run DISM and SFC

Corrupted system files can include certificate-related components:

DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow

Solution 6: Check Group Policy Certificate Settings

Enterprise environments may have certificate policies that block updates.

gpresult /h C:\gpreport.html

Open the report and look for certificate-related policies. Common ones:

  • Certificate Path Validation Settings
  • Trusted Root Certification Authorities

If you have admin access:

gpedit.msc

Navigate to Computer Configuration > Windows Settings > Security Settings > Public Key Policies and verify certificate settings.

Solution 7: Reset Windows Update Components

Combine certificate fixes with a full Windows Update reset:

net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver

certutil -urlcache * delete
ren "C:\Windows\SoftwareDistribution" "SoftwareDistribution.old"
ren "C:\Windows\System32\catroot2" "catroot2.old"
rd /s /q "C:\Windows\System32\catroot2"

net start cryptsvc
net start wuauserv
net start bits
net start msiserver

Restart your PC and check for updates.
