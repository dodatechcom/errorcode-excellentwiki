[Solution] Error 0x8024001d Windows 11/10 — Windows Update Installer Error Fix

Error 0x8024001d is a Windows Update installer error that prevents cumulative and feature updates from being applied. The update downloads successfully but fails during the installation phase.

This error affects both Windows 10 and 11 and typically appears when the Windows Update agent encounters a conflict with existing update components or pending operations.

Description

The full error message typically reads:

“Installation Failure: Windows failed to install the following update with error 0x8024001d.”

Or in the Windows Update history:

“Feature update to Windows 11, version 23H2 — Error 0x8024001d”

Error 0x8024001d maps to WU_E_UH_POSTREBOOTACTIONFAILED, meaning a post-reboot action required by the update agent failed to complete. This is often caused by interrupted update cycles, corrupted update databases, or services that didn’t restart cleanly.

Common scenarios include:

  • Cumulative updates failing during the install phase after download
  • Feature updates stalling at 99% then rolling back
  • Servicing stack updates refusing to apply
  • .NET Framework updates failing with installer errors

Common Causes

  1. Interrupted update cycle — A previous update was partially applied and the system rebooted prematurely.
  2. Corrupted SoftwareDistribution cache — The download cache contains incomplete or damaged update files.
  3. Windows Update services stuck — The wuauserv or trustedinstaller service is in a hung state.
  4. Pending restart operations — Previous updates left pending file replacements that conflict.
  5. Corrupted servicing stack — The servicing stack that manages update installation is damaged.

Solutions

Solution 1: Run the Windows Update Troubleshooter

Windows includes a built-in troubleshooter that can detect and fix common update issues automatically.

  1. Open Settings (Win + I).
  2. Go to System > Troubleshoot > Other troubleshooters.
  3. Click Run next to Windows Update.
  4. Follow the on-screen prompts and apply any recommended fixes.
  5. Restart your PC and try the update again.

For an automated approach, run the troubleshooter from the command line:

msdt.exe /id WindowsUpdateDiagnostic

Solution 2: Restart Windows Update Services

Stuck or hung services are the most common cause of this error. Restart them cleanly:

net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop trustedinstaller

Wait 30 seconds, then restart them:

net start wuauserv
net start cryptSvc
net start bits
net start msiserver

Using PowerShell for a more thorough restart:

$services = @("wuauserv", "cryptSvc", "bits", "msiserver", "trustedinstaller")
foreach ($svc in $services) {
    Stop-Service -Name $svc -Force -ErrorAction SilentlyContinue
    Start-Sleep -Seconds 5
    Start-Service -Name $svc -ErrorAction SilentlyContinue
}

Solution 3: Clear the SoftwareDistribution Folder

A corrupted download cache prevents the installer from reading update files properly.

net stop wuauserv
net stop bits

Rename the cache folder so Windows recreates it:

ren "C:\Windows\SoftwareDistribution" "SoftwareDistribution.old"

Restart the services:

net start wuauserv
net start bits

Windows will recreate the SoftwareDistribution folder automatically on the next update check.

Solution 4: Run DISM and SFC to Repair System Files

Corrupted system files can prevent the update installer from functioning.

DISM /Online /Cleanup-Image /RestoreHealth

Follow with System File Checker:

sfc /scannow

Both tools need administrative privileges. Run them sequentially and restart your PC after completion.

Solution 5: Delete Pending.xml to Clear Stuck Operations

If a previous update left a pending operations file, it can block new installations:

del "C:\Windows\WinSxS\pending.xml" /f /q
del "C:\Windows\WinSxS\poqexec.log" /f /q

If these files are locked, boot into Safe Mode and delete them from there:

  1. Press Win + R, type msconfig, press Enter.
  2. Go to the Boot tab and check Safe boot.
  3. Click OK and restart.
  4. Delete the files, then restart normally.

Solution 6: Manually Install the Update

If the automatic installer keeps failing, download and install the update manually:

  1. Note the KB number from the failed update in Windows Update history.
  2. Visit the Microsoft Update Catalog.
  3. Search for the KB number.
  4. Download the correct version for your system (x64 or ARM64).
  5. Run the downloaded .msu file as administrator.
