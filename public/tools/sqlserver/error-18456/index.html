Error 18456: Login Failed

Error 18456 occurs when SQL Server rejects a login attempt. The client cannot authenticate due to invalid credentials, disabled accounts, or misconfigured authentication mode.

Common Causes

  • Incorrect username or password
  • SQL Server is in Windows Authentication mode but SQL login is used
  • The SQL login is disabled or locked out
  • The login does not have CONNECT permission to the server

How to Fix

Enable SQL Server Authentication Mode

-- In SQL Server Management Studio
-- Right-click server > Properties > Security
-- Set to "SQL Server and Windows Authentication mode"

Create or Enable a SQL Login

-- Enable existing login
ALTER LOGIN sa ENABLE;
ALTER LOGIN sa WITH PASSWORD = 'new_password';

-- Create new login
CREATE LOGIN app_user WITH PASSWORD = 'SecureP@ss123';
GRANT CONNECT SQL TO app_user;

Check Login State

SELECT name, is_disabled, type_desc
FROM sys.server_principals
WHERE name = 'app_user';

Enable SA Account

ALTER LOGIN sa ENABLE;
ALTER LOGIN sa WITH PASSWORD = 'YourNewPassword123!';

Check Error Log for Details

-- The error state in the log reveals the specific reason
-- State 2: user ID not found
-- State 5: user ID not valid
-- State 8: wrong password

Examples

-- Wrong password
sqlcmd -S server -U sa -P wrongpass
-- Error 18456: Login failed for user 'sa'.

-- Windows auth when SQL auth expected
sqlcmd -S server -U sqluser -P pass
-- Error 18456: Login failed for user 'sqluser'
-- Fix: enable SQL Server Authentication mode
