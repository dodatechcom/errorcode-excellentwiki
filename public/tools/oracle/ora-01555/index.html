ORA-01555: Snapshot Too Old

ORA-01555 occurs when a long-running query needs to read data that has been overwritten in the undo (rollback) tablespace. The undo data needed for consistent read is no longer available.

Common Causes

  • The undo tablespace is too small for the workload
  • A long-running query references data modified by concurrent transactions
  • The UNDO_RETENTION setting is too low
  • The undo tablespace has autoextend disabled

How to Fix

Increase Undo Retention

ALTER SYSTEM SET undo_retention = 3600;  -- 1 hour

Resize the Undo Tablespace

ALTER DATABASE DATAFILE '/u01/oradata/undotbs01.dbf' RESIZE 20G;
-- or add a new datafile
ALTER TABLESPACE undotbs1 ADD DATAFILE '/u01/oradata/undotbs02.dbf'
    SIZE 10G AUTOEXTEND ON;

Enable Guaranteed Undo Retention

ALTER TABLESPACE undotbs1 RETENTION GUARANTEE;

Optimize Long-Running Queries

-- Break long queries into smaller batches
-- Instead of:
SELECT * FROM large_table;

-- Use:
SELECT * FROM large_table WHERE id BETWEEN 1 AND 100000;
SELECT * FROM large_table WHERE id BETWEEN 100001 AND 200000;

Check Current Undo Usage

SELECT tablespace_name, SUM(bytes) / 1024 / 1024 AS mb
FROM dba_undo_extents
GROUP BY tablespace_name;

Examples

-- Long-running report
SELECT /*+ PARALLEL(4) */ * FROM sales WHERE quarter = 'Q1';
-- ORA-01555: snapshot too old: rollback segment number 5 with name "_SYSSMU5$"
-- Fix: increase undo_retention or optimize the query

-- Concurrent DML during report
-- Session 1: long query
-- Session 2: DELETE FROM sales WHERE ...
-- Fix: use UNDO_RETENTION GUARANTEE
