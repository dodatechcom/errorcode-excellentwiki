ORA-12899: Value Too Large for Column

ORA-12899 occurs when an INSERT or UPDATE tries to store a value that is wider than the column’s defined size. Oracle enforces strict column length limits.

Common Causes

  • The value being inserted exceeds the column’s VARCHAR2/CHAR size
  • Multi-byte character encoding requires more bytes than expected
  • The column was defined with a smaller size than required
  • Implicit conversion from a larger type expands the data

How to Fix

Check Column Definitions

SELECT column_name, data_type, data_length, char_length
FROM user_tab_columns
WHERE table_name = 'USERS';

Widen the Column

ALTER TABLE users MODIFY email VARCHAR2(255);

Truncate the Value Before Inserting

INSERT INTO users (name, email)
VALUES ('Alice', SUBSTR('very_long_email@example.com', 1, 100));

Check Character Set Impact

-- AL32UTF8 uses up to 4 bytes per character
SELECT lengthb('café'), length('café') FROM dual;
-- 5 bytes, 4 characters

Use CLOB for Large Values

ALTER TABLE users ADD notes CLOB;

Examples

-- Column too small
INSERT INTO users (email) VALUES ('very.long.email.address@domain.com');
-- ORA-12899: value too large for column "APP"."USERS"."EMAIL" (actual: 30, maximum: 20)

-- Fix: ALTER TABLE users MODIFY email VARCHAR2(100);

-- Multi-byte character issue
INSERT INTO users (name) VALUES ('Übergrößenmaßstab');
-- ORA-12899 in UTF8: actual 38, maximum 20
-- Fix: increase column size to account for multi-byte characters
