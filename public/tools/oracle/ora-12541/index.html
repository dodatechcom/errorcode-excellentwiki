Home / Tools / ORA-12541: TNS:no listener

The ORA-12541: TNS:no listener error means the Oracle client cannot connect to the database server because no TNS listener is accepting connections on the specified host and port. This is a network-level issue, not a database authentication problem.

Common Causes

  • The Oracle listener service is not running on the database server
  • The host or port in the connection string is incorrect
  • A firewall is blocking the listener port (default: 1521)
  • The listener is bound to a different interface than the client is connecting to

How to Fix

Check if the listener is running on the server:

lsnrctl status

Start the listener if it is stopped:

lsnrctl start

Verify the listener port and configuration:

cat $ORACLE_HOME/network/admin/listener.ora

Test connectivity to the listener port:

tnsping your_database_host:1521

Examples

# Attempting to connect when the listener is down
sqlplus user/password@//dbhost:1521/ORCL
-- ERROR: ORA-12541: TNS:no listener

# Wrong port in connection string
sqlplus user/password@//dbhost:1522/ORCL
-- ERROR: ORA-12541: TNS:no listener

Confirm the listener is active:

lsnrctl status
# Output should show "The command completed successfully" with listening endpoints
